## 当前主线判断

Tony 的个人系统不是公司工程治理系统，而是：

```text
学习 -> 研究 -> 判断 -> 沉淀 -> 复用 -> 项目推进
```

所以入口也要按人的工作流来组织：

```text
今日入口 -> 知识导航门户 -> 领域/问题入口 -> 主题/项目/Playbook
```

## System Design Decision

The system has three durable layers:

1. Knowledge assets: long-term notes, maps, playbooks, and domain systems.
2. Human navigation: readable home pages, maps, learning paths, and project pages.
3. AI collaboration protocol: inbox boundaries, shared memory, agent roles, workflows, and review gates.

The current Agent design rule is:

```text
workflow first, persona second
```

Personal Agent names are capability aliases over Hermes / Codex workflows, not a second organizational layer.

The current cross-tool access model is:

```text
Obsidian = 个人知识生产中心
GitHub private repo = 长期版本事实源
output-feishu = 可发布中间层
飞书知识库 = 任意地方访问 / 移动端阅读 / 分享协作
飞书 CLI = 自动同步执行器
```
