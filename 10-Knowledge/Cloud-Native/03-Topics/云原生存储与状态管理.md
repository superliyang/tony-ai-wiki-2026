# 云原生存储与状态管理

## 一句话理解

云原生不只适合 stateless workloads。真正成熟的云原生平台，还必须回答状态、持久化、卷供应、数据位置和生命周期怎么管理。

## 为什么它值得单独拉出来

Kubernetes 最开始更容易让人联想到 stateless service，但真实系统里总会出现：

- 数据库
- 消息队列
- 对象存储
- 文件系统
- 需要持久卷和备份的 stateful workload

如果没有清晰的存储抽象，平台只会停在“跑应用”，无法进入更完整的基础设施层。

## 这条主线通常包括什么

- `PersistentVolume` / `PersistentVolumeClaim`
- `StorageClass`
- dynamic provisioning
- CSI drivers
- stateful workload patterns
- block / file / object storage 的不同抽象

## 学习重点

- `StorageClass` 代表的不只是一个卷配置，而是默认存储路径与供应策略
- CSI 是 Kubernetes 存储扩展的重要边界
- `Rook` 这类项目说明，云原生存储也能被操作化和平台化
