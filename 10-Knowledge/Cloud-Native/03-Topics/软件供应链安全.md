# 软件供应链安全

## 一句话理解

`Software Supply Chain Security` 关注的不是运行中的服务本身，而是软件从源码、构建、打包、签名、发布到部署这一整条链路是否可信。

## 为什么它在云原生里特别重要

云原生交付高度自动化，镜像、依赖、CI/CD、artifact registry 和 GitOps 都会进入生产链路。也正因为自动化程度高，一旦链路某个环节被污染，影响会被快速放大。

## 这条主线通常包括什么

- artifact signing
- provenance
- policy enforcement
- build integrity
- dependency and image trust
- SLSA 这类成熟度框架

## 学习重点

- 签名、验证、provenance 分别在解决什么问题
- Sigstore 为什么是高频入口
- SLSA 更像能力成熟度模型，而不是单一工具
- 为什么 supply chain security 最终要和 cluster policy 结合
