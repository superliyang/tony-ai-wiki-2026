# 服务自助接入工作流

## 简介

`服务自助接入工作流` 关注的是：开发者如何从“我要创建一个服务”走到“它被纳入平台标准路径”，中间经过 portal、template、catalog、docs 和基础治理。

## 输入

- service template choice
- owner and metadata
- runtime needs
- environment and deployment requirements

## 步骤

1. 在 developer portal 中选择模板
2. 填写 owner、repo、runtime、依赖等基础信息
3. 由 template 生成代码骨架、CI 配置、docs 和 metadata
4. 让服务进入 catalog，并接上默认观测、部署和治理路径
5. 对高风险能力附加 approval 或 policy checks

## 输出

- standardized service scaffold
- service metadata entry
- docs and ownership linkage
- default deployment and observability hooks

## 评估

- 新服务创建时间
- 标准化接入比例
- 平台默认路径 adoption
- 后续维护成本是否下降
