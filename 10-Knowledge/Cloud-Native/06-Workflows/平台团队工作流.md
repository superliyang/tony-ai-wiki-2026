# 平台团队工作流

## 简介

`平台团队工作流` 关注的是：平台团队如何把 catalog、templates、deployment workflow、policy 和 feedback loop 组织成一个长期运行的产品体系。

## 输入

- developer pain points
- platform standards and policies
- common service bootstrapping needs
- adoption and friction signals

## 步骤

1. 找出开发者高频、重复、容易出错的交付路径
2. 把这些路径抽象成 golden path 与 software template
3. 通过 portal / catalog 暴露给开发者
4. 用 policy、approval、ownership 和 docs 形成 guardrails
5. 追踪 adoption、feedback 和 friction，再持续迭代

## 输出

- developer portal entry points
- service templates
- onboarding docs
- owner and service metadata
- governance hooks

## 评估

- 新服务上线路径是否更短
- 开发者是否更少依赖人工 ticket
- 平台默认路径是否被采用
- governance 是否被内化到流程里
