# Backstage

## 定位

`Backstage` 是平台工程领域最有代表性的开源 framework 之一。它更像一个 developer portal / internal developer platform 的基础框架，而不是单一运维工具。

## 它解决什么问题

- 把服务、文档、模板、owner 和平台能力收进统一入口
- 降低微服务和基础设施分散后带来的认知混乱
- 让平台团队把 golden path、software templates、docs 和 catalog 产品化

## 为什么它重要

Backstage 把平台工程从“理念”拉到可见的产品层：开发者到底通过什么入口去发现服务、创建项目、查看文档、理解依赖、进入标准化工作流。

## 学习重点

- software catalog 在平台工程里扮演什么角色
- software templates 如何把 golden path 具体化
- TechDocs 为什么代表 `docs as code` 的平台化实践
- Backstage 更适合做入口层，而不是直接替代 Kubernetes、GitOps 或 observability 系统
