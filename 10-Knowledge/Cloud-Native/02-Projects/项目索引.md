## 这些项目为什么这样排

- `Kubernetes`：编排与控制平面的核心节点
- `Prometheus`：metrics 与 alerting 的代表项目
- `OpenTelemetry`：现代 observability 数据采集和标准化的关键节点
- `Argo CD`：GitOps 在实际平台里的高频入口
- `Istio`：service mesh 与流量治理的代表项目
- `Backstage`：platform engineering 的代表产品入口
- `Kyverno`：policy-as-code 与 cluster governance 的代表项目
- `Sigstore`：软件供应链安全的重要入口
- `Knative`：Kubernetes 之上的 serverless 与 event-driven 抽象
- `containerd`：节点执行层与 container runtime 的关键节点
- `Rook`：云原生存储平台化的重要入口
- `KServe`：AI 推理服务平台的重要入口
- `Kubeflow Trainer`：AI 训练作业编排的重要入口
- `KubeRay`：分布式 AI workload 接入 Kubernetes 的代表项目
