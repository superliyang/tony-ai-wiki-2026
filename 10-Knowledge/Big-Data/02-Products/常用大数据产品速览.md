# 常用大数据产品速览

## 这页怎么用

这页给后续聊天做产品导航，不做购买建议。

先把产品放到架构层，再讨论优缺点。

## 1. Cloud Warehouse / Data Platform

### Snowflake

- 位置：cloud data warehouse / data cloud
- 典型价值：SQL 分析、治理、数据共享、企业数据平台体验
- 主要问题：成本模型、生态 lock-in、与 lakehouse / open table 的边界
- 入口：[[../01-Companies/Snowflake|Snowflake]]

### BigQuery

- 位置：serverless cloud data warehouse
- 典型价值：低运维 SQL 分析、Google Cloud 集成、交互式分析
- 主要问题：云生态耦合、查询成本治理、数据建模和权限边界
- 入口：[[../01-Companies/Google BigQuery|Google BigQuery]]

## 2. Lakehouse Platform

### Databricks

- 位置：lakehouse platform
- 典型价值：Spark、Delta、notebook / workflow、ML / AI、catalog / governance 一体化
- 主要问题：平台复杂度、成本、与 warehouse / open table 的边界
- 入口：[[../01-Companies/Databricks|Databricks]]

### Iceberg / Delta / Hudi

- 位置：open table format / lakehouse table layer
- 典型价值：snapshot、schema evolution、partition evolution、事务语义、多引擎复用
- 主要问题：catalog、compaction、权限、生态兼容和运维责任
- 入口：[[../05-Topics/Apache Iceberg 与 Lakehouse 表格式|Apache Iceberg 与 Lakehouse 表格式]]

## 3. Streaming / Event Platform

### Kafka / Confluent

- 位置：event log / streaming platform
- 典型价值：事件日志、多消费者、回放、实时数据集成
- 主要问题：topic 设计、schema、retention、消费幂等、运维复杂度
- 入口：[[../05-Topics/Kafka 与事件日志|Kafka 与事件日志]]、[[../01-Companies/Confluent|Confluent]]

### Flink

- 位置：stateful stream processing
- 典型价值：event time、状态、窗口、实时特征和实时指标
- 主要问题：状态膨胀、迟到数据、checkpoint、实时/离线对齐
- 入口：[[../05-Topics/Flink 与流处理|Flink 与流处理]]

## 4. Batch / Compute / Query

### Spark

- 位置：batch compute / distributed processing
- 典型价值：ETL、历史重算、大表 join、特征和训练数据构建
- 主要问题：shuffle、数据倾斜、小文件、重跑和成本
- 入口：[[../05-Topics/Spark 与批处理|Spark 与批处理]]

### Trino / Presto

- 位置：interactive SQL query engine
- 典型价值：跨数据源交互式查询、lakehouse 查询、ad hoc analysis
- 主要问题：数据源性能差异、权限、并发、成本归因
- 状态：待拆页

### ClickHouse / Druid / Pinot

- 位置：OLAP serving / real-time analytics
- 典型价值：低延迟分析、高并发看板、实时聚合查询
- 主要问题：建模方式、写入模式、实时性和查询延迟 trade off
- 状态：待拆页

## 5. Transformation / Semantic Layer

### dbt

- 位置：transformation / analytics engineering
- 典型价值：SQL transformation、测试、文档、数据模型协作
- 主要问题：指标 owner、semantic layer、运行成本、模型膨胀
- 入口：[[../05-Topics/Semantic Layer 与指标治理|Semantic Layer 与指标治理]]

### Semantic / Metrics Layer

- 位置：指标与业务语义层
- 典型价值：统一指标口径，供 BI、API、AI agent 复用
- 主要问题：治理流程、owner、权限、粒度和消费契约
- 入口：[[../05-Topics/Semantic Layer 与指标治理|Semantic Layer 与指标治理]]

## 6. Orchestration / Governance

### Airflow / Dagster / Prefect

- 位置：orchestration
- 典型价值：调度、依赖、重跑、backfill、任务运维
- 主要问题：资产感知、重跑语义、复杂 DAG、on-call 成本
- 状态：待拆页

### DataHub / OpenMetadata / OpenLineage

- 位置：catalog / metadata / lineage
- 典型价值：数据发现、血缘、owner、影响分析、治理协作
- 主要问题：元数据质量、接入覆盖、治理动作是否真的闭环
- 状态：待拆页

## 最短记忆

- Snowflake / BigQuery：偏 cloud warehouse / analytics platform
- Databricks：偏 lakehouse + Spark + ML / AI platform
- Kafka / Confluent：偏事件日志和 streaming platform
- Flink：偏有状态实时计算
- Spark：偏批处理和历史重算
- Iceberg / Delta / Hudi：偏 lakehouse 表格式
- dbt / semantic layer：偏建模、指标和业务语义复用
- Airflow / Dagster：偏编排和运维
- DataHub / OpenMetadata：偏元数据、catalog 和治理
