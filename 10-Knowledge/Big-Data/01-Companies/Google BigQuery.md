# Google BigQuery

## 一句话定位

BigQuery 是 Google Cloud 的 serverless cloud data warehouse / analytics platform，代表“少运维、强 SQL、云原生分析”的路线。

## 放在全景架构哪一层

- Cloud warehouse
- SQL analytics
- Interactive query
- BI / analytics serving
- ML / AI data integration 的数据底座

## 为什么值得研究

- 它代表 serverless analytics 体验
- 它适合理解云厂商如何把数据仓库、对象存储、治理、BI、ML 和 AI 服务打通
- 它常用于对比 Snowflake、Databricks、Redshift 和 Fabric

## 读它时重点看

- serverless 查询与成本模型
- partition / clustering / query optimization
- 和 Google Cloud 数据、AI、BI 生态的集成
- BigLake / open table / lakehouse 相关能力
- governance、data sharing 和权限模型

## 容易误解

- 不要只把 BigQuery 当 SQL 数据库
- 它更像云原生 analytics platform
- 它的优势和限制都要放在 Google Cloud 生态里看
