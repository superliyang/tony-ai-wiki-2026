# Databricks

## 一句话定位

Databricks 是 lakehouse 路线代表公司，核心是把 Spark、Delta Lake、notebook / workflow、ML / AI、governance 和数据平台体验整合起来。

## 放在全景架构哪一层

- Lakehouse platform
- Spark / compute
- Table format / Delta ecosystem
- Workflow / notebook
- ML / AI data workload
- Catalog / governance

## 为什么值得研究

- 它代表从 Spark 生态走向 lakehouse platform 的路线
- 它强调开放数据湖上的 warehouse-like 管理能力
- 它适合理解 batch、lakehouse、ML、AI 数据供给如何合并成平台

## 读它时重点看

- Lakehouse 架构叙事
- Delta Lake / Unity Catalog
- Spark 与 SQL workload 的结合
- ML / feature / AI 数据链路
- open format 与托管平台之间的取舍
- 和 Snowflake 的定位差异

## 容易误解

- 不要把 Databricks 只当 Spark notebook
- 也不要把 lakehouse 简化成“对象存储 + 文件”
- 真正要看它如何把 compute、catalog、governance、ML 和 AI workflow 组合成平台
