# 游戏实时运营指标链路图

```mermaid
flowchart LR
  A["客户端行为事件"] --> G["Event Gateway / SDK"]
  B["服务端权威事件"] --> G
  C["支付/经济/库存事件"] --> G
  D["崩溃/延迟/区服日志"] --> G
  G --> K["Kafka Topics"]
  K --> F["Flink: 实时指标、会话、异常"]
  F --> O["运营大盘 / 告警 / 活动触发"]
  K --> R["Raw Landing"]
  R --> I["Iceberg Tables"]
  I --> S["Spark: 留存、LTV、漏斗、Cohort"]
  S --> P["产品复盘 / BI / 实验分析"]
  S --> Q["数据质量与口径校验"]
  Q --> G
```

## 读图要点

- 客户端事件适合补体验细节，服务端事件负责关键事实校准
- Kafka 让高吞吐行为、战斗、经济、质量和运营事件解耦复用
- Flink 适合实时观察、窗口聚合、异常检测和活动监控
- Iceberg / Lakehouse 保存版本、活动、实验和行为历史
- Spark 做留存、LTV、漏斗、cohort、活动 ROI 和长期复盘
