# AI 架构师作品集样例索引

> 这些是“可替换为真实经历”的作品集样例，不是虚构战绩。用途是帮助你把项目讲成架构能力证据：业务边界、系统设计、eval、安全治理、成本延迟、上线运营和复盘。

## 三类作品分别证明什么

| 作品类型 | 证明能力 | 面试信号 |
|---|---|---|
| 企业知识库 RAG | 把非结构化知识变成可靠问答系统 | 你理解权限、引用、检索质量、幻觉控制和知识新鲜度 |
| 审批型 Agent | 把 AI 接入真实业务动作 | 你理解 tool use、状态机、高风险动作、人工确认、幂等和审计 |
| 企业 LLMOps 平台 | 把单点 AI 项目升级为组织级平台 | 你理解模型网关、eval gate、prompt registry、观测、成本和治理 |

## 推荐使用方法

1. 先读 [[../07-Templates/AI 架构师作品集模板|AI 架构师作品集模板]]，理解标准结构。
2. 再选一个与你经历最接近的样例。
3. 把“样例背景”替换成自己的真实业务背景。
4. 保留架构表达骨架：`目标 -> 边界 -> 架构 -> 数据 -> 模型工具 -> eval -> 安全 -> 成本 -> 复盘`。
5. 为每个作品补 3 类证据：架构图、评测报告、上线复盘。

## 面试准备建议

- 初级转型：优先准备一个 RAG 作品，证明能把 AI 应用做稳。
- 中高级架构：至少准备一个 Agent 或 LLMOps 作品，证明能处理执行链路和平台治理。
- 架构负责人：三个作品都要能讲，并能说明为什么不是所有场景都应该用 Agent。
