# Knowledge Assets

`10-Knowledge/` is the long-term human-readable knowledge layer.

Content here should be reviewed, structured, and useful for future reading or reuse. AI-generated material should normally start in `00-Inbox-AI/` and enter this layer only after review or explicit instruction.

## Initial Domains

- `AI/`
- `AI-Engineering/`
- `AI-Open-Source/`
- `Advertising/`
- `Security/`
- `International-Payments/`
- `Personal-Knowledge-System/`
- `_Imported/tony2026/`: old vault knowledge assets preserved as source context.

## Domain Shape

A mature domain should eventually have:

```text
专题总览.md
学习路径.md
主题索引.md
地图.md
恢复笔记.md
Concepts/
Systems/
Papers/
Projects/
Playbooks/
```

The structure can grow only when the domain needs it.

## Evolution Rule

Do not try to finish the whole knowledge base in one pass. Promote, connect, and refactor in bounded batches so the vault keeps absorbing new material without losing old context.

## Expert Learning Rule

When a domain becomes important, create or update its domain structure and expert question layer. The minimum useful package is:

- domain master map;
- expert question list;
- beginner traps;
- expert tradeoffs;
- one-page review card.
