## Hermes Intake Rule

Hermes 发现开源项目时，先写到 working vault，Codex 再按 [[90-Agent-System/workflows/knowledge-intake-and-promotion]] 处理。

项目候选不要直接变成散点笔记。它必须进入：

```text
Project -> Category -> Organization -> Pattern -> Watchlist / Map
```

例如 Unsloth 应先成为 `03-Projects/Unsloth.md`，再决定是否更新训练工具链分类、Unsloth AI 组织页、单 GPU 微调内存优化模式和重点 Watchlist。
