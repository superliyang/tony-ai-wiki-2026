# 用 Skill 与 Self-Improving Plugin 维护开源项目知识库

## 分工

- skill：负责知识库本体
- plugin：负责旁路沉淀 learnings、promotion、watchlist 判断偏好

## 当前最稳的组合

- 领域骨架：`/Users/tony/.codex/skills/ai-open-source-research-system`
- 知识库维护：`/Users/tony/.codex/skills/knowledge-vault-research-sync`
- 旁路自改进：`/Users/tony/plugins/self-improving-memory-lab`

## 什么时候该让 plugin 介入

- 对某类 repo 的分类规则开始稳定重复
- 对“什么值得进入 watchlist”的判断开始重复
- 对“什么字段最值钱”的偏好开始重复

## 什么时候不要让 plugin 硬上

- 还在摸索分类边界时
- 还没把项目页模板稳定下来时
- 还没建立最小 eval gate 时
