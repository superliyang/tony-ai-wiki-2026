# GitHub 开源项目 Intake 工作流

用于判断一个开源项目是否值得进入 AI-Open-Source 控制塔。

## Steps

1. 读取 Hermes / human source。
2. 判断是否值得入库：工程价值、学习价值、迁移价值、稳定信号。
3. 判断 stack layer：底座 / 壳层 / 平台 / 子系统。
4. 选择目标：Project / Organization / Category / Pattern / Case Study / Watchlist。
5. 创建或更新项目卡。
6. 更新项目索引、分类索引、组织索引、模式索引和 Watchlist。
7. 写 promotion log。
