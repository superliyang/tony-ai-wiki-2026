# Compound Engineering Plugin

## 它解决什么问题

`Compound Engineering Plugin` 试图把工程工作变成可复利的循环：strategy、ideate、brainstorm、plan、work、debug、review、compound learning。

## 为什么现在值得关注

它非常符合我们前面总结的 AI Coding 专家路径：不是让 agent 一次性写更多代码，而是让每次工程工作沉淀成更好的计划、评审和知识。

## 它在 stack 的哪一层

- 属于 `engineering workflow plugin`
- 是 planning / review / learning loop 的产品化样本

## 核心组件与架构

- `/ce-strategy`
- `/ce-brainstorm`
- `/ce-plan`
- `/ce-work`
- `/ce-debug`
- `/ce-code-review`
- `/ce-compound`

## 最适合它的场景

- 需求澄清
- 实现计划
- 多 agent review
- 调试闭环
- 把经验写回可复用知识

## 和相邻项目怎么区分

- 和 [[SuperClaude Framework]]：Compound 更强调工程复利闭环；SuperClaude 更强调完整开发平台
- 和 [[CCPM]]：Compound 更偏工作方法；CCPM 更偏任务管理和并行执行

## 风险与边界

- 流程感强，不适合所有轻量小修
- 应避免把 ceremony 做成负担，优先用于复杂或高风险任务
