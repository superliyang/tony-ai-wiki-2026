# 广告投放来源与校验

本页记录广告投放知识包里需要持续校验的来源、数字和边界条件。

## Source State

当前内容来自 Hermes 的 Exa 搜索和综合整理，Codex 已做第一轮结构修正，但尚未逐条完成 URL 级事实核验。

因此：

- 结构性框架可作为学习入口；
- 具体比例、benchmark、行业统计不能直接作为业务决策依据；
- 后续深挖时应把每个关键数字绑定到具体 URL、发布时间和适用场景。

## 已知需要校验的关键断言

| Claim | Current Use | Verification Need |
|---|---|---|
| 全球 IVT 率约 20.6% | 说明广告欺诈规模 | 需要确认来源、统计口径、年份、是否全球/程序化/展示广告 |
| CTV 欺诈率约 12.4% | 标记 CTV 高风险 | 需要确认是否为某报告样本，而非全行业固定值 |
| 平台 ROAS 通常高估 30-100% | 解释增量测试必要性 | 需要按渠道、拉新/重定向、行业拆分 |
| Meta CAPI 可恢复 20-35% 信号 | 说明服务端归因收益 | 需要确认来自 Meta、代理商案例还是第三方报告 |
| Google Enhanced Conversions 可恢复 5-15% 信号 | 说明信号恢复收益 | 需要确认口径和适用条件 |
| D30 ROAS 40%+ 及格、60%+ 优秀 | 移动游戏复习卡 | 需要按品类、变现模型、地区和现金约束校准 |
| Payback <30 天可放量 | 现金流放量规则 | 需要按资金成本、毛利、留存曲线和投放规模调整 |
| Geo Holdout 4-6 周、$50k+/月渠道起始 | 增量测试门槛 | 需要确认适用样本量和统计显著性要求 |

## Recommended Primary Sources

优先核验这些来源类型：

- 平台官方文档：Google Ads、Meta、Apple、IAB Tech Lab、MRC。
- 行业报告：AppsFlyer、Adjust、Singular、Liftoff、Sensor Tower、eMarketer。
- 公开公司材料：The Trade Desk 年报、投资者信和产品说明。
- 测量/反作弊机构：DoubleVerify、IAS、HUMAN、Fraudlogix。
- 实操案例：移动游戏 UA agency、增长团队复盘、MMP 方法论。

## Source Quality Levels

| Level | Meaning | Can Promote? |
|---|---|---|
| L1 | 官方规范、平台文档、监管/标准组织 | 可以作为事实基础 |
| L2 | 头部行业报告、公开公司材料 | 可以作为趋势和 benchmark |
| L3 | 代理商/工具厂商博客 | 可作为案例，但需标注商业立场 |
| L4 | 二级媒体、个人博客、汇总文章 | 只能作为线索 |

## Review Rule

当某条内容要进入业务决策、playbook 或长期记忆时，需要补齐：

- URL；
- publish date；
- source type；
- claim scope；
- whether it is benchmark, case study, opinion, or official rule。
