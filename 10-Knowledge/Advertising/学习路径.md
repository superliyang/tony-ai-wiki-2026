# 广告投放学习路径

这条路径把广告投放当作一个领域系统来学：先建立结构，再理解指标和技术，再进入专家问题和实战权衡。

## 0. 先定范围

当前第一版重点是：

- 程序化广告与 RTB；
- DSP / SSP / Ad Exchange 架构；
- 移动游戏 UA；
- 归因、信号损失和增量测试；
- 广告欺诈、品牌安全和隐私合规；
- UA、产品、财务之间的 KPI 协同。

暂未系统覆盖：

- 搜索广告；
- 社媒内容投放；
- 品牌营销；
- 零售媒体；
- 达人和创作者投放；
- 线下广告和传统媒体。

## 1. 新手：建立全局结构

目标：知道广告投放领域在优化什么、有哪些角色、钱和信号如何流动。

阅读顺序：

1. [[广告投放领域全貌图]]
2. [[10-Knowledge/Advertising/06-Maps/广告投放总览图]]
3. [[10-Knowledge/Advertising/05-Topics/术语表]]

检查自己是否理解：

- 广告主、DSP、ADX、SSP、媒体分别要什么？
- CPM、CPC、CPA、ROAS、LTV、Payback Period 之间是什么关系？
- 为什么“低 CPI”不等于“好投放”？

## 2. 进阶：理解关键问题

目标：能看懂投放决策背后的指标冲突和系统约束。

阅读顺序：

1. [[广告投放专家问题解答]]
2. [[10-Knowledge/Advertising/05-Topics/指标体系与口径治理]]
3. [[10-Knowledge/Advertising/05-Topics/归因、SKAN 与信号损失]]
4. [[来源与校验]]

重点问题：

- 平台 ROAS 和真实增量 ROAS 为什么不同？
- D7、D30、D90 分别适合什么决策？
- SKAN / AdAttributionKit / MMP / CAPI 各解决什么问题？
- Open Exchange、PMP、Programmatic Guaranteed 的适用边界是什么？

## 3. 专家：进入实战系统

目标：能围绕一个具体业务场景提出专家级问题，并设计验证路径。

推荐深钻方向：

| Direction | Output |
|---|---|
| SKAN 信号工程 | conversion value 设计、coarse/fine 对齐、历史校准方法 |
| 增量测试 | geo holdout、PSA holdout、预算阈值、显著性和机会成本 |
| DSP 架构 | OpenRTB、竞价延迟、pCTR/pCVR、pacing、预算和缓存 |
| 服务端归因 | Meta CAPI、Google Enhanced Conversions、sGTM、隐私边界 |
| 广告欺诈治理 | IVT、sellers.json、ads.txt、pre-bid filtering、SPO |
| 组织协同 | UA、产品、财务共享 KPI 和奖金机制 |

## 4. 复习问题

- 如果某渠道 D7 很好但 D30 变差，你先怀疑渠道质量、归因、产品留存还是队列结构？
- 如果平台 ROAS 是 3.5x，增量测试是 1.8x，你如何调整预算目标？
- 如果 iOS 只能拿到 coarse conversion value，你如何设计仍可行动的信号？
- 如果 IVT 从 15% 上升到 30%，你如何排查并阻止继续损失？
