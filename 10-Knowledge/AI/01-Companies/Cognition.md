# Cognition

## 简介

Cognition 按官方首页的说法，是一个 `Agent Lab`。它最重要的代表系统是 `Devin`，并且官方把 Devin 定位成 `AI software engineer`。

## 你可以把这家公司当成什么来理解

可以把 Cognition 当成“更强自治软件工程 agent 公司”来理解：

- 它不是只做一个代码助手
- 它更强调把完整工程任务交给 agent 持续推进
- 它代表了“软件工程委托系统”这条更激进的产品路线

## 为什么重要

- 它把 coding agent 的叙事从“协作式助手”继续推向“自治式工程 worker”
- 它很适合帮助我们理解 agent 产品在自治边界上的下一步分化
- 它也是研究 `sessions / knowledge / skills / scheduling` 这类系统能力的好案例

## 学习这家公司时，重点看什么

- autonomous software engineering
- long-running delegated work
- knowledge and skills
- scheduled sessions

## 关键问题

- 软件工程 agent 到底能自治到什么程度
- “协作式 coding agent”和“委托式 software engineer”在产品形态上差在哪里
- Cognition 的真正壁垒是在模型、产品叙事，还是系统执行能力
