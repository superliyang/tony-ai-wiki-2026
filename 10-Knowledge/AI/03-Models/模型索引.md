## 2026-06 Refresh Lens

近期模型索引需要增加一个工程视角：模型不只按“聊天能力”看，还要按 agent workflow 中的角色看。

| Lens | 问题 | 近期信号 |
|---|---|---|
| Agentic Coding | 是否适合 repo 级代码修改、测试、review | OpenAI GPT-5.5、Anthropic Claude Opus 4.8 都把 coding / agentic tasks 放在核心发布叙事里 |
| Long-Horizon Work | 是否能承担多步骤、多工具、长上下文任务 | Claude Opus 4.8 的 dynamic workflows、OpenAI GPT-5.5 的 multi-part task / tool use 都说明长任务成为模型比较主线 |
| Model Role | 更适合作为 planner、executor、reviewer、judge，还是 cheap worker | effort control、fast mode、Pro/standard 分层说明模型选择要接入成本和任务路由 |
| Computer / Browser Use | 是否能操作真实软件、网页、文件和 UI | OSWorld、Online-Mind2Web、Search agent 方向越来越像真实工作入口 |
| Enterprise Connector Governance | 模型是否能在企业 connector、权限、审计和数据边界里运行 | Claude release notes 和 Google Search/agent updates 都指向连接器治理 |
| Open Toolchain | 是否有可用的本地部署、fine-tuning 或 coding agent 生态 | Qwen Code / Qwen3-Coder、vLLM、Unsloth 需要和开源工程栈一起看 |

详见 [[10-Knowledge/AI/07-Maps/2026-06 AI 近期变化雷达]]。

## 近三个月模型变化判断

| Model / Family | Date | 变化 | 对 Tony 知识图谱的意义 |
|---|---|---|---|
| GPT-5.5 / GPT-5.5 Pro | 2026-04 | OpenAI 将发布重点放在 agentic coding、knowledge work、scientific research、computer work 和 cybersecurity safeguards | 模型索引要显式记录“可长期执行任务”的能力，而不是只按 chat benchmark |

| Gemini / Google AI product surface | 2026-05 | Google May AI updates 强调 agentic era、Gemini app、Search、developer / Workspace 等产品化入口 | Google 生态要按“模型 + 产品 surface + workflow”看 |
| Open model coding stack | 2026-Q2 | Qwen Code / Qwen3-Coder、vLLM、Unsloth 继续影响本地训练、serving、coding agent | Open-weight model 不只看模型卡，还要看工具链闭环 |

## 模型卡最小字段

后续新增模型卡时，至少写这些字段：

| Field | Why |
|---|---|
| `as_of` | 防止模型能力判断过期 |
| Release / source date | 区分事实发布日期和笔记更新日期 |
| Primary strengths | coding、reasoning、vision、research、tool use、local serving 等 |
| Agent role | planner / executor / reviewer / judge / cheap worker |
| Workflow fit | ChatGPT/Codex/Claude Code/Cursor/Hermes/本地 serving |
| Safety / governance | cyber safeguards、connector 权限、enterprise controls |
| Cost / speed tier | 决定是否适合长期 agent loop |
| Source links | 官方发布、system card、model card、API docs |

## Promotion Note

模型事实变化快。正式笔记需要保留发布日期、来源和适用时间，不把短期榜单写成长期判断。
