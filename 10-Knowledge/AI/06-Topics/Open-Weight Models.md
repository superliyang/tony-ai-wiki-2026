# Open-Weight Models

## 这个主题是什么

`Open-Weight Models` 关注模型权重开放所带来的研究、开发、部署和生态扩散问题。

## 为什么重要

- 它直接影响开发者生态和二次创新速度
- 它改变了模型能力的传播方式
- 它让竞争不再只发生在少数闭源 API 公司之间
- 最近半年，open-weight 没有退场，反而在全球竞争里进一步明确了自己的位置

## 过去半年这条线为什么仍然关键

最近半年，一个常见误解是“闭源 frontier 太强，open-weight 不重要了”。但实际官方动态说明并不是这样：

- [[DeepSeek]] 继续把 reasoning-first 和 agent-ready 模型做成开放路线
- [[Alibaba Cloud]] 继续通过 `Qwen` 与 `Qwen Code` 强化开发者生态
- [[Zhipu AI]] 通过 `GLM-4.7-Flash`、`GLM-4.6V` 强调开源与原生工具调用
- [[Mistral AI]] 通过 `Mistral 3` 把 open-weight、多模态、多语言重新拉到前沿讨论中心
- [[Google DeepMind]] 通过 [[../03-Models/Gemma 4|Gemma 4]] 把 `open model + agentic workflow + edge / device` 再往前推
- [[Meta AI]] 的开放路线仍然是整个全球 open-weight 生态的重要背景板

## 你先要抓住什么

- open-weight 不等于 open-source，但比纯 API 更开放
- 开放权重会显著改变部署方式、定制空间和社区创新速度
- 它让模型竞争从“谁能调用”转向“谁能修改、部署、蒸馏、微调”
- 在最近半年里，它还越来越与 `sovereign AI`、`enterprise deployability` 和 `coding ecosystem` 绑在一起

## 关键问题

- 开放权重真正给开发者和创业公司带来了哪些自由
- 为什么 open-weight 路线常常和推理成本、部署可控性、生态传播绑定
- 它和闭源最强模型之间，长期会是什么关系
- 为什么 open-weight 路线在欧洲、中国和开发者生态里仍然很有生命力
