## 建议学习路径

1. 如果你在梳理整个 AI 体系，先读 [[../AI 总控制塔|AI 总控制塔]]，再读 [[07-Maps/AI 五大专题审察与组织|AI 五大专题审察与组织]]，再配合 `Canvas` 和 `Base` 看结构
2. 先在 [[../AI-Foundations/专题总览|AI-Foundations]] 建立历史、范式和基础概念
3. 再到 [[06-Topics/AI 领域知识点总纲：点、线、面与层|AI 领域知识点总纲：点、线、面与层]] 和 [[07-Maps/AI 知识点点线面图|AI 知识点点线面图]] 建立点线面骨架
4. 然后先顺着五条主干 drill-down 往下读：[[06-Topics/模型形成线：Pretraining、Transformer、Foundation Models、Reasoning 与 Multimodal|模型形成线]]、[[06-Topics/能力升级线：Prompt、Context、RAG、Tool Use、Agent、Memory 与 Multi-Agent|能力升级线]]、[[06-Topics/运行时工程线：Inference、Serving、Evaluation、Release Gate 与 Security|运行时工程线]]、[[06-Topics/安全治理线：AI Safety、AI Security、Prompt Injection、Approval、Audit 与 Release Gate|安全治理线]]、[[06-Topics/产品落地线：Model、Workflow、Case Study、Rollout、ROI 与 Governance|产品落地线]]
5. 再进入 [[06-Topics/AI 五条主干专家工作台|AI 五条主干专家工作台]]，按 `关键问题 -> 核心指标 -> 易混边界 -> 面试表达要点` 把五条主干从知识线升级成判断线
6. 再回 [[06-Topics/AI 主题索引|AI 主题索引]] 理解现代 AI 主线
7. 再阅读 [[03-Models/模型索引|模型索引]]，把技术概念映射到代表模型
8. 如果你想先把 `2026-04-29` 之后的新变化接上，插入 [[11-Recent-Supplements/2026-04-29 至 2026-05-15 AI 拓扑补线：Governed Agents、Device Intelligence 与 Evaluation Institutionalization|2026-04-29 至 2026-05-15 AI 拓扑补线]]
9. 如果你想先把 `2026-04-07` 之后的新变化接上，插入 [[11-Recent-Supplements/2026-04-07 至 2026-04-29 AI 拓扑补线：Frontier Work Models、Sandbox Agents 与 Enterprise Model Lifecycle|2026-04-07 至 2026-04-29 AI 拓扑补线]]
10. 如果你想先把截至 `2026-04-07` 的模型格局刷到最新，插入 [[11-Recent-Supplements/截至 2026-04-07 的 2026 新模型刷新|截至 2026-04-07 的 2026 新模型刷新]]
11. 如果你想先知道什么最值得重投入，把这条优先级判断线插进来：[[11-Recent-Supplements/最近半年最值得重投入学习的 AI 主线（截至 2026-04-07）|最近半年最值得重投入学习的 AI 主线（截至 2026-04-07）]]
12. 如果你想集中补 recent route，而不是在根目录四处找，先进入 [[11-Recent-Supplements/补线索引|补线索引]]
13. 如果你想补上 `2025-2026` 里最容易漏掉的研究型 agent、memory 和 runtime 变化，插入 [[11-Recent-Supplements/2025-2026 AI 新路线补线：Deep Research、Memory 与 Agent Runtime|2025-2026 AI 新路线补线：Deep Research、Memory 与 Agent Runtime]]
14. 如果你想补上 `2025-2026` 的模型治理与监管变化，插入 [[11-Recent-Supplements/2025-2026 AI 治理补线：Model Spec、Preparedness、Transparency 与 AI Act|2025-2026 AI 治理补线：Model Spec、Preparedness、Transparency 与 AI Act]]
15. 如果你想把最近五年的关键论文直接接回模型、系统和产品路线，先读 [[../近五年关键 AI 论文与路线映射（2021-2025）|近五年关键 AI 论文与路线映射（2021-2025）]]，再回 [[04-Papers/论文索引|论文索引]]
16. 再阅读 [[09-Systems/系统索引|系统索引]]，理解产品、平台和 runtime 这一层
17. 如果你想建立更像专家的回顾方式，插入一条 `人物 -> 公司 -> 系统 -> 案例` 的证据线：先看 [[../哪些 AI 人物最值得长期跟踪|哪些 AI 人物最值得长期跟踪]]、[[../怎么读 AI 组织：能力画像、生态位与 Adoptability|怎么读 AI 组织：能力画像、生态位与 Adoptability]]，再回 [[02-People/人物索引|人物索引]]、[[01-Companies/公司索引|公司索引]] 和 [[../AI-Applications/04-Case-Studies/案例索引|案例索引]]
18. 如果你想系统理解 Agent，先顺着 `Agent -> Deep Research -> Prompt/Context -> Tool Use -> MCP / Browser -> Memory / Planning / Multi-Agent -> Agent 平台` 读
19. 如果你想成为 AI Coding 专家，先读 [[06-Topics/AI Coding 专家能力体系|AI Coding 专家能力体系]]，再看 [[07-Maps/AI Coding 专家能力图谱|AI Coding 专家能力图谱]]、[[../AI-Engineering/08-Maps/AI Coding 团队落地路线图|AI Coding 团队落地路线图]]、[[../AI-Engineering/08-Maps/AI Coding 安全治理决策图|AI Coding 安全治理决策图]]
20. 如果你想理解 AI infra / serving，顺着 `AI 基础设施与 GPU Cloud -> Inference Serving -> AI Infra 与推理服务生态图` 读
21. 如果你想理解实验、评测、registry 和反馈闭环，顺着 `MLOps 与 LLMOps -> MLOps 与 LLMOps 生态图 -> MLflow / W&B / Langfuse / Phoenix / Promptfoo` 读
22. 如果你想走一条 `MacBook Pro Max M4` 本地专家路径，先读 `Apple Silicon 本地 AI 开发 -> MLX / Ollama / llama.cpp -> Mac AI Expert Path -> MacBook Pro Max M4 学 AI 与大模型：完整路线`
23. 如果你想理解 AI security 与模型治理一起怎么约束上线，顺着 `AI Safety -> AI Security -> AI Safety 与 AI Security -> 模型行为治理 -> Prompt Injection 与 Jailbreaks -> Azure AI Content Safety / NeMo Guardrails -> AI Security Threat Map -> AI Security 控制点图` 读
24. 然后进入 [[../AI-Engineering/专题总览|AI-Engineering]] 看工程实现与治理
25. 如果你在准备 AI 技术专家/架构师面试，先读 [[06-Topics/AI 面试题专项：技术专家与架构师|AI 面试题专项：技术专家与架构师]]，再进入 [[10-AI-Interview-Kit/专题总览|AI Interview Kit 专题总览]]；建议先读 `完整面试题蓝图 -> 资深候选人表达总纲 -> 开场与项目深挖题库 -> 场景题包索引`，再刷技术题库
26. 最后进入 [[../AI-Applications/专题总览|AI-Applications]] 看业务 workflow 与组织落地
