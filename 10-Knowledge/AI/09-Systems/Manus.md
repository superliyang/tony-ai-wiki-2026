# Manus

## 简介

`Manus` 是一个更偏“自主执行型”的 AI agent 产品。按 Manus 官方文档的说法，它是一个 `autonomous general AI agent`，目标不是只回答问题，而是替你完成任务并交付结果。

## 为什么重要

- 它代表了一类“结果导向型 agent 产品”
- 它把 agent 的卖点从“会不会说”进一步推进到“能不能自己把事情做完”
- 它很适合拿来和 `ChatGPT Agent`、`Claude Code`、`OpenClaw` 做定位对比

## 官方定位里最值得记住的点

- 它强调自己不是传统 chatbot
- 它强调自己运行在一个带网络、文件系统、软件安装能力的 sandbox 环境
- 它强调自己能围绕长期任务持续工作，并交付完整结果

这说明 Manus 的产品重点是：

- autonomy
- task completion
- result delivery
- virtual computer style execution environment

## 你可以把它当成什么来理解

- 如果 `ChatGPT` 更像通用 assistant
- `Claude Code` 更像 coding agent
- `OpenClaw` 更像 personal agent runtime

那么 `Manus` 更像：

- `autonomous task operator`
- `result-oriented general agent`

## 学习时重点看什么

- 它的“自主执行”到底依赖哪些系统能力
- 它和 computer-use / browser-use / sandbox execution 的关系
- 它和长期在线 runtime 的差别在哪里
- 它为什么强调“交付结果”，而不只是交付过程

## 它和 OpenClaw 的关键差异

- `Manus` 更像一个任务代理产品
- `OpenClaw` 更像一个长期在线、跨渠道、自托管的 assistant runtime

也就是说：

- Manus 更强调“替你完成一件事”
- OpenClaw 更强调“持续存在、持续协作、持续维护上下文”

## 它和 ChatGPT Agent 的关键差异

- `ChatGPT Agent` 还是强依托于 ChatGPT 这个统一产品入口
- `Manus` 的叙事更激进，更强调“独立完成任务”和“交付完整工作结果”

## 它和 Claude Code 的关键差异

- `Claude Code` 主要扎在开发者工作流里
- `Manus` 更像面向更广泛任务类型的执行代理
