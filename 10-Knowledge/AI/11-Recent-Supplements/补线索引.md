# 补线索引

> 这里不是新的“长期主干层”，而是 AI 知识库里专门承接 `近两年新路线 / 新模型刷新 / 治理补线` 的过渡层。

## 这个文件夹解决什么问题

过去补线文档平铺在 `01-Areas/` 根部，短期好找，长期会有两个问题：

- recent route、model refresh、governance supplement 会越来越多，根目录会失去可读性
- 补线页既不像稳定 `Topics / Models / Systems / Papers`，也不该永远散落在总入口旁边

所以现在把这类文档统一收进 `11-Recent-Supplements`。

## 后续补线策略

以后新补线文档，优先放这里，当它满足下面任一条件时：

- 它主要在解释 `最近 1-2 年` 的新路线变化
- 它是 `阶段性模型刷新` 或 `季度/半年度判断`
- 它是把多个新发布串成一条 `recent route`
- 它还没有稳定到值得直接长成独立 `主题 / 系统 / 模型 / 论文` 主干

不放这里的内容：

- 已经稳定成长期概念的主题
- 具体公司、模型、系统、论文卡片
- 面向长期复习的主干页和工作台页

## 补线文档怎么沉淀回主干

一篇补线文档的理想生命周期是：

1. 先在这里作为 `recent route` 或 `refresh note`
2. 把结论下沉到 `06-Topics / 03-Models / 09-Systems / 04-Papers`
3. 等主干层长厚以后，这里保留为“路线总览页”，而不是唯一知识承载页

也就是说：

- 这里负责 `先串起来`
- 主干层负责 `长期承重`
