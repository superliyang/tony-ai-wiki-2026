# 2026-06 AI 近期变化雷达

这页记录第一批内容刷新中，AI broad domain 里会影响学习路径的近期变化。它不是新闻流，而是判断哪些变化应该进入模型、公司、论文、主题和工程分流。

## 当前判断

| 变化 | 影响 | 主库动作 |
|---|---|---|
| Coding / agentic coding model 继续成为模型竞争主线 | broad AI 不应只按 chat model 看模型谱系，coding agent capability 需要成为模型索引的一条显式视角 | 更新 [[10-Knowledge/AI/03-Models/模型索引]] |
| Claude / Gemini / GPT 系列都在强化复杂推理、长任务、多模态、agentic coding | 学习路径应从“模型能力对比”转向“模型在工作流中的角色选择” | 连接 [[10-Knowledge/AI-Engineering/学习路径]] |
| Qwen Code / Qwen3-Coder 代表开源 coding agent 与模型协同路线 | China AI 与 open-source coding agent 不应只看模型，还要看工具链 | 连接 [[10-Knowledge/AI-Open-Source/学习路径]] |
| 模型发布和能力事实变化快 | 模型页必须带 `as_of` 和来源日期，不把短期榜单写成稳定结论 | 更新模型卡模板 |

## 模型索引需要新增的阅读维度

| Dimension | 为什么重要 |
|---|---|
| Agentic Coding | 衡量模型是否能长时间读 repo、改代码、跑测试、处理终端任务 |
| Long-Horizon Work | 不只是单轮问答，而是多步骤、多文件、多工具任务 |
| Multimodal Reasoning | UI、图表、视频、代码库、文档混合输入成为真实工作入口 |
| Cost / Speed Tiering | Opus / Sonnet / Haiku、Pro / Flash 一类分层影响 agent 编排成本 |
| Open Model Toolchain | Qwen / DeepSeek / Llama / Gemma 等需要和本地工具链、serving、fine-tuning 一起看 |

## 对学习路径的影响

[[10-Knowledge/AI/学习路径]] 的 Track A 不应只读公司和模型，还应该追问：

1. 这个模型适合 chat、research、coding、tool use、vision、还是本地部署？
2. 它在 agent workflow 里是 planner、executor、reviewer、judge，还是 cheap worker？
3. 它的能力变化是否会改变 [[10-Knowledge/AI-Engineering/学习路径]] 的工程选型？
4. 它是否有对应的开源工具链或生态样本进入 [[10-Knowledge/AI-Open-Source/学习路径]]？
