# 留存、LiveOps 与用户获取

## 这个品类的增长不只是买量

`skills gaming` 很容易让人以为只要：

- 买量
- 拉首充
- 让玩家打比赛

就够了。

但从 `Papaya`、`AviaGames`、`WorldWinner` 这些官方动作看，真正稳定增长依赖的是一整套运营节奏：

- 赛事活动
- 目标激励
- 奖励层级
- 节点型 championship
- 内容矩阵导流
- 玩家研究与持续迭代

## 从官方动作里能看到什么

### `Papaya`

`Papaya` 的 `World Solitaire Championship` 说明了一件事：

- skills game 也能被做成大型赛事和品牌事件
- 赛事本身可以变成 retention 和品牌资产的一部分

### `AviaGames`

`AviaGames` 在 `GDC 2026` 提到：

- payments / payouts 会直接影响 trust 与 retention
- brand + performance partnership 是增长的重要抓手

它还专门和 MBA 项目合作研究 `Solitaire Clash`，说明：

- 头部公司已经把“玩家洞察”当成持续经营动作，而不是一次性市场调研

### `WorldWinner`

从 `Goal Power`、`Progressive Payouts`、`Lifetime Rewards`、`Premier Club` 这些动作能看出来：

- retention 不是只靠比赛本身
- 还要靠 meta reward、分层活动、目标系统和 loyalty 机制

## 这个品类里最值得学的 retention 结构

### 1. 回来就能玩的短 session

- 不需要重新学习很多系统
- 一上来就能进入比赛或挑战

### 2. 有“今天为什么回来”的理由

- daily qualifiers
- event windows
- progressive payouts
- goals / missions

### 3. 有“我在变强”的感受

- skill progression
- better matchmaking outcomes
- better tournament performance

### 4. 有“我被这个生态承接了”的感觉

- portfolio cross-promo
- rewards system
- club / VIP / reward tiers

## 对 hackathon 的现实建议

你不需要在 demo 里做完完整 LiveOps 系统，但你至少要能讲清：

- Day 1 回流理由是什么
- Week 1 活动钩子是什么
- 如果这个游戏做成 portfolio，它会和什么类型的游戏互相导流

## 一个非常实用的判断

如果一款 skills game 只能解释“第一局为什么玩”，却解释不了“为什么回来”，那它只完成了一半。
