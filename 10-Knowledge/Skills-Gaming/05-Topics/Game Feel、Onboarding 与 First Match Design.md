# Game Feel、Onboarding 与 First Match Design

## 这页为什么重要

你现在这套知识库里，系统、商业、风控、增长、AI 工具都已经很强了。

剩下最容易让人“懂很多，但做不出好游戏”的缺口，就是：

- `game feel`
- onboarding
- first match 体验

真正让玩家愿意再来一局的，往往不是你后台多漂亮，而是：

- 第一局会不会迷路
- 输了是不是也愿意再来
- 结果界面是不是让人清楚、服气、想重开

## Game Feel 不是玄学，它至少包含 5 件事

### 1. 输入反馈

- 点一下有没有立即回应
- 成功、失败、命中、组合有没有明显反馈
- 动画、音效、粒子是不是帮助了判断，而不是只是装饰

### 2. 结果解释

在 `skills gaming` 里尤其关键：

- 为什么得这个分
- 为什么输了
- 时间、精度、错误、奖励分别贡献了什么

### 3. 节奏控制

- 首局是不是太慢
- 新手是不是太快失败
- 结果页是不是拖太久
- rematch CTA 是不是足够直接

### 4. 情绪曲线

一个好的首局体验，通常是：

- 很快理解
- 很快上手
- 很快感到“我还能更好”

而不是：

- 很快被教育自己不行
- 很快觉得规则麻烦

### 5. 信任感

在这个赛道里，`feel` 还包括：

- 比赛是不是可信
- 分数是不是可信
- 对手和结算是不是可信

## Onboarding 的最小原则

### 不要教完整系统，只教赢下第一局需要的最小动作

对于 hackathon V1，更好的 onboarding 是：

- 3 步以内
- 可跳过
- 直接服务于第一场对局
- 每一步都立刻在真实玩法里验证

### 先让玩家动手，再解释完整规则

更好的顺序通常是：

1. 玩家立刻做一个动作
2. 系统给清晰正反馈
3. 再补一句规则解释
4. 快速进入首局

## First Match 设计的关键问题

- 玩家 10 秒内能理解目标吗
- 玩家 30 秒内能拿到第一波成就感吗
- 玩家 90 秒内能看到自己为什么输赢吗
- 首局结束后，CTA 是不是明确指向 `再来一局`

## 对 Word Sprint Duel 的直接建议

- 教学只保留最小规则：看词、拼出、计时、错误会扣节奏
- 结果页必须强调：
  - 正确数
  - 速度
  - 错误
  - 与对手差距
- 不要把首局做成考试
- 要把首局做成“我已经懂了，而且我能再快一点”

## 一句判断

如果你们最终只修一个非系统性问题，优先修：

- onboarding clarity
- result explainability
- rematch desire

因为这三件事最直接决定 demo 会不会被认为“真的像个游戏”。
