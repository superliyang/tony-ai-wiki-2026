# 竞技类型与玩法分类

> 竞技不是只有实时 PVP。休闲竞技的设计空间很大，关键是选择适合目标用户、团队能力和商业约束的竞技结构。

## 按竞争对象分类

| 类型 | 说明 | 适合场景 |
|---|---|---|
| Self-challenge | 和自己的历史成绩比 | 轻休闲、新手、练习模式 |
| Leaderboard | 和排行榜玩家比 | 异步、活动、长期目标 |
| Friend challenge | 和好友比 | 社交传播、轻竞技 |
| Async match | 同题/同条件异步比较 | skills gaming、短局竞技 |
| Realtime PVP | 同时在线对战 | 强竞技、强互动 |
| Tournament | 多人赛、淘汰赛、积分赛 | 活动、奖励、付费/报名 |
| Team competition | 队伍协作或阵营 | 社交和中长期运营 |

## 按技能类型分类

| 技能 | 玩法例子 | 设计重点 |
|---|---|---|
| 反应 | 反应点击、躲避、抢时机 | 延迟、公平、反馈 |
| 精准 | 射击、投掷、路径控制 | 手感、命中反馈、误差空间 |
| 规划 | 益智、路线、排序、策略 | 深度、学习曲线、局内信息 |
| 记忆 | 翻牌、词汇、图案 | 内容公平、题库、重复性 |
| 速度 | 限时消除、打字、找物 | 节奏、压力、设备差异 |
| 风险决策 | 押注、继续/停止、资源取舍 | 风险解释、随机边界 |
| 组合技巧 | 多技能混合 | 避免规则过重 |

## 按对局结构分类

### 1. 同题异步比较

双方玩同一套题、同一张图或同一关卡，然后比较分数。

优点：

- 易控公平。
- 不要求同时在线。
- 适合短局和技能验证。

风险：

- 内容重复后可背板。
- 题库和 seed 管理重要。
- 需要防作弊和结果验证。

### 2. 实时轻对抗

玩家同时进入，互相影响但规则较轻。

优点：

- 情绪强。
- 直播和传播效果好。
- 胜负感直接。

风险：

- 网络、匹配、延迟压力大。
- 新手挫败更强。
- 制作和运维成本高。

### 3. 排行榜冲榜

玩家不断刷新个人成绩，进入榜单。

优点：

- 易做活动。
- 适合单人玩法。
- 开发成本相对低。

风险：

- 作弊动机强。
- 榜单被高手占据后普通玩家无感。
- 需要分层榜单和短周期活动。

### 4. 锦标赛

限时或报名制比赛。

优点：

- 目标强。
- 活动感强。
- 适合奖励和 monetization。

风险：

- 公平、匹配、奖励、争议处理复杂。
- 如果涉及真钱或奖品，合规压力更高。

## 按商业和风险分类

| 类型 | 商业能力 | 风险 |
|---|---|---|
| 纯休闲竞技 | 广告、内购、活动 | 低到中 |
| 社交挑战 | 邀请、分享、轻付费 | 中 |
| 奖励竞技 | 活动奖励、虚拟奖品 | 中到高 |
| Real-money skill game | entry fee、prize、提现 | 高，需合规和风控 |

## 选择竞技类型的决策树

### 如果团队小、周期短

优先：

- self-challenge
- leaderboard
- async match

谨慎：

- realtime PVP
- 强经济系统
- 复杂锦标赛

### 如果要做 skills gaming

优先：

- 同题异步比较。
- 明确技能表达。
- 可验证分数。
- 可解释结果。

谨慎：

- 随机掉落决定胜负。
- 强 P2W 数值成长。
- 难以审计的实时交互。

### 如果目标是社交传播

优先：

- 好友挑战。
- 排行榜截图。
- 差一点赢的结果页。
- 短视频能看懂的操作。

## 分类不是目的，取舍才是目的

一个游戏可以混合多种竞技类型，但早期必须明确主竞技结构：

- 如果主结构是异步比较，就先把同条件、公平计分、结果解释做好。
- 如果主结构是实时对抗，就先把网络、匹配、延迟和新手保护做好。
- 如果主结构是排行榜，就先把分层榜单、反作弊和活动周期做好。
- 如果主结构是锦标赛，就先把报名、赛制、奖励、争议处理做好。
