# 欧美 Skills Gaming 市场与竞品格局

## 先抓住一个核心判断

这个市场不是单纯的“休闲手游 + 现金奖励”，而是三件事叠在一起：

- `casual game loop`
- `skill-based competition`
- `wallet / payout / fairness / risk operations`

也就是说，它既是游戏生意，也是比赛生意，还是一层重运营和风控的基础设施生意。

## 当前看到的 4 类典型玩家

### 1. `Skillz`：更像平台方

从官网和 developer accelerator 的表达看，`Skillz` 的核心定位是：

- 为开发者提供 skill competition 平台
- 帮开发者把游戏接进 competition / monetization 体系
- 用 developer support 和 accelerator 拉新内容进入平台

这意味着它的核心竞争力更偏：

- 开发者生态
- 对战、公平性和 monetization infrastructure
- 平台分成模型

### 2. `Papaya`：更像第一方内容组合 + 运营平台

`Papaya` 官网强调的是：

- own games portfolio
- skill-based matchmaking
- 高频 tournaments
- 节庆活动和社交化玩法

这意味着它更像：

- 自营内容矩阵
- 高强度运营和活动设计
- 依赖 portfolio 扩品类和扩时长

### 3. `AviaGames`：更像大 portfolio 的内容发行商

`AviaGames` 官网展示了很多 title，也突出 `Pocket7Games` 这种聚合入口。

这意味着它的优势更可能在：

- 更强内容覆盖
- 更强分发和交叉导流
- 用多品类游戏承接不同 skill surface

### 4. `WorldWinner / FanDuel Faceoff`：老牌 cross-platform 路线

`WorldWinner` 的官方表达里，最值得注意的是：

- web + mobile 的 cross-platform
- 20+ years industry experience
- original + licensed games
- 和 `FanDuel` 账户体系打通的 distribution path

这条路线提醒我们：

- 这个赛道不只属于移动端新公司
- 品牌、流量和合规化分发能力也很重要

## 对你最有用的行业结论

### 结论 1：先搞清楚你要做“内容”还是“平台”

对 hackathon 来说，最危险的事就是把平台梦做成项目范围。

更务实的做法通常是：

- 先做一款内容和比赛结构都清楚的小游戏
- 不要在第一版里重造一个 `Skillz`

### 结论 2：这个品类比普通休闲游戏更依赖“公平性感知”

你不只是在做 fun loop，还在做：

- matchmaking 的可信度
- scoring 的可解释性
- payout 流程的确定性
- 风险控制的合理性

### 结论 3：portfolio 能力很重要，但第一步不是 portfolio

大公司最后会做：

- 多品类矩阵
- cross-promo
- 统一钱包/账户/赛事入口

但创新马拉松第一步更应该做：

- 一个 session 够短
- 一种 skill 足够明确
- 一套 scoring 足够清楚
- 一条价值主张能讲明白

## 对内部创新马拉松的启发

你们要比的，不应该是谁做的系统最多，而应该是：

- 谁最清楚 target player
- 谁能把 skill / fairness / progression 讲清楚
- 谁能把 AI 用在提效，而不是堆噱头
