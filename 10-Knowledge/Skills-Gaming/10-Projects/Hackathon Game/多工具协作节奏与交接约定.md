# 多工具协作节奏与交接约定

## 原则

- 一次只允许一个主问题处于“由人拍板”状态
- 所有 AI 工具都围绕同一份玩法规则和 scope 工作
- 先把交接格式定清楚，再并行

## 推荐节奏

### Step 1：人拍板

- 锁玩法
- 锁 score 规则
- 锁 scope

### Step 2：Cursor 主做客户端

- UI shell
- state machine
- duel loop
- result page

### Step 3：Claude Code 主做后端和链路

- API contracts
- event ingestion
- result shaping
- eligibility branch

### Step 4：Codex 做 sidecar

- QA checklists
- scripts
- docs
- review materials

### Step 5：Google AI Studio 做内容层

- copy
- art direction
- trailer
- voice

## 交接格式

每次交接至少写清：

- 当前目标
- 已完成
- 未完成
- 风险
- 下一步建议

## 不要这样做

- 不要多个工具同时改同一块核心逻辑
- 不要让 AI 自己决定玩法规则
- 不要在还没锁 scope 时并行做太多资产
