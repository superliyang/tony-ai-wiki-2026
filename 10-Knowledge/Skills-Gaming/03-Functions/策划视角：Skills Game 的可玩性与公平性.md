# 策划视角：Skills Game 的可玩性与公平性

## 先抓住 4 个问题

1. 这款游戏考的 skill 到底是什么
2. 玩家怎么快速理解 scoring
3. 输了会不会觉得是“我菜”，而不是“系统坑我”
4. 玩 30 秒和玩 30 局的体验，skill expression 有没有层次

## 好的 skills game 常见特征

- session 短
- input 简单
- scoring 清楚
- 有 skill ceiling
- luck 成分可控
- 观感上公平

## 不好的设计信号

- 一局太长
- 输赢原因不清楚
- 新手和高手 skill gap 不可解释
- 很多体验来自随机波动而不是 skill

## 对 hackathon 的直接建议

先做：

- 一个明确 skill
- 一套简单 scoring
- 一种短局模式

先不要做：

- 复杂 meta
- 很多资源系统
- 太多币种和 progression
