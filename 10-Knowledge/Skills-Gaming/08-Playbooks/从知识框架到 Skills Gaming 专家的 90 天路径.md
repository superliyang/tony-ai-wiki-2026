# 从知识框架到 Skills Gaming 专家的 90 天路径

## 目标

把你从：

- `知道赛道很多结构`

推进到：

- `能判断玩法、系统、增长、信任和前沿技术取舍`
- `能带一款 skills game 原型走完设计、实现、测试、复盘`

## 第 1 阶段：赛道和产品判断（1-3 周）

重点：

- 市场与竞品
- 商业模型
- 玩法原型
- skill vs luck 边界

完成标志：

- 能讲清楚为什么某种 loop 适合 `skills gaming`
- 能解释 `Skillz / Papaya / AviaGames / WorldWinner` 的差异

## 第 2 阶段：系统和信任边界（4-6 周）

重点：

- tournament / score / fairness
- payout / risk / support
- eligibility / responsible play
- 争议与 anti-cheat 闭环

完成标志：

- 能画出一条最小 `competition architecture`
- 能说明争议与作弊为什么不是客服问题，而是系统问题

## 第 3 阶段：前沿技术与工具判断（7-9 周）

重点：

- AI-native creation
- editor assistants
- `UGS / PlayFab / Photon` 这类平台差异
- 什么时候该用 AI，什么时候不该

完成标志：

- 能说清楚不同技术层分别解决什么问题
- 能给 hackathon 或 prototype 选出一套合理技术栈

## 第 4 阶段：真实作品与复盘（10-12 周）

重点：

- 做一个可玩 demo
- 跑第一轮内部测试
- 记录 onboarding、drop-off、rematch、disputes
- 做一轮 postmortem

完成标志：

- 至少做出一版真实可玩原型
- 至少做出一轮结构化复盘

## 一句判断

到 90 天时，你不一定已经是“整个游戏行业专家”，但如果你真的完成这四段，
你会非常接近：

- `skills gaming` 方向的产品/系统复合型专家
