# Roblox Studio Assistant 与 MCP

## 为什么它值得研究

因为 Roblox 正在把“游戏创作 assistant”推进到更 agentic 的阶段。

官方信号已经不只是：

- 帮你写点代码

而是：

- 更 context-aware
- 能做 multi-step tasks
- 能兼容 `MCP`
- 能和第三方服务协作

## 官方最关键的 2 个信号

### `RDC 2025`

Roblox 官方写得很明确：

- Assistant 更 agentic、更 context aware
- later this month compatible with `MCP`
- 可以和第三方服务如 `Figma`、`Blockade Labs` 协同

### `2026-02` 更新

Roblox 又继续往前推进：

- Studio built-in MCP server
- external LLM support for Assistant
- playtest automation

## 对游戏团队意味着什么

这条路代表：

- 游戏引擎内 assistant 会越来越像真正的开发代理
- MCP 在 editor 场景里不只是概念，而是工作流协议层

## 对你们这次 hackathon 的启发

即使你们不用 Roblox 做产品，也值得学它的方向：

- editor agent
- built-in context
- third-party orchestration
- playtest automation
