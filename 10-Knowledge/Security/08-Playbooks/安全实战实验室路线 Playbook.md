# 安全实战实验室路线 Playbook

> 用途：把案例、playbook 和能力模型变成可操作练习，形成安全作品集。

## 安全边界

所有练习只允许在授权环境、自己控制的本地实验环境、公司批准的演练环境或公开靶场中进行。

本路线聚焦防御、评审、检测、响应和治理，不以真实未授权攻击为目标。

## 总路线

```text
资产盘点 -> 威胁建模 -> 安全评审 -> 基线治理 -> 检测工程 -> 事件演练 -> 复盘整改 -> 作品集
```

## Lab 0：实验环境与记录规范

目标：先建立安全学习的“可复现”和“可证明”习惯。

要做：

- 为每个实验创建独立记录。
- 写清楚授权范围、环境、目标、时间和数据。
- 所有命令、配置、截图、日志都只保留必要信息。
- 不记录真实密钥、客户数据、个人隐私或生产凭据。

产物：

- 实验记录模板。
- 资产和环境说明。
- 风险边界说明。

## Lab 1：资产与攻击面盘点

目标：训练“先知道保护什么和暴露什么”。

练习：

- 选一个本地 demo 系统或个人项目。
- 列出应用、API、数据、账号、依赖、CI/CD、日志。
- 标记公网入口、管理入口、第三方入口、数据出口。
- 为每个资产补 owner、敏感等级、暴露面、日志状态。

产物：

- 资产清单。
- 攻击面清单。
- crown jewels list。

## Lab 2：威胁建模与控制点设计

目标：训练从系统设计里找攻击路径。

练习：

- 画一个简单数据流图。
- 标出信任边界。
- 按身份、输入、数据、第三方、运维入口列风险。
- 为每个风险设计预防、检测、响应、证据控制。

产物：

- threat model。
- 控制点清单。
- 残余风险说明。

## Lab 3：应用与 API 安全评审

目标：训练上线评审能力。

练习：

- 选一个 demo API 或内部练习系统。
- 检查认证、授权、输入、输出、错误处理、日志、限流、数据返回。
- 重点关注资源归属、租户边界、批量操作、数据导出。
- 写出阻断项、风险接受项、补偿控制和 release gate。

产物：

- API 安全评审报告。
- release gate 清单。
- 修复建议。

关联：

- [[./应用与 API 安全评审 Playbook|应用与 API 安全评审 Playbook]]
- [[../05-Topics/应用安全与 API 安全|应用安全与 API 安全]]

## Lab 4：身份、密钥与零信任评审

目标：训练身份新边界思维。

练习：

- 列出用户、管理员、service account、API key、CI/CD secret。
- 检查 MFA、过期、轮换、权限范围、owner、审计。
- 为高风险系统设计应用级访问和设备态势要求。
- 写一份零信任落地路线。

产物：

- 权限矩阵。
- secret inventory。
- 高权限账号治理方案。
- 零信任路线图。

关联：

- [[./零信任落地 Playbook|零信任落地 Playbook]]
- [[../05-Topics/身份与访问安全|身份与访问安全]]

## Lab 5：云原生安全基线

目标：训练云、容器、Kubernetes 和 IaC 的基础治理。

练习：

- 用模拟配置或实验环境检查云 IAM、网络暴露、存储公开、日志。
- 为 Kubernetes 练习 RBAC、secret、network policy、admission、镜像来源限制。
- 为 IaC 写最小 policy 检查项。
- 设计云审计日志和高风险变更告警。

产物：

- 云安全基线。
- Kubernetes 安全基线。
- IaC policy 清单。
- 云日志覆盖矩阵。

关联：

## Lab 6：供应链安全评审

目标：训练从代码到发布的信任链治理。

练习：

- 选一个仓库，梳理依赖、构建、artifact、镜像、发布流程。
- 检查 branch protection、CODEOWNERS、secret scanning、SCA、SBOM。
- 检查 CI/CD secret 权限、runner 隔离、部署审批、artifact 签名。
- 写供应链风险评审。

产物：

- 供应链安全评审。
- SBOM / 依赖清单。
- CI/CD 加固建议。
- 第三方风险记录。

## Lab 7：漏洞分诊与攻击面治理

目标：训练“不是所有高分漏洞都先修”的风险排序。

练习：

- 构造一组漏洞样例：公网、内网、供应链、云配置、应用/API。
- 为每个问题判断暴露面、可利用性、影响、补偿控制、owner。
- 制定 P0/P1/P2/P3、SLA、例外和验证方式。

产物：

- 漏洞优先级表。
- 修复 SLA 看板。
- 风险接受记录。
- 验证报告。

关联：

## Lab 8：SOC 检测工程

目标：训练把攻击场景转成可运营检测。

练习：

- 选三个场景：异常登录、数据导出异常、云 access key 创建。
- 写 detection hypothesis。
- 定义日志源、字段、触发逻辑、排除逻辑、严重级别。
- 写分诊步骤和响应动作。

产物：

- 三条检测规则设计。
- 日志覆盖矩阵。
- 告警分诊卡。
- 响应动作表。

## Lab 9：事件响应桌面演练

目标：训练组织响应，而不是只修一个技术点。

练习：

- 选择一个案例：云密钥泄露、数据外泄、账号接管、供应链脚本被篡改。
- 组织一条时间线。
- 写分级、止血、取证、通报、恢复、复盘任务。
- 补充检测和治理改进。

产物：

- incident timeline。
- 事件响应记录。
- post-incident review。
- 改进 backlog。

关联：

## Lab 10：GRC 与审计证据包

目标：训练把技术控制变成组织可证明能力。

练习：

- 选一个控制目标，例如高权限账号治理或数据导出审批。
- 写控制描述、owner、频率、证据、例外、指标。
- 准备一份审计证据包。
- 设计 risk register 记录。

产物：

- control mapping。
- risk register。
- audit evidence pack。
- 指标看板草案。

## 作品集标准

完成一轮后，至少应沉淀：

- 一份资产与攻击面清单。
- 一份威胁模型。
- 一份 API 安全评审。
- 一份零信任或身份治理路线。
- 一份云原生安全基线。
- 一份供应链安全评审。
- 一份漏洞分诊表。
- 三条检测规则设计。
- 一份事件响应复盘。
- 一份审计证据包。
- 一份红队 / 蓝队 / 紫队演练计划和检测覆盖矩阵。

这些产物比“看过多少安全文章”更能证明掌握。
