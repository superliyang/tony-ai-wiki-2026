# 手游账号安全检测规则库

> 这页回答：手机游戏账号安全和撞库治理，应该有哪些可落地的检测场景、处置动作和误伤复核机制。

## 规则设计原则

- 规则不要只看单点事件，要看“链路”：登录 -> 会话 -> 换绑 -> 资产 -> 客服 -> 支付。
- 规则不要只按 IP，要按账号、设备、IP、ASN、地区、版本、渠道、完整性、行为一起看。
- 规则必须配处置动作、误伤处理和复盘指标。
- 高风险规则先 shadow 观察，再逐步进入拦截。

## 检测规则矩阵

| ID | 场景 | 触发逻辑 | 动作 | 误伤风险 |
|---|---|---|---|---|
| GA-001 | 撞库扫描 | 同一 IP/ASN/设备簇短时间尝试大量账号且失败率高 | 限速、验证码、样本入库 | 网吧/校园网络 |
| GA-002 | 密码喷洒 | 多账号使用少量常见密码失败 | 限速、泄露密码提示、密码重置建议 | 大促期间正常失败 |
| GA-003 | 账号被多源尝试 | 同一账号被多个 IP/设备快速尝试 | 账号冷却、通知用户 | 用户多设备登录 |
| GA-004 | 新设备成功登录 | 陌生设备 + 异常 IP/ASN + 成功登录 | step-up、会话标记 | 换手机/旅行 |
| GA-005 | ATO 后换绑 | 高风险登录后短时间换绑手机/邮箱/第三方账号 | 冷却期、二次验证、人工审核 | 正常找回 |
| GA-006 | ATO 后资产转移 | 高风险登录后赠送/交易/出售/大额消耗 | 冻结资产、人工复核 | 正常高活跃玩家 |
| GA-007 | 找回后敏感动作 | 找回成功后快速换绑/资产转移/退款 | 延迟生效、升级审核 | 真实被盗找回 |
| GA-008 | 客服社工聚类 | 同源 IP/设备/材料提交多个找回请求 | 工单升级、材料复核 | 家庭/工作室共享网络 |
| GA-009 | 设备农场 | 同设备簇或模拟器簇登录大量账号 | 限速、设备降权、挑战 | 多开玩家 |
| GA-010 | 完整性失败高价值动作 | integrity fail 后登录高价值账号或发起交易 | 拦截/二次验证 | 误判设备环境 |
| GA-011 | 异常地区漂移 | 短时间跨远距离登录且设备变化 | step-up、通知用户 | VPN/旅行 |
| GA-012 | 退款套利 | 充值后快速消耗/转移再退款或拒付 | 冻结、人工审核、渠道对账 | 正常退款 |
| GA-013 | GM 高危操作异常 | 非工作时段、大批量、无审批补偿/解封/发道具 | 阻断、审批、审计告警 | 紧急运营活动 |
| GA-014 | 经济搬运链 | 多账号向少数账号集中转移资产 | 冻结、关联账号调查 | 公会/好友正常赠送 |
| GA-015 | 排行榜异常跃迁 | 新设备/完整性失败账号战绩突增 | 延迟上榜、回放复核 | 高手换设备 |

## 规则上线分级

| 模式 | 适用场景 |
|---|---|
| Shadow | 新规则，先观察命中样本和误伤 |
| Alert | 需要人工分诊，但暂不自动拦截 |
| Challenge | 触发验证码、二次验证、设备确认 |
| Delay | 延迟换绑、交易、发货、退款生效 |
| Freeze | 冻结资产、会话、交易或高风险动作 |
| Block | 明确恶意或高风险不可逆动作 |

## 处置动作库

| 动作 | 说明 | 注意 |
|---|---|---|
| IP/ASN 限速 | 对代理池和 IDC 有效 | 不要只靠封 IP |
| 账号冷却 | 保护被集中尝试账号 | 要有用户提示 |
| 新设备 step-up | 保护成功登录前后 | 不要全量打扰 |
| session revoke | 撤销可疑会话 | 要避免踢掉正常用户 |
| 资产冻结 | 防止盗号后搬运 | 要有申诉和复核 |
| 换绑冷却 | 防止接管账号 | 高风险账号优先 |
| 人工审核 | 处理高价值或高误伤场景 | 要有 SLA |
| 用户通知 | 新设备、换绑、改密、资产操作 | 要防钓鱼文案 |
| 黑名单/情报 | IP、设备、账号簇、工具特征 | 要设置过期机制 |

## 误伤复核

误伤常见来源：

- 网吧、校园、公司出口。
- VPN / 旅行。
- 玩家换手机。
- 高活跃玩家频繁交易。
- 公会内部赠送。
- 家庭共用设备。
- 老设备完整性失败。

复核要看：

- 历史设备是否一致。
- 行为是否符合玩家历史。
- 是否有资产异常流出。
- 是否有客服工单。
- 是否有相同源攻击其他账号。
- 是否有完整性失败和模拟器特征。

## 指标闭环

| 指标 | 目标 |
|---|---|
| 撞库成功率 | 越低越好 |
| 高风险登录后资产转移率 | 越低越好 |
| 规则误伤率 | 可解释、可控 |
| 用户申诉成功率 | 用来校正规则 |
| 平均处置时间 | 越短越好 |
| 规则绕过周期 | 越长越好 |
| 复犯账号/设备簇 | 持续下降 |
