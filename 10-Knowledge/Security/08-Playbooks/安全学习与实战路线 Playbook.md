# 安全学习与实战路线 Playbook

> 用途：把安全知识库变成一条可执行路线：先学什么、怎么练、能产出什么、什么时候算掌握。

## 总路线

不要按工具学习安全，要按能力演进：

`安全思维 -> 基础控制 -> 攻击面理解 -> 工程评审 -> 运营响应 -> 治理证明 -> 架构设计`

## 阶段 1：建立安全世界观

目标：知道安全到底保护什么，风险从哪里来。

必读：

- [[../06-Maps/安全上帝视角全景架构图|安全上帝视角全景架构图]]
- [[../06-Maps/安全点线面能力地图|安全点线面能力地图]]
- [[../05-Topics/安全领域主干|安全领域主干]]
- [[../05-Topics/安全能力模型：知道什么、会做什么|安全能力模型：知道什么、会做什么]]

要能回答：

- 资产、攻击面、威胁、控制、检测、响应、治理之间是什么关系？
- 为什么网络安全只是安全的一部分？
- 为什么身份、数据、云和供应链变得越来越关键？

练习：

- 选一个熟悉系统，列出资产、入口、身份、数据、第三方、日志。
- 用 [[./安全快速诊断 Playbook|安全快速诊断 Playbook]] 写一份 1 页风险诊断。

通过标准：

- 能把一个模糊安全问题翻译成 `资产 -> 攻击面 -> 威胁 -> 影响 -> 控制`。

## 阶段 2：掌握基础控制面

目标：掌握安全系统的基础组件。

必读：

- [[../05-Topics/身份与访问安全|身份与访问安全]]
- [[../05-Topics/数据安全与隐私保护|数据安全与隐私保护]]
- [[../05-Topics/密码学、密钥与证书基础|密码学、密钥与证书基础]]
- [[../05-Topics/网络安全|网络安全]]
- [[../05-Topics/应用安全与 API 安全|应用安全与 API 安全]]

要能回答：

- 认证和授权的区别是什么？
- 为什么最小权限比单纯防火墙更重要？
- 加密、脱敏、tokenization 分别解决什么问题？
- API 安全为什么经常败在授权和业务逻辑上？

练习：

- 为一个 API 写认证、授权、限流、日志、错误处理检查表。
- 为一个敏感数据表设计访问控制、加密、脱敏、保留删除方案。

通过标准：

- 能对一个系统提出基础安全控制组合，而不是只说“加 WAF”或“做加密”。

## 阶段 3：理解攻击面和工程落地

目标：知道风险如何在真实工程系统里出现。

必读：

- [[../06-Maps/攻击面与控制点图|攻击面与控制点图]]
- [[../05-Topics/威胁建模|威胁建模]]
- [[../05-Topics/安全工程与 DevSecOps|安全工程与 DevSecOps]]
- [[../05-Topics/云原生与基础设施安全|云原生与基础设施安全]]
- [[../05-Topics/供应链安全|供应链安全]]
- [[../05-Topics/漏洞管理与攻击面管理|漏洞管理与攻击面管理]]

要能回答：

- 一个系统从设计到上线有哪些安全 gate？
- 云 IAM、Kubernetes、CI/CD、依赖为什么会变成攻击入口？
- 漏洞优先级为什么不能只看 CVSS？

练习：

- 用 [[./应用与 API 安全评审 Playbook|应用与 API 安全评审 Playbook]] 评审一个系统。
- 为一个服务画出数据流图和信任边界。
- 列出 CI/CD 里的 secret、依赖、构建、发布风险。

通过标准：

- 能把安全控制放进研发和发布流程里。

## 阶段 4：掌握运营与事件响应

目标：攻击来了能看见、判断、止血、恢复和复盘。

必读：

- [[../05-Topics/安全运营、检测与响应|安全运营、检测与响应]]
- [[../06-Maps/安全运营与事件响应闭环图|安全运营与事件响应闭环图]]
- [[./安全事件响应 Playbook|安全事件响应 Playbook]]
- [[../05-Topics/安全指标与成熟度模型|安全指标与成熟度模型]]

要能回答：

- 哪些日志必须优先接入？
- 告警如何分级？
- MTTD、MTTR 和控制覆盖率分别说明什么？
- 事件响应和漏洞修复有什么不同？

练习：

- 写一个云密钥泄露的事件响应时间线。
- 为登录异常、数据导出异常、权限变更异常设计检测点。
- 做一次 tabletop exercise 的复盘模板。

通过标准：

- 能组织一次常见安全事件响应，并产出复盘和改进任务。

## 阶段 5：形成治理与架构能力

目标：把安全变成组织能力，而不是个人经验。

必读：

- [[../05-Topics/企业安全架构|企业安全架构]]
- [[./企业安全架构落地 Playbook|企业安全架构落地 Playbook]]
- [[../05-Topics/零信任架构|零信任架构]]
- [[../05-Topics/安全治理、风险与合规|安全治理、风险与合规]]
- [[../05-Topics/地区合规与监管坐标|地区合规与监管坐标]]
- [[../01-Standards/安全标准与法规索引|安全标准与法规索引]]

要能回答：

- 安全团队什么时候做 gatekeeper，什么时候做 platform enabler？
- 风险接受需要哪些条件？
- 合规要求如何映射到技术控制和证据？
- 企业安全能力路线图应该如何排序？

练习：

- 为一个中型企业写安全能力路线图。
- 建一个 risk register 样例。
- 把一个合规要求映射到控制、日志、证据和 owner。

通过标准：

- 能做安全架构决策，并解释取舍、成本、风险和落地路径。

## 安全学习不要踩的坑

- 只学攻击技巧，不学资产、控制和治理。
- 只学合规条款，不理解真实攻击路径。
- 只买工具，不建立 owner、流程和证据。
- 只看漏洞严重性，不看暴露面和业务影响。
- 只写安全建议，不跟踪修复、检测和复盘。
- 只做一次性项目，不建设默认安全平台。
