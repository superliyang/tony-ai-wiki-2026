# 手游撞库与账号安全治理 Playbook

> 这份 Playbook 面向手机游戏被撞库后的止血、排查、加固和体系化建设。

## 先判断：这是不是撞库

撞库常见信号：

- 登录失败量突然上升，但不是单个账号持续失败，而是大量账号分散尝试。
- 成功登录来自陌生设备、代理 IP、异常 ASN、异常国家地区或异常时间段。
- 同一 IP / ASN / 设备簇尝试大量账号。
- 同一账号被多个 IP 或设备快速尝试。
- 登录成功后立刻发生换绑、改密、道具转移、充值退款、商城消费、客服申诉。
- App 版本、渠道包、模拟器、Root/越狱信号或设备完整性失败比例异常。

## War Room 角色

| 角色 | 负责什么 |
|---|---|
| 安全负责人 | 攻击研判、处置策略、证据保全、复盘 |
| 账号/后端 | 登录、会话、找回、换绑、token、限速 |
| 客户端 | 设备完整性、版本、渠道包、模拟器/Root/越狱信号 |
| 数据/风控 | 指标、规则、样本聚类、资产损失评估 |
| 运营/客服 | 玩家沟通、申诉、误伤处理、补偿策略 |
| 支付/经济 | 充值、退款、道具、货币、交易、账本 |
| 法务/合规 | 用户通知、隐私、监管、证据留存 |

## 0-2 小时：止血开关

### 1. 拉攻击大盘

必须看到：

- 登录失败量、成功量、失败/成功比。
- Top IP、Top ASN、Top 国家地区、Top 设备、Top App 版本、Top 渠道。
- 被尝试账号数量、成功账号数量、高价值账号数量。
- 成功后敏感动作：换绑、改密、找回、道具转移、充值退款、客服申诉。

### 2. 先做低误伤限速

优先策略：

- 同一 IP 对多账号失败：限速。
- 同一账号被多 IP 尝试：账号级冷却。
- 同一设备尝试多账号：设备级限速。
- 高风险 ASN / 代理 / IDC：提高风险分。
- 登录失败后继续高频尝试：指数退避。

不建议第一时间全局强验证码，除非攻击已经压垮登录服务。

### 3. 保护敏感动作

对高风险账号或高风险登录后动作立即加保护：

- 换绑邮箱/手机号：二次验证 + 冷却期。
- 修改密码：二次验证 + 旧设备通知。
- 道具赠送/交易/出售：临时限额或冻结。
- 大额消耗/提现/退款：人工审核或延迟生效。
- 客服找回：提高证据要求，防社工。

## 2-24 小时：确认影响面

## 查询问题

| 问题 | 需要的数据 |
|---|---|
| 有多少账号被尝试？ | login_attempt 表 |
| 有多少账号成功登录？ | login_success + risk signals |
| 成功账号里多少有资产损失？ | inventory / economy ledger |
| 是否发生换绑和找回滥用？ | account_change / recovery logs |
| 是否有同一设备或代理簇？ | device_id / fingerprint / IP / ASN |
| 是否来自旧版本或改包？ | app version / channel / integrity verdict |
| 是否有客服流程被利用？ | support tickets / operator actions |

## 处置分层

| 风险等级 | 条件 | 处置 |
|---|---|---|
| P0 | 大量成功登录 + 资产转移 | 全局高风险策略、冻结高风险资产转移、war room 持续响应 |
| P1 | 大量尝试 + 少量成功 | 风险限速、step-up、可疑 session 下线 |
| P2 | 局部代理池尝试 | IP/ASN/设备限速、规则观察 |
| P3 | 单账号异常 | 用户通知、改密建议、客服跟进 |

## 24-72 小时：加固策略

### 账号与认证

- 引入泄露密码检测：注册、登录、改密时识别已泄露密码。
- 高风险登录 step-up：短信、邮箱、设备确认、平台二次验证。
- 会话治理：刷新 token 轮换、异常会话撤销、设备列表管理。
- 找回治理：找回链路加风控，不允许找回后立即转移资产。
- 通知机制：新设备登录、换绑、改密、敏感资产操作要通知用户。

### Bot 与设备

- Android：接入 Play Integrity，服务端验证 token，不在客户端本地做最终判断。
- iOS：接入 App Attest / DeviceCheck，保护敏感 API 和高风险动作。
- 设备画像：设备、账号、IP、渠道、版本、模拟器、Root/越狱、完整性信号统一入风控。
- 行为识别：登录频率、输入速度、失败模式、登录后动作链路。

### 游戏资产保护

- 账本化：道具、货币、奖励、充值、退款必须可追溯。
- 冻结机制：高风险登录后，对交易、赠送、出售、大额消耗设置延迟或冻结。
- 异常经济规则：高价值账号异常转移、高频小额搬运、批量角色转移。
- 误伤复核：封禁、冻结、资产回滚必须有证据和申诉流程。

### API 与服务端

- 登录接口按账号、IP、设备、ASN 多维限速。
- 敏感接口做对象级授权，不能只信客户端传参。
- 支付回调验签、订单幂等、奖励发放幂等。
- 登录、支付、经济、客服、后台操作统一 trace id。

## 7-30 天：形成账号安全平台

目标不是“补几个规则”，而是形成一套账号安全与游戏风控底座：

```text
信号采集 -> 风险评分 -> 策略决策 -> 动作执行 -> 用户/客服闭环 -> 复盘调参
```

## 必建能力

| 能力 | 说明 |
|---|---|
| 风险信号中心 | IP、ASN、设备、完整性、账号历史、行为、资产、客服 |
| 策略引擎 | 限速、验证码、step-up、冻结、人工审核、放行 |
| 账号安全中心 | 登录、会话、MFA、找回、换绑、设备列表 |
| 经济风控 | 道具、货币、交易、充值、退款、补偿 |
| 检测规则库 | 撞库、密码喷洒、批量注册、异常换绑、资产搬运 |
| Case 管理 | 告警、证据、处置、申诉、复盘 |
| 指标看板 | 攻击量、成功率、误伤率、损失、响应时间 |

## 检测规则样例

### 撞库检测

```text
同一 IP / ASN / 设备簇在 10 分钟内尝试登录 N 个不同账号，且失败率高于阈值。
动作：限速、验证码、提高风险分、记录样本。
```

### 账号接管检测

```text
账号从陌生设备成功登录后 30 分钟内发生换绑、改密、资产转移或客服申诉。
动作：step-up、撤销会话、冻结敏感操作、通知用户。
```

### 资产搬运检测

```text
高风险登录后出现异常赠送、交易、出售、批量消耗或异常退款。
动作：冻结交易、人工复核、保留证据、必要时回滚。
```

### 客服社工检测

```text
同一设备/IP/身份材料关联多个找回请求，或找回后快速换绑和资产转移。
动作：升级审核、延迟生效、限制敏感操作。
```

## 指标体系

| 指标 | 用途 |
|---|---|
| 撞库尝试账号数 | 攻击规模 |
| 撞库成功率 | 防护有效性 |
| 高风险成功登录数 | 真实风险 |
| 资产损失金额/道具价值 | 业务影响 |
| 可疑 session 撤销数 | 止血动作 |
| step-up 通过/失败率 | 策略质量 |
| 误伤率和申诉成功率 | 用户体验 |
| 平均响应时间 MTTD/MTTR | 安全运营能力 |
| 规则命中后复犯率 | 黑产适应程度 |

## 与领导沟通时要讲清楚

- 撞库不是公司密码库一定泄露，而是外部泄露密码被拿来批量尝试。
- 真正损失不在“失败登录多”，而在“成功登录后资产和账号控制权被转移”。
- 防护不能只靠验证码，要把账号、设备、IP、行为、资产、客服和支付串起来。
- 安全目标不是零攻击，而是降低成功率、限制损失半径、缩短发现和处置时间、沉淀长期能力。
