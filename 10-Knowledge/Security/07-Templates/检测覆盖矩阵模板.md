# 检测覆盖矩阵模板

> 用途：把攻击场景、ATT&CK TTP、日志源、检测规则和响应动作放在一张表里。

## 矩阵

| 场景 | 资产 | ATT&CK Tactic | Technique | 日志源 | 检测逻辑 | 分诊上下文 | 响应动作 | 覆盖状态 | Owner |
|---|---|---|---|---|---|---|---|---|---|
| 账号异常 | 高权限账号 |  |  | IdP / EDR |  |  | 强制登出 / 禁用账号 |  |  |
| 云密钥泄露 | 云账号 |  |  | Cloud audit log |  |  | 禁用 key / 轮换凭据 |  |  |
| 数据导出异常 | 敏感数据 |  |  | DB / App audit |  |  | 冻结导出 / 升级事件 |  |  |
| CI/CD 异常 | 发布流水线 |  |  | CI/CD logs |  |  | 暂停流水线 / 轮换 secret |  |  |
| K8s 高权限滥用 | 集群 |  |  | K8s audit log |  |  | 回收权限 / 隔离 workload |  |  |

## 覆盖状态

- `无日志`：没有必要日志源。
- `有日志无规则`：日志存在但未写检测。
- `有规则无分诊`：告警存在但上下文不足。
- `可分诊无响应`：能判断但没有动作。
- `闭环`：检测、分诊、响应、证据、复盘都可执行。

## 复查问题

- 是否覆盖关键资产？
- 是否覆盖真实攻击路径？
- 是否过度依赖单一日志源？
- 检测是否有排除逻辑？
- 告警是否能定位主体、资源、时间和影响？
- 响应动作是否经过授权和演练？
