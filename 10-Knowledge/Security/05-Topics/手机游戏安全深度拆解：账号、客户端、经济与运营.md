# 手机游戏安全深度拆解：账号、客户端、经济与运营

> 这页回答：手机游戏安全为什么不能只看“撞库”或“外挂”，而要按端到端业务资产、攻击路径、控制面和运营闭环来设计。

## 一句话定位

手机游戏安全的核心不是保护一个系统，而是保护一条连续业务链：

`玩家身份 -> 设备与客户端 -> 登录会话 -> 游戏行为 -> 战斗结算 -> 虚拟经济 -> 支付充值 -> 客服运营 -> 数据与后台 -> 安全运营`

只要其中任意一段失守，黑产就能把技术风险变成真实业务损失。

## 资产分层

| 层级 | 资产 | 为什么重要 |
|---|---|---|
| 账号资产 | 账号、角色、绑定手机/邮箱、第三方登录、会话 | 撞库和盗号的直接目标 |
| 设备资产 | 设备指纹、完整性信号、模拟器/Root/越狱状态 | 黑产自动化和改包入口 |
| 客户端资产 | APK/IPA、资源包、热更、脚本、配置、SDK | 改包、Hook、逆向、供应链投毒入口 |
| 游戏公平性 | 战斗、匹配、排行榜、回放、行为轨迹 | 外挂和刷榜影响玩家留存 |
| 虚拟经济 | 货币、道具、背包、交易、赠送、奖励、库存 | 撞库成功后的资产搬运目标 |
| 支付资产 | 订单、渠道回调、充值、退款、拒付、对账 | 直接资金损失与渠道风险 |
| 运营权限 | GM、客服、活动、补偿、封禁、数据查询 | 内部滥用和社工攻击放大器 |
| 数据资产 | PII、未成年人信息、聊天、行为、风控画像 | 隐私、合规和舆情风险 |
| 交付资产 | CI/CD、依赖、SDK、签名、制品、配置中心 | 供应链攻击进入生产路径 |

## 黑产攻击链

### 1. 撞库与账号接管链

```text
外部泄露账号密码 -> 代理池/设备农场批量尝试 -> 少量成功登录 -> 换绑/改密/下线原设备 -> 转移道具/货币/权益 -> 客服申诉或转卖
```

关键防线：

- 登录风险分层。
- 泄露密码与弱密码识别。
- 新设备 step-up。
- 换绑冷却期。
- 资产转移冻结。
- 客服找回防社工。

### 2. 客户端自动化链

```text
改包/Hook/模拟器/脚本 -> 绕过客户端校验 -> 批量登录/注册/任务/活动 -> 产出资源 -> 转移或售卖
```

关键防线：

- Android Play Integrity。
- iOS App Attest / DeviceCheck。
- 签名与资源完整性。
- 关键逻辑服务端化。
- 自动化行为检测。
- 活动与奖励风控。

### 3. 战斗与结算作弊链

```text
内存修改/加速/封包篡改 -> 伪造战斗过程或结果 -> 异常结算 -> 排行榜/奖励/段位收益 -> 影响公平和经济
```

关键防线：

- 服务端权威计算。
- 战斗回放校验。
- 行为异常模型。
- 结算幂等与二次校验。
- 排行榜延迟发布和复核。

### 4. 支付与经济套利链

```text
伪造订单/重放回调/退款套利/拒付 -> 重复发货或未扣回 -> 虚拟资产套利 -> 黑市变现
```

关键防线：

- 渠道回调验签。
- 订单幂等。
- 账本一致性。
- 充值-消耗-退款联动。
- 高风险退款冻结。

### 5. 后台与客服放大链

```text
客服社工/运营后台越权/GM 误操作 -> 改绑/补偿/发道具/解封 -> 黑产绕过技术防线
```

关键防线：

- RBAC / PAM。
- 高危操作审批。
- 双人复核。
- 全量审计。
- 工单与账号风险联动。

## 控制面分解

### 1. 预防控制

- MFA / step-up：用于新设备、高价值账号、换绑、找回、交易和退款。
- 多维限速：账号、IP、设备、ASN、国家地区、失败模式。
- 设备完整性：Play Integrity、App Attest、DeviceCheck。
- 服务端权威：战斗、结算、奖励、库存不能只信客户端。
- 最小权限：GM、客服、数据平台、CI/CD 都要最小权限与审批。

### 2. 检测控制

- 撞库检测：大量账号尝试、高失败率、代理池、设备簇。
- ATO 检测：新设备成功登录后发生敏感操作。
- 经济检测：异常产出、异常转移、异常消耗、退款套利。
- 客户端检测：完整性失败、模拟器集中、版本异常、Hook 行为。
- 后台检测：高危操作、异常客服工单、批量补偿、非工作时段操作。

### 3. 响应控制

- 可疑 session 撤销。
- 账号临时保护。
- 资产冻结与延迟转移。
- 高风险退款人工审核。
- 封禁、解封和回滚证据链。
- 客服申诉分级。

### 4. 恢复与复盘

- 账号找回和资产恢复流程。
- 玩家通知和客服话术。
- 误伤回滚和补偿策略。
- 规则命中、误伤、绕过路径复盘。
- 把事件转成规则、监控、评审项和上线门禁。

## 组织责任面

| 团队 | 必须负责的安全面 |
|---|---|
| 账号/平台 | 登录、会话、MFA、找回、换绑、账号风控 |
| 服务端 | API 鉴权、战斗结算、库存账本、支付幂等 |
| 客户端 | 完整性、反调试、签名、渠道包、敏感数据本地存储 |
| 数据/风控 | 信号接入、规则、模型、看板、风险样本 |
| 安全 | 威胁模型、检测规则、事件响应、证据、复盘 |
| 运营 | 活动规则、补偿、封禁、公告、玩家体验 |
| 客服 | 申诉、找回、防社工、误伤处理 |
| 支付 | 渠道验签、对账、退款、拒付、风控 |
| 法务/隐私 | 用户通知、个人信息、未成年人、监管口径 |

## 架构师应该问的 30 个问题

### 账号

- 新设备登录是否触发风险评估？
- 找回后是否允许立即换绑和转移资产？
- 高价值账号是否有更强保护？
- 是否能撤销可疑 session？
- 是否知道外部泄露密码命中情况？

### 客户端

- Android / iOS 是否有设备完整性验证？
- 完整性 verdict 是否在服务端验证？
- 改包、Hook、模拟器是否进入风控画像？
- 热更和资源包是否签名？
- 敏感配置是否能被客户端轻易提取？

### API 与服务端

- 是否存在只靠客户端校验的业务规则？
- 战斗结果是否服务端可验证？
- 奖励发放是否幂等？
- 对象级授权是否覆盖背包、订单、角色、工单？
- 所有关键链路是否有 trace id？

### 经济与支付

- 货币和道具是否账本化？
- 异常产出和异常转移是否能实时发现？
- 充值、发货、退款、拒付是否闭环？
- 高风险登录后是否限制资产转移？
- 经济规则变更是否安全评审？

### 运营后台

- GM 权限是否最小化？
- 发道具、封禁、解封、补偿是否审批？
- 客服能否越权改绑？
- 高危操作是否全量审计？
- 内部账号是否有 MFA 和离职回收？

### 安全运营

- 撞库攻击能在几分钟内被发现？
- 有哪些规则能识别 ATO 成功后的资产搬运？
- 告警是否能直接关联账号、设备、IP、订单、工单？
- 误伤如何申诉和复核？
- 每次攻击后规则是否回归测试？
