# 企业安全架构

## 定位

企业安全架构是把分散的安全控制组织成一套可运行、可审计、可演进的系统。

它的目标不是“每个点都有工具”，而是：

`业务资产 -> 风险模型 -> 控制架构 -> 检测响应 -> 治理证据 -> 持续改进`

## 核心视图

### 1. 资产视图

- 应用
- 数据
- 身份
- 云资源
- 终端
- 第三方
- 关键业务流程

### 2. 风险视图

- 外部攻击
- 内部威胁
- 配置错误
- 数据泄露
- 供应链投毒
- 合规违规
- 业务滥用

### 3. 控制视图

- 身份访问
- 网络隔离
- 应用安全
- 数据保护
- 云安全
- 供应链安全
- 安全运营

### 4. 治理视图

- policy
- risk register
- control owner
- exception
- audit evidence
- security metrics

## 架构师最该问的问题

- 哪些资产最关键
- 哪些风险最可能且影响最大
- 哪些控制是预防，哪些是检测
- 哪些控制缺少 owner
- 哪些系统缺少日志
- 哪些合规要求需要证据
- 哪些事故复盘没有转成机制
