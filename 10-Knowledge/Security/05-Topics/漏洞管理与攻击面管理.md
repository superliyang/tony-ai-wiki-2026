# 漏洞管理与攻击面管理

## 定位

漏洞管理关注已知弱点如何发现、分级、修复和验证。

攻击面管理关注组织到底暴露了哪些资产、入口、服务、依赖和第三方路径。

两者必须结合：不知道攻击面，漏洞管理就不完整；只知道漏洞，不知道业务暴露面，也无法排序。

## 核心流程

1. 资产发现
2. 暴露面识别
3. 漏洞扫描与情报关联
4. 风险分级
5. 修复 owner 分配
6. SLA 跟踪
7. 修复验证
8. 例外与风险接受
9. 指标和复盘

## 分级不只看 CVSS

还要看：

- 是否公网暴露
- 是否已有 exploit
- 是否影响敏感数据
- 是否有补偿控制
- 是否在关键业务路径
- 是否可横向移动
- 是否有供应链影响

## 常见指标

- critical vulnerability count
- mean time to remediate
- SLA breach rate
- exposed asset count
- unknown asset count
- patch coverage
- exception aging
