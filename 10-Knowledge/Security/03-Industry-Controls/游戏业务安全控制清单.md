# 游戏业务安全控制清单

> 游戏安全的核心是：账号资产、虚拟经济、公平性、客户端完整性、支付充值、未成年人和社区安全。

## 保护对象

- 玩家账号、角色、道具、货币、交易和排行榜。
- 客户端、反作弊、补丁、资源包、脚本接口。
- 登录、匹配、战斗、结算、商城、充值、退款。
- 游戏服务器、实时通信、反作弊数据、运营后台。
- 未成年人数据、聊天内容、举报和社区治理记录。

## 主要威胁

- 外挂、脚本、内存修改、封包篡改、模拟器农场。
- 账号盗号、撞库、黑产洗号、道具盗卖。
- 虚拟经济刷币、重复领取、退款套利、充值欺诈。
- DDoS、游戏服漏洞、匹配/结算逻辑绕过。
- 客户端逆向、资源篡改、签名绕过。
- 聊天诈骗、未成年人隐私、内容合规和社区滥用。

## 控制清单

更细的端到端控制矩阵见：[[./手机游戏端到端安全控制矩阵|手机游戏端到端安全控制矩阵]]。
如果当前问题是撞库、盗号或账号接管，优先使用：[[../08-Playbooks/手游撞库与账号安全治理 Playbook|手游撞库与账号安全治理 Playbook]]。

| 控制域 | 必做控制 | 证据 |
|---|---|---|
| 账号安全 | MFA/二次验证、设备风险、异常登录、找回保护 | 登录日志、风控命中、申诉记录 |
| 客户端安全 | 完整性校验、反调试、签名、敏感逻辑服务端化 | 客户端评审、反作弊策略 |
| 反作弊 | 行为检测、内存/进程检测、回放分析、封禁复核 | 检测规则、封禁证据、误封复盘 |
| 经济安全 | 交易限额、重复领取防护、库存一致性、异常产出检测 | 风控规则、经济报表、case |
| 支付充值 | 渠道验签、订单幂等、退款风控、对账 | 订单日志、验签策略、对账记录 |
| 后台权限 | 运营后台 RBAC、敏感操作审批、全量审计 | 权限矩阵、审批、审计日志 |
| DDoS 与可用性 | 高防、弹性、游戏服隔离、降级预案 | 压测、演练、事故复盘 |
| 社区与未成年 | 内容审核、举报、隐私最小化、年龄相关控制 | 审核记录、隐私评审、处理记录 |

## 平台组合

- IAM / PAM：运营后台、GM 权限、封禁和道具发放。
- AppSec / 客户端安全：移动端/PC 客户端、API、补丁链路。
- SIEM / 风控平台：登录、交易、作弊、聊天、后台操作。
- DDoS / 网络安全：游戏服、登录服、匹配服、实时通信。
- DLP / 隐私：未成年人、聊天内容、用户画像和客服数据。
- GRC：平台规则、隐私合规、事故证据和客服申诉。

## 关键指标

- 账号盗号率和找回成功率。
- 外挂检测命中率与误封率。
- 虚拟经济异常产出数量。
- 支付退款欺诈率。
- DDoS 导致的不可用时间。
- 敏感后台操作审计覆盖率。

## 常见陷阱

- 把作弊检测只放在客户端，服务端没有行为和经济一致性校验。
- 只封号不复盘，导致黑产换路径继续打。
- 运营后台权限过大，人工补偿、道具发放和封禁缺少审计。
- 忽略未成年人、聊天、社区和隐私治理，把游戏安全只等同于反外挂。
