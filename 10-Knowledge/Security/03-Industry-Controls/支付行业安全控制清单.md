# 支付行业安全控制清单

> 支付安全的核心是：保护支付凭证、交易链路、资金账户、风控决策、清结算数据和合规证据。

## 保护对象

- 卡数据、PAN、CVV、token、支付凭证、设备指纹。
- 商户、用户、资金账户、提现账户、收款账户。
- 交易下单、授权、扣款、退款、拒付、清结算和对账链路。
- 风控模型、规则、黑名单、人工审核记录。
- 支付通道、收单机构、第三方 SDK、Webhook 和回调。

## 主要威胁

- 账号接管、盗刷、撞库、钓鱼、设备劫持。
- API 越权、金额篡改、重复扣款、重放攻击、签名绕过。
- 卡数据泄露、日志误打敏感字段、测试环境泄露真实数据。
- Webhook 伪造、回调重放、商户配置错误。
- 内部高权限滥用、退款/提现流程绕过。
- 第三方支付通道、SDK、供应商和对账文件泄露。

## 控制清单

| 控制域 | 必做控制 | 证据 |
|---|---|---|
| 身份与权限 | MFA、商户/运营后台 RBAC、高权限审批、JIT 权限 | 登录日志、权限矩阵、审批记录 |
| 支付凭证保护 | Tokenization、加密、密钥轮换、禁止明文卡数据入库 | 数据流图、KMS/Vault 记录 |
| API 安全 | 请求签名、幂等、重放防护、金额服务端校验、rate limit | API 评审、网关策略、测试报告 |
| 风控 | 设备、行为、地理、交易速度、异常退款/提现规则 | 风控规则、命中记录、人工审核 |
| 日志与检测 | 支付关键事件进 SIEM，异常交易和高权限操作告警 | 检测规则、告警 case、响应记录 |
| 数据安全 | PAN 脱敏、最小化存储、日志脱敏、测试数据脱敏 | DLP 规则、扫描报告、抽样证据 |
| 供应链 | 支付 SDK、通道、Webhook、对账文件安全评审 | 供应商评估、接口验签、传输记录 |
| 合规证据 | PCI DSS 范围界定、控制映射、审计证据 | 控制库、证据包、例外记录 |

## 平台组合

- IAM / PAM：运营后台、高权限退款、通道配置、密钥操作。
- Secrets / KMS：支付签名密钥、通道密钥、证书、webhook secret。
- AppSec / DevSecOps：支付 API、SDK、依赖、secret scanning、release gate。
- SIEM / SOAR：异常登录、异常交易、高权限操作、Webhook 攻击。
- DLP / DSPM：卡数据、身份数据、对账文件、日志敏感字段。
- GRC：PCI DSS、客户审计、供应商风险、控制证据。

## 关键指标

- 高风险交易拦截率与误杀率。
- 支付 API 高危问题进入生产数量。
- 卡数据和敏感日志泄露事件数。
- 高权限操作审计覆盖率。
- Webhook 验签覆盖率。
- 拒付、退款异常和账号接管事件趋势。

## 常见陷阱

- 只把支付安全理解成 PCI DSS，而忽略业务逻辑和资金风险。
- 只做交易风控，不做后台权限和密钥治理。
- 回调只看来源 IP，不做签名、时间戳、幂等和重放防护。
- 日志、BI、客服后台、对账文件成为卡数据和个人信息的旁路泄露点。
