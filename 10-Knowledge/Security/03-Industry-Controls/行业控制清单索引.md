## 行业迁移原则

安全主干不变：

`资产 -> 攻击面 -> 威胁 -> 控制 -> 检测 -> 响应 -> 证据`

行业差异在于：

- 资产不同：支付重交易和卡数据，医疗重健康数据，SaaS 重租户和企业客户数据，游戏重账号资产和公平性，AI 重模型、数据、推理链路和 agent 行为。
- 威胁不同：支付有盗刷和欺诈，医疗有勒索和 PHI 外泄，SaaS 有租户隔离和供应链，游戏有外挂和黑产，AI 有 prompt injection、数据泄露和模型滥用。
- 证据不同：支付看 PCI DSS，医疗看 HIPAA 等监管，SaaS 看 SOC 2/ISO 27001，游戏看平台与未成年人/隐私要求，AI 看 AI 风险管理与安全评测。
- 平台组合不同：行业不是“多买工具”，而是组合 IAM、AppSec、CNAPP、SIEM、DLP、GRC、业务风控和审计证据。

## 官方参考入口

- 支付：[PCI Security Standards Council](https://www.pcisecuritystandards.org/)
- 医疗：[HHS HIPAA Security Rule](https://www.hhs.gov/hipaa/for-professionals/security/index.html)
- SaaS：[AICPA SOC 2](https://www.aicpa-cima.com/resources/landing/system-and-organization-controls-soc-suite-of-services)
- 通用安全：[NIST Cybersecurity Framework](https://www.nist.gov/cyberframework)
- 应用/API：[OWASP ASVS](https://owasp.org/www-project-application-security-verification-standard/)、[OWASP API Security Top 10](https://owasp.org/API-Security/)
- AI：[NIST AI RMF](https://www.nist.gov/itl/ai-risk-management-framework)、[OWASP Top 10 for LLM Applications](https://owasp.org/www-project-top-10-for-large-language-model-applications/)
