## 2026-06 Agent Security Refresh

AI / Agent 安全现在不能只归在“应用安全”下面。Agent 会调用工具、读写文件、连接 SaaS、执行技能、维护 memory；控制点必须覆盖 tool permission、MCP server trust、skills supply chain 和 runtime review gate。

## 控制设计原则

| Principle | 说明 |
|---|---|
| Least Tool Privilege | agent 只拿当前任务必需工具和最小参数权限 |
| Explicit Trust Boundary | MCP server、connector、skill、workspace、secret 都要有边界 |
| Human Review Gate | 高风险写入、删除、发布、外发、权限提升必须 review |
| Traceability | 每次工具调用、文件修改、审批和回滚都要可追踪 |
| Runtime Kill Switch | 长任务 agent 必须能暂停、取消、回滚或降级 |
| Supply Chain Review | skills、MCP server、插件和脚本要记录来源、版本、权限 |
