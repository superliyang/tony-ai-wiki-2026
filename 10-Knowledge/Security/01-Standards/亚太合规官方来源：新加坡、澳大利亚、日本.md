# 亚太合规官方来源：新加坡、澳大利亚、日本

## 定位

亚太地区不是一个统一合规体系。学习时不要说“APAC 合规”就结束，而要按国家、数据主体、数据类型和监管机构拆开。

本页先覆盖三个常见入口：

- 新加坡：PDPA / PDPC。
- 澳大利亚：Privacy Act / OAIC / Notifiable Data Breaches。
- 日本：APPI / PPC。

## 官方来源

### 新加坡

### 日本

- Personal Information Protection Commission：<https://www.ppc.go.jp/en/>
- APPI guidance：<https://www.ppc.go.jp/en/legal/>

## 新加坡 PDPA

### 关键问题

- 是否收集、使用、披露个人数据？
- 是否有 consent、notification、purpose limitation？
- 是否有 protection、retention limitation、accuracy 等义务？
- 是否触发 data breach notification？
- 是否涉及跨境传输和第三方处理？

### 技术与证据

- consent and notice records。
- data inventory。
- access control。
- retention and deletion schedule。
- breach assessment record。
- vendor processing record。

## 澳大利亚 Privacy Act / NDB

### 关键问题

- 是否适用 Privacy Act 和 Australian Privacy Principles？
- 是否发生 eligible data breach？
- 是否需要通知 OAIC 和受影响个人？
- 是否有合理安全措施保护个人信息？

### 技术与证据

- APP mapping。
- personal information inventory。
- breach assessment。
- notification decision record。
- security controls and audit logs。
- vendor and cross-border disclosure record。

## 日本 APPI

### 关键问题

- 是否处理 personal information 或 special care-required personal information？
- 是否向第三方提供？
- 是否涉及跨境提供？
- 是否发生个人数据泄露等通报场景？
- 是否能响应数据主体请求？

### 技术与证据

- data inventory。
- purpose notification / publication。
- consent and third-party transfer record。
- cross-border transfer assessment。
- breach record。
- access, correction, deletion request records。
