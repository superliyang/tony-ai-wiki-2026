# Security 案例复盘索引

> 案例层的目标不是记住新闻，而是训练安全判断：攻击从哪里来，控制哪里断了，检测为什么没看见，组织应该如何改。

- [[./Log4Shell 复盘|Log4Shell 复盘]]：开源组件漏洞、攻击面盘点、漏洞优先级、应急修复。
- [[./MOVEit Transfer 复盘|MOVEit Transfer 复盘]]：托管文件传输系统、零日利用、数据外泄、第三方风险。
- [[./xz Utils 后门复盘|xz Utils 后门复盘]]：开源维护者信任、供应链投毒、发布链路与异常发现。
- [[./Codecov Bash Uploader 复盘|Codecov Bash Uploader 复盘]]：CI/CD 脚本供应链、环境变量泄露、凭据轮换。
- [[./Okta Support System 复盘|Okta Support System 复盘]]：身份供应商、支持系统、客户上传文件、会话/令牌风险。

## 复盘方法

每个案例都按同一结构读：

1. 资产：被保护对象是什么？
2. 攻击面：入口在哪里？
3. 攻击链：攻击如何推进？
4. 控制缺口：哪类控制没生效？
5. 检测响应：什么时候发现，如何止血？
6. 组织教训：哪些默认机制要改？
7. 迁移练习：如果发生在我们系统里，应该怎么做？
