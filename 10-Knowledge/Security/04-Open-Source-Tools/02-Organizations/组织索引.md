# 安全开源组织索引

## 组织与社区集群

| 组织 / 社区 | 代表项目 | 观察重点 |
|---|---|---|
| ProjectDiscovery | nuclei、subfinder、httpx、naabu、katana | 攻击面工具链和模板生态 |
| OWASP | ZAP、Amass、WSTG、Juice Shop、Cheat Sheet Series | 应用安全标准、教学和测试 |
| Aqua Security | Trivy、kube-bench、tfsec | 云原生和供应链安全工具 |
| Anchore | Syft、Grype | SBOM 与漏洞扫描 |
| OpenSSF / Sigstore | Scorecard、cosign | 开源供应链健康度和签名 |
| HashiCorp / OpenBao | Vault、OpenBao | secret 和凭据治理 |
| Ory | Kratos、Hydra | 身份、OAuth2/OIDC |
| TheHive Project | TheHive、Cortex | 事件响应和分析器生态 |
| Zeek / OISF | Zeek、Suricata | 网络安全监控和 IDS |
| Protect AI / Promptfoo / garak | LLM Guard、promptfoo、garak | AI 安全评测和防护 |
| SpecterOps / GhostPack | BloodHound、Rubeus、Seatbelt | AD 攻击路径和身份安全 |

## 组织观察维度

- 是否持续维护核心项目。
- 是否形成工具链生态，而不是单点项目。
- 是否有规则、模板、插件或社区贡献机制。
- 是否适合个人实验、团队 CI/CD 或企业平台。
