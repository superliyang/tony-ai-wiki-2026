# Security Open Source Tools 专题总览

> 这里不是“安全工具收藏夹”，而是安全开源项目控制塔：按领域、组织、项目、模式、Watchlist 和 Intake 流程来管理。
> 目标是把 GitHub 热门开源安全工具变成可学习、可实验、可落地、可复盘的知识系统。

## 分层方式

- `01-Categories`：稳定工具类别，例如攻击面、AppSec、DevSecOps、云原生、SecOps、DFIR、AI 安全。
- `02-Organizations`：工具背后的组织和社区，例如 OWASP、ProjectDiscovery、Aqua、Anchore、TheHive、Ory、Protect AI。
- `03-Projects`：项目索引与少量旗舰项目卡，不把每个 repo 都写成深度分析。
- `04-Patterns`：从多个工具抽象出的工程模式，例如规则即代码、CLI 管线、证据化 DevSecOps。
- `06-Maps`：生态图和工具选型地图。
- `07-Templates`：项目卡模板。
- `08-Workflows`：GitHub Intake、刷新和评分流程。
- `09-Watchlist`：真正值得持续跟踪的项目。

## 当前原则

- 不按 star 排名学习，而按安全工作流学习。
- 不把红队/扫描工具当“炫技清单”，必须绑定授权边界和实验环境。
- 不把单个工具等同于能力，工具要接入 owner、流程、指标和证据。
- 不追求一次收完所有项目，先建立可持续 intake 机制。
