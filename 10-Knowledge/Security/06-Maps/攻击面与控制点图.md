# 攻击面与控制点图

## 总图

```mermaid
flowchart LR
  attacker["攻击者 / 滥用者"]

  subgraph surface["攻击面"]
    net["网络入口"]
    endpoint["终端/客户端"]
    app["应用/API"]
    identity["身份/密钥"]
    cloud["云/基础设施"]
    supply["供应链/第三方"]
    data["数据/隐私"]
    people["人员/流程"]
  end

  subgraph controls["控制点"]
    netc["分段、WAF、DDoS、ZTNA"]
    endc["EDR、MDM、App 加固"]
    appc["AuthZ、输入校验、限流、SAST/DAST"]
    iamc["MFA、PAM、Secret 管理、Access Review"]
    cloudc["CSPM、CNAPP、K8s Policy、IaC Scan"]
    supplyc["SCA、SBOM、签名、CI/CD 加固"]
    datac["分类、加密、脱敏、DLP、审计"]
    peoplec["培训、审批、职责分离、反钓鱼"]
  end

  subgraph detect["检测响应"]
    logs["日志与 Trace"]
    rules["检测规则 / 行为分析"]
    triage["告警分诊"]
    respond["隔离、止血、取证、恢复"]
  end

  attacker --> surface
  net --> netc
  endpoint --> endc
  app --> appc
  identity --> iamc
  cloud --> cloudc
  supply --> supplyc
  data --> datac
  people --> peoplec
  controls --> detect
  logs --> rules --> triage --> respond
  respond -. "复盘改进控制" .-> controls
```

## 用法

每遇到一个安全问题，按这个顺序问：

1. 它属于哪个攻击面
2. 已有哪些控制点
3. 哪个控制点缺失或失效
4. 有没有检测信号
5. 事件后如何复盘改进
