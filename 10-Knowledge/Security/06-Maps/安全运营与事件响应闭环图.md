# 安全运营与事件响应闭环图

## 总图

```mermaid
flowchart TB
  subgraph collect["1.采集"]
    idlog["身份日志"]
    netlog["网络日志"]
    applog["应用/API日志"]
    cloudlog["云与容器日志"]
    edrlog["终端告警"]
    datalog["数据访问日志"]
  end

  subgraph detect["2.检测"]
    normalize["标准化与关联"]
    rules["规则 / Sigma / Use Case"]
    behavior["行为分析"]
    intel["威胁情报"]
  end

  subgraph triage["3.分诊"]
    severity["分级"]
    owner["分派 Owner"]
    enrich["上下文补全"]
    falsepositive["误报处理"]
  end

  subgraph response["4.响应"]
    contain["止血与隔离"]
    evidence["取证与证据保全"]
    eradicate["根除与修复"]
    notify["通报与沟通"]
  end

  subgraph recover["5.恢复与改进"]
    restore["恢复服务"]
    postmortem["复盘"]
    regression["检测规则和控制回归"]
    backlog["安全 Backlog"]
  end

  collect --> detect --> triage --> response --> recover
  recover -. "新检测用例" .-> detect
  recover -. "新控制点" .-> response
  recover -. "治理与审计证据" .-> triage
```

## 关键指标

- MTTD：平均发现时间
- MTTR：平均恢复时间
- containment time：止血时间
- false positive rate：误报率
- recurrence rate：复发率
- playbook coverage：剧本覆盖率
