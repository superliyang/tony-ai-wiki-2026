# 企业安全架构图

## 总图

```mermaid
flowchart TB
  business["业务目标与风险<br/>收入、信任、连续性、合规"]

  subgraph governance["治理层"]
    policy["策略与标准"]
    risk["风险登记与例外"]
    audit["审计证据"]
    metrics["安全指标"]
  end

  subgraph platform["安全平台层"]
    iam["身份平台<br/>SSO、MFA、IAM、PAM"]
    data["数据保护<br/>分类、加密、DLP、保留"]
    appsec["安全开发平台<br/>SAST、SCA、DAST、Threat Modeling"]
    cloud["云安全平台<br/>CSPM、CNAPP、K8s、Runtime"]
    soc["安全运营平台<br/>SIEM、SOAR、EDR、Threat Intel"]
  end

  subgraph delivery["工程交付层"]
    sdlc["SDLC / DevSecOps"]
    cicd["CI/CD 与供应链"]
    release["Release Gate"]
    incident["Incident Feedback"]
  end

  subgraph assets["业务资产层"]
    users["用户与员工"]
    apps["应用与 API"]
    infra["云与基础设施"]
    datasets["数据与日志"]
    vendors["供应商与第三方"]
  end

  business --> governance
  governance --> platform
  platform --> delivery
  delivery --> assets
  assets -. "日志、风险、事故、审计证据" .-> governance
  assets -. "检测信号" .-> soc
  incident -. "复盘反哺" .-> sdlc

  classDef gov fill:#f5f5f5,stroke:#777,color:#222;
  classDef plat fill:#e8f2ff,stroke:#4b83c4,color:#10233d;
  classDef del fill:#e9f7ef,stroke:#5ba36e,color:#123d20;
  classDef asset fill:#fff4e6,stroke:#d99a3a,color:#4d2b00;
  class policy,risk,audit,metrics gov;
  class iam,data,appsec,cloud,soc plat;
  class sdlc,cicd,release,incident del;
  class users,apps,infra,datasets,vendors asset;
```

## 怎么读

- `治理层` 定义责任、风险、控制和证据
- `安全平台层` 把控制做成可复用能力
- `工程交付层` 把安全嵌进设计、构建、发布和复盘
- `业务资产层` 是安全真正服务的对象
