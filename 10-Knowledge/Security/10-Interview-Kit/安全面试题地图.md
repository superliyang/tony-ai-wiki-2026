# 安全面试题地图

> 这不是题库答案，而是题型地图。重点是知道面试官在考什么，以及应该从哪个角度组织回答。

## 基础风险思维

### 典型问题

- 漏洞、威胁、风险、控制有什么区别？
- 你如何判断一个安全问题的优先级？
- 什么是残余风险？什么时候可以风险接受？

### 回答要点

- 从资产和影响开始。
- 结合暴露面、可利用性、补偿控制。
- 风险接受必须有 owner、期限、证据和复查。

关联：[[../08-Playbooks/安全快速诊断 Playbook|安全快速诊断 Playbook]]

## 应用与 API 安全

### 典型问题

- 如何设计一个 API 的鉴权和授权？
- 如何发现和防止 IDOR？
- 系统上线前你会做哪些安全评审？
- WAF 能不能替代应用安全？

### 回答要点

- 认证证明“你是谁”，授权决定“你能访问什么”。
- 后端必须校验资源归属和租户边界。
- WAF 是补偿控制，不替代权限、输入校验和业务逻辑安全。

关联：[[../08-Playbooks/应用与 API 安全评审 Playbook|应用与 API 安全评审 Playbook]]

## 身份与零信任

### 典型问题

- MFA、SSO、PAM 分别解决什么？
- 为什么说身份是新边界？
- 零信任和 VPN 替代有什么区别？
- service account 应该怎么治理？

### 回答要点

- 零信任是持续访问决策，不是单个产品。
- 访问决策要结合身份、设备、上下文、资源敏感度和风险。
- 机器身份和长期凭据必须有 owner、过期、轮换和审计。

关联：[[../08-Playbooks/零信任落地 Playbook|零信任落地 Playbook]]

## 云原生安全

### 典型问题

- 云安全和传统数据中心安全有什么不同？
- Kubernetes 最容易出哪些安全问题？
- 如何防止存储桶或数据库公网暴露？
- IaC scanning 和 CSPM 分别解决什么？

### 回答要点

- 云边界由身份、配置、网络、工作负载和自动化共同组成。
- 重点看云 IAM、网络暴露、存储公开、K8s RBAC、secret、镜像和 runtime。
- IaC 是变更前控制，CSPM 是运行中姿态治理。

关联：[[../08-Playbooks/云原生安全基线 Playbook|云原生安全基线 Playbook]]

## 供应链安全

### 典型问题

- SCA、SBOM、artifact signing 分别解决什么？
- 如何防止 CI/CD secret 泄露？
- 如何评审第三方 SDK？
- xz Utils 或 Codecov 案例给了什么教训？

### 回答要点

- SBOM 是可见性，不是完整控制。
- 供应链要覆盖依赖、代码仓库、构建、artifact、发布和第三方。
- CI/CD secret 要最小权限、分环境、短期化、可轮换。

关联：[[../08-Playbooks/供应链安全评审 Playbook|供应链安全评审 Playbook]]

## 漏洞与攻击面

### 典型问题

- 为什么 CVSS 高不一定优先修？
- 如何建立漏洞管理流程？
- 如何处理公网暴露但业务不能马上下线的漏洞？
- 如何证明漏洞已修复？

### 回答要点

- 排序要结合暴露面、在野利用、资产重要性、数据影响和补偿控制。
- 风险接受必须有 owner、期限和补偿控制。
- 修复后要复扫、验证、查日志、补 release gate。

关联：[[../08-Playbooks/漏洞管理与攻击面管理 Playbook|漏洞管理与攻击面管理 Playbook]]

## SOC 与事件响应

### 典型问题

- SIEM、SOAR、EDR、XDR 有什么关系？
- 如何设计一条检测规则？
- 账号接管或云密钥泄露怎么响应？
- MTTD 和 MTTR 怎么提升？

### 回答要点

- 检测规则要从攻击场景出发。
- 告警必须有上下文、分诊步骤、owner 和响应动作。
- 响应流程是确认、分级、止血、取证、根除、恢复、通报、复盘。

关联：[[../08-Playbooks/SOC 检测工程 Playbook|SOC 检测工程 Playbook]]

## 数据安全与隐私

### 典型问题

- 加密、脱敏、匿名化、tokenization 有什么区别？
- 如何评审一个数据导出功能？
- 数据删除请求如何落地？
- 跨境数据传输要考虑什么？

### 回答要点

- 不收集不必要数据是最强控制。
- 数据安全要覆盖分类、访问、加密、脱敏、审计、保留删除和第三方共享。
- 合规问题要和地区、数据主体、数据类型、处理目的绑定。

关联：[[../08-Playbooks/数据安全与隐私评审 Playbook|数据安全与隐私评审 Playbook]]

## GRC 与架构

### 典型问题

- 如何把合规要求转成技术控制？
- 安全团队应该做 gatekeeper 还是 enabler？
- 企业安全建设第一年怎么排优先级？
- 如何向管理层汇报安全风险？

### 回答要点

- 合规证明控制存在，风险管理决定优先级。
- 长期目标是把安全平台化、模板化、默认化。
- 管理层关心风险、趋势、影响、取舍和资源。

关联：[[../08-Playbooks/企业安全架构落地 Playbook|企业安全架构落地 Playbook]]

## 行为与案例题

### 典型问题

- 讲一次你处理安全问题的经历。
- 遇到业务要求绕过安全 gate，你怎么办？
- 如果你发现一个高风险问题但团队不愿修，怎么推动？
- 复盘一个公开安全事件，你学到了什么？

### 回答结构

```text
背景 -> 风险判断 -> 沟通对象 -> 解决方案 -> 验证结果 -> 反思
```
