# 安全工具平台分类索引

> 这一层回答：真实组织里安全能力到底靠哪些工具、平台、流程和集成落地。
> 工具不是能力本身；能力 = `问题定义 + 资产数据 + 控制点 + 工具平台 + owner + 流程 + 指标 + 证据`。

- [[./身份、密钥与访问平台|身份、密钥与访问平台]]：IAM、SSO、MFA、PAM、Secrets、KMS、证书与机器身份。

- [[./云原生安全与 CNAPP 平台|云原生安全与 CNAPP 平台]]：CSPM、CWPP、KSPM、CIEM、IaC、容器与运行时。
- [[./SOC、SIEM、SOAR 与 XDR 平台|SOC、SIEM、SOAR 与 XDR 平台]]：日志、检测、分诊、响应、编排、终端与威胁情报。

## 横向平台能力

| 能力 | 解决的问题 | 常见落地形态 |
|---|---|---|
| 资产与暴露面 | 不知道保护什么、哪里暴露 | CMDB、EASM、ASM、云资产同步、标签 |
| 身份与权限 | 谁能访问什么、是否过权 | IdP、IAM、PAM、CIEM、JIT、MFA |
| 密钥与凭据 | secret 泄露、难轮换、难审计 | Vault、KMS、Secret Manager、证书自动化 |
| 安全开发 | 漏洞进了代码和依赖 | SAST、SCA、DAST、IaC Scan、Code Review |
| 云原生安全 | 云配置、K8s、容器、运行时风险 | CNAPP、CSPM、CWPP、KSPM、Falco 类运行时 |
| 检测与响应 | 看不见攻击、告警无人处理 | SIEM、SOAR、EDR、XDR、NDR、MDR |
| 数据与隐私 | 不知道敏感数据在哪、谁访问 | DSPM、DLP、分类分级、数据血缘、隐私工单 |
| 治理与审计 | 控制无法证明、风险无人闭环 | GRC、IRM、持续合规、证据自动采集 |

## 选型原则

- 先问问题，不先问产品：要解决 `账号滥用`、`公网暴露`、`漏洞堆积`、`日志不可见`、`审计证据` 还是 `隐私合规`。
- 先补数据源，再补平台：没有资产、身份、日志、代码、云账号接入，平台只能给空结论。
- 先设计流程，再开告警：没有 owner、SLA、分诊和例外机制，工具会变成噪声机器。
- 先小闭环，再大平台：优先在一个业务线跑通 `发现 -> 分诊 -> 修复 -> 验证 -> 证据`。
- 先集成已有系统，再采购新系统：SSO、工单、CMDB、CI/CD、云账号、日志平台往往决定落地成败。
