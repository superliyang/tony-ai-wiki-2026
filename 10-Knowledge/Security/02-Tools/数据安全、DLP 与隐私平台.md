# 数据安全、DLP 与隐私平台

> 数据安全平台的第一性问题是：敏感数据在哪里、怎么流动、谁能访问、是否最小化、如何证明合规。

## 能力边界

| 子能力 | 解决的问题 | 常见输出 |
|---|---|---|
| 数据发现 | 不知道敏感数据在哪 | 数据资产清单、敏感字段 |
| 分类分级 | 不知道重要性和监管属性 | PII、PCI、PHI、商业秘密 |
| DSPM | 云和数据平台里的数据风险 | 公开数据、过权访问、影子数据 |
| DLP | 数据外发与泄露 | 阻断、提示、审计、例外 |
| Masking / Tokenization | 展示和分析最小化 | 脱敏视图、token 替代 |
| Privacy Ops | 数据主体请求和同意管理 | DSAR、删除、更正、同意记录 |
| Data Retention | 过度保留和删除困难 | 保留策略、删除任务、证据 |

## 落地闭环

1. 数据地图：系统、表、字段、数据主体、地区、owner。
2. 分类分级：敏感性、监管属性、业务重要性。
3. 访问治理：最小权限、审批、审计、异常访问检测。
4. 数据流治理：跨境、第三方共享、导出、备份、分析平台。
5. 数据保护：加密、脱敏、DLP、tokenization、删除。
6. 合规证据：处理活动、访问日志、请求处理、事故通报。

## 选型检查点

- 是否能覆盖数据库、对象存储、SaaS、数据仓库、日志和端点？
- 分类规则是否可解释、可调优、可审计？
- DLP 是否支持提示、阻断、例外和业务上下文？
- 是否能和隐私请求、数据保留、跨境评估、GRC 证据联动？
- 是否能接入 SIEM 发现异常访问和批量导出？

## 关键指标

- 敏感数据发现覆盖率
- 未分类数据资产数量
- 高敏数据过权访问数量
- DLP 有效阻断率与误报率
- 数据主体请求完成时间
- 数据保留/删除策略覆盖率

## 典型陷阱

- DLP 一上来就强阻断，业务绕过或误伤严重。
- 分类分级只做制度，不落到字段、表、日志和权限。
- 隐私合规只由法务处理，没有数据地图和技术证据。
- 数据安全和 SOC 割裂，异常访问无人响应。
