# 跨境电商卡支付判断清单

## 这页解决什么问题

跨境电商不只是“卡能不能扣上”，而是“卡支付能不能和本地支付方式、履约、退款、拒付一起形成可盈利系统”。这页帮你把卡支付深水区映射到电商场景里。

## 最优先看的 6 个判断点

1. 不同市场的卡支付是否仍然值得作为主方式，而不是被本地方式替代
2. 授权后 capture、reversal 和退款体验是否合理
3. 是否因为 `MCC`、descriptor、主体和 `MID` 策略不佳而影响授权信任
4. 是否把高风险订单的认证和风控分层做细
5. 是否把履约、退款、拒付和 pre-dispute 串成一条线
6. 当前商户模式和通道组合是否适合继续扩市场

## 逐项判断

### 1. Authorization / Capture / Reversal

- 订单是即时 capture 还是先 auth 后 capture
- 发货取消时有没有及时 reversal
- 用户是否因为预授权/释放不及时而投诉

### 2. MCC / Descriptor / MID / 主体

- 不同国家和业务线是否需要拆 `MID`
- descriptor 是否足够本地化、用户可识别
- `MCC` 是否真实匹配业务，避免发卡行误判

### 3. 3DS / 风控 / 电商订单质量

- 高客单价、高风险地址、异常设备是否做了更细认证
- 是否把 3DS 策略和物流、订单欺诈一起看
- 是否分国家看 challenge 对最终净成功率的影响

### 4. Pre-dispute / 退款治理

- 商品不符、履约慢、退款慢是否已转化为拒付热点
- 哪些订单值得优先自动退款止损
- 客服和支付运营是否共享争议原因分类

### 5. 模式与通道组合

- 当前用 `PSP` 就够了，还是需要多 acquirer / orchestration
- 市场扩张时，主体、`MID`、支付方式和本地收单是否一起演进
- 是否为了单笔费率牺牲了长期本地化与成功率

### 6. 本地方式与卡的平衡

- 市场主流是不是应该优先推本地方式
- 卡支付是否更适合作为补充、国际卡兜底或高客单方式
- 卡支付优化是否已到边际收益递减，应该把精力转到本地方式覆盖

## 最值得盯的指标

- 市场维度卡支付成功率
- auth 到 capture 转化率
- reversal 及时率
- 履约相关争议占比
- 卡支付净毛利
- 本地方式 vs 卡支付的净收益比较
