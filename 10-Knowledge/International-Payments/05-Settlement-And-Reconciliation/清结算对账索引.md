## 2026-06 Refresh Lens

清结算对账不能只看“钱有没有到”，要看 5 个层次：

| 层次 | 核心问题 | 常见风险 |
|---|---|---|
| Transaction Ledger | 订单、授权、捕获、退款、拒付、调账状态是否一致 | 状态机不完整、幂等失败 |
| PSP / Acquirer Report | PSP 报表和 acquirer settlement file 是否能还原资金流 | 字段缺失、时区、手续费口径不同 |
| Scheme Rules | 卡组织规则、原因码、费用、结算窗口是否影响账务 | 规则版本变化、reason code 映射错误 |
| Merchant Data | descriptor、MCC、merchant name、location 是否一致 | 用户认不出扣款、发卡行误判 |
| Treasury / FX | 多币种、汇率、结算周期、资金池和对冲 | FX 损益、资金延迟、账实不符 |

## 对账检查清单

- 是否有统一交易状态机，覆盖 auth、capture、void、refund、chargeback、representment。
- 是否能把 PSP transaction id、acquirer reference、scheme reference、内部 order id 对齐。
- 是否区分 gross amount、net amount、fee、tax、FX、reserve、adjustment。
- 是否按 settlement date、transaction date、report date 三个时间维度看差异。
- 是否记录卡组织规则、PSP 报表字段和 merchant data 的版本变化。
- 是否把拒付、退款和调账回写到 BI 与财务报表。
