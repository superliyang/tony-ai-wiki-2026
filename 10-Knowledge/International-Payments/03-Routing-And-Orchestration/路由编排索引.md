## 2026-06 Refresh Lens

支付路由和编排不能只看“哪条通道成功率高”。2026-04 Visa rules 和 merchant data 标准提醒我们：成功率、争议、风控、数据质量、合规规则是同一张图。

| Layer | 关注点 | 路由影响 |
|---|---|---|
| Scheme Rules | Visa / Mastercard / local scheme 规则版本 | 决定 3DS、dispute、merchant data、retry 约束 |
| Merchant Data | merchant name、MCC、wallet/operator、descriptor | 影响 issuer recognition、fraud model、dispute handling |
| Authentication | 3DS / Visa Secure / issuer challenge | 影响 friction、approval、liability shift |
| PSP / Acquirer Fit | 地区、币种、MCC、risk appetite、tokenization | 决定 routing candidate pool |
| Failure Taxonomy | soft decline、hard decline、issuer unavailable、risk decline | 决定 retry / fallback / user action |
| Compliance Baseline | PCI DSS、data handling、storage、logging | 限制可观测数据和重试策略 |

## 路由决策问题

| 问题 | 应记录的数据 |
|---|---|
| 这笔交易为什么失败？ | decline code、issuer response、3DS result、PSP/acquirer、region、MCC |
| 是否可以 retry？ | hard/soft decline 分类、scheme retry rule、issuer/network availability |
| 是否应该换路由？ | acquirer performance、成本、fraud rate、chargeback、local method fit |
| 是否应该触发 3DS？ | risk score、issuer behavior、liability shift、user friction |
| 是否是数据质量问题？ | merchant descriptor、MCC、billing address、token、device / browser data |
