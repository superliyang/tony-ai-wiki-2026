# 国际支付总览图

```mermaid
flowchart TD
    A[国际支付总览] --> B[业务发起]
    A --> C[报文网络]
    A --> D[账户关系]
    A --> E[清算]
    A --> F[结算]
    A --> G[合规与操作风险]

    B --> B1[付款人/收款人/商户]
    C --> C1[SWIFT]
    D --> D1[Nostro/Vostro]
    E --> E1[应收应付与头寸确认]
    F --> F1[最终资金变动]
    G --> G1[AML/KYC/制裁/审查]

    C1 --> D1
    D1 --> E1
    E1 --> F1
    G1 -.影响整个链路.-> C1
    G1 -.影响整个链路.-> E1
    G1 -.影响整个链路.-> F1
```

## 怎么看这张图

- 如果你想理解“跨境支付为什么复杂”，先看六层结构
- 如果你想理解“为什么报文发出不等于到账”，重点看报文层、账户层、清算层、结算层的差别
- 如果你想把业务问题映射到底层原因，就看合规与操作风险如何贯穿整条链路
