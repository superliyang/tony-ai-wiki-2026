## 案例层怎么用

案例层不只是收集“哪个公司用了什么”，更重要的是回答：

- 它们到底在解决哪类平台问题
- 开发者入口、catalog、template、docs 和治理是怎么被连起来的
- 哪些做法说明平台工程已经进入产品化阶段
- AI 时代的 workload 是如何被重新组织成训练、推理和 distributed platform 三类能力的
- service mesh 和 supply-chain security 是如何被拉进平台控制面的
- runtime、serverless 和 storage 是怎样在案例层体现成不同平台边界的
- AI 平台团队与 AI serving 平台，是如何在案例层体现成 adoption path 的不同阶段
- day-2 operations 是如何在案例层体现成 mesh operations 与 runtime security 两条主线的
- AI-era enterprise adoption 是如何从实验边界走到平台化运营边界的
