# 云原生安全

## 一句话理解

云原生安全不是给现有系统“外挂”一层安全，而是随着 `containers + Kubernetes + CI/CD + GitOps + APIs` 一起重构安全边界、身份模型和治理方式。

## 为什么它和传统安全不一样

云原生环境有几个变化：

- workload 更动态
- 基础设施更声明式
- 交付链更自动化
- 多租户和权限边界更复杂
- 软件供应链问题直接影响运行时风险

所以安全必须更早进入构建、交付和平台治理链路。

## 这条主线通常包括什么

- cluster security
- workload identity
- policy enforcement
- image / artifact trust
- runtime security
- supply chain security

## 学习重点

- 为什么 admission control 和 policy-as-code 很重要
- 为什么 cloud native security 不能只盯 runtime
- 平台工程、GitOps 和安全治理为什么会越来越耦合
