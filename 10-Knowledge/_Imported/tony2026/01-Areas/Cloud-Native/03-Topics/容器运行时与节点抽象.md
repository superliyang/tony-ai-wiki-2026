# 容器运行时与节点抽象

## 一句话理解

Kubernetes 并不直接运行容器。真正负责镜像拉取、容器生命周期和底层执行的是 `container runtime`。理解这一层，才能看清 Kubernetes 为什么只是控制平面，而不是全部运行时本体。

## 为什么这层重要

很多人学 Kubernetes 时只看到 Pod、Deployment、Service，却忽略节点上真正干活的是 runtime。运行时层决定了：

- 容器如何被创建和销毁
- 镜像如何被拉取和缓存
- cgroup / namespace 如何被配置
- kubelet 如何和底层执行环境协作

## 在 Kubernetes 里的关键点

- 每个 node 都需要一个兼容 `CRI` 的 runtime
- kubelet 和 container runtime 的 cgroup driver 需要匹配
- runtime 决定的是执行与生命周期，不是高层编排策略

## 学习重点

- `containerd` 为什么是云原生世界的重要 runtime 节点
- kubelet、CRI、runtime 三者如何配合
- 为什么“懂 Kubernetes”如果不懂 runtime，很多排障和性能问题会悬空
