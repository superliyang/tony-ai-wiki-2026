# 可观测性

## 一句话理解

`Observability` 关注的不是“有没有监控”，而是系统能否通过外部输出来帮助你理解内部状态并采取行动。

## 为什么它在云原生里更重要

云原生系统实例多、变化快、链路长，如果没有良好的 telemetry，你几乎无法理解系统为什么变慢、失败或不稳定。

## 这条主线通常包括什么

- `Metrics`：用于趋势、容量、告警
- `Logs`：用于事件与上下文
- `Traces`：用于跨服务链路理解
- `Telemetry pipeline`：用于采集、处理、导出和分析

## 学习重点

- monitoring 和 observability 的边界
- Prometheus 与 OpenTelemetry 分别站在哪一层
- 为什么 observability 最终会影响研发效率与系统稳定性
