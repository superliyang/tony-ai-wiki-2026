# 什么是云原生

## 一句话理解

云原生不是某个单一产品，而是一套围绕 `弹性基础设施 + 容器化工作负载 + 声明式系统 + 自动化运维 + 可观测性 + 平台化交付` 逐步发展出来的 operating model。

## 为什么它不是“上云”

“上云”更多是在说资源放在哪里；云原生更关心系统如何被构建、部署、运行和演进。

云原生强调的是：
- workload 可迁移
- 环境可复现
- 系统可自动收敛
- 交付过程可审计
- 运行状态可观测

## 云原生的几根主线

- `Containers`：标准化运行单元
- `Kubernetes`：声明式编排与控制平面
- `GitOps`：把交付和变更管理纳入 source of truth
- `Observability`：让复杂分布式系统可理解
- `Platform Engineering`：让开发者在更高层抽象上交付软件

## 为什么这门课值得学

学云原生，不只是为了会用 Kubernetes。更重要的是理解现代基础设施和软件交付为什么越来越像：

- API 驱动系统
- 控制循环系统
- 平台化系统
- 运营与开发更深耦合的系统
