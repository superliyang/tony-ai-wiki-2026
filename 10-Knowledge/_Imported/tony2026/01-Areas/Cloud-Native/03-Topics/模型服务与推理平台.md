# 模型服务与推理平台

## 一句话理解

模型服务与推理平台关注的是：如何把模型以可扩缩、可治理、可观测的方式接到在线服务层，而不只是“起一个容器 API”。

## 为什么它在云原生里变成独立问题

传统应用服务和模型服务不完全一样。模型服务往往还要处理：

- 模型装载与生命周期
- 多模型、多版本管理
- 在线推理的弹性与延迟
- autoscaling 与冷启动
- GPU / CPU 异构资源选择

## 这条主线通常包括什么

- model serving abstractions
- inference endpoints
- autoscaling for inference
- canary / rollout for model versions
- request routing and model runtime management

## 学习重点

- KServe 这类平台为什么存在，而不是直接用通用 Deployment
- 模型服务在平台上更像“专门的 workload type”
- 推理平台会和 AI-Engineering 的 serving、inference optimization 强关联
