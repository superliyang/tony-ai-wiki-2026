# 运行时安全

## 一句话理解

运行时安全回答的问题不是“镜像是不是安全”，而是：当 workload 已经跑起来以后，如何持续发现 drift、异常行为、权限越界和运行时攻击迹象。

## 为什么它不能被供应链安全替代

供应链安全关注的是：

- build
- sign
- verify
- admit

而运行时安全关注的是：

- 进程行为是否异常
- workload 是否 drift
- 权限是否越界
- seccomp / capability / privilege 边界是否真正生效
- 集群里是否已经发生横向移动或恶意执行

所以这两者是一前一后、互补而不是替代关系。

## 运行时安全通常包括什么

- admission-time prevention
- runtime drift detection
- process / syscall monitoring
- seccomp / capability hardening
- suspicious network behavior
- incident response and containment

## 你应该从中看到什么

- runtime security 是 day-2 operations 的核心组成，而不是单独的安全附加项
- `RuntimeDefault seccomp` 这类默认约束，会逐渐成为平台默认基线
- 真实组织更看重的是“如何在不把开发者卡死的前提下，把 runtime security 变成默认能力”
