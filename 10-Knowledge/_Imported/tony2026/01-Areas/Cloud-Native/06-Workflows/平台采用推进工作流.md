# 平台采用推进工作流

## 简介

`平台采用推进工作流` 关注的是：平台团队如何从“能力已经存在”推进到“开发团队真的开始使用，而且持续使用”。

## 输入

- 平台 roadmap
- 已具备的 portal、catalog、templates、golden path
- adoption 阻力和 friction 信号
- 高价值候选团队或 use case

## 步骤

1. 识别最值得先推进 adoption 的团队和场景
2. 明确他们当前最痛的交付路径和重复劳动
3. 用模板、catalog、docs、default workflow 形成更顺滑的默认路径
4. 为高风险步骤补 approval 和 guardrails
5. 跟踪 adoption、阻塞点和例外请求，迭代平台入口与标准

## 输出

- 更清晰的 adoption path
- pilot 团队与推广顺序
- friction backlog
- 平台治理与 adoption 的联动节奏

## 评估

- adoption 是否来自真实场景而不是一次性宣导
- 默认路径是否显著优于手工路径
- 平台改动是否减少 support 和 ticket 负担
