# SuperClaude Framework

## 它解决什么问题

`SuperClaude Framework` 通过 commands、agents、modes、MCP servers 和方法论，把 Claude Code 包装成更结构化的开发平台。

## 为什么现在值得关注

它是社区里很有代表性的“Claude Code 配置框架”：不只是一个插件，而是把开发生命周期拆成 research、brainstorm、implement、test、project management 等结构化命令。

## 它在 stack 的哪一层

- 属于 `configuration framework / workflow framework`
- 更像一套 Claude Code 上层方法论和工具包

## 核心组件与架构

- `/sc:*` slash commands
- specialized agents
- behavioral modes
- optional MCP integrations
- installer / doctor / project docs

## 最适合它的场景

- 想快速体验结构化 Claude Code 工作流
- 想学习 commands + agents + MCP 如何组合
- 想参考社区如何组织一套开发方法论

## 和相邻项目怎么区分

- 和 [[claude-code-templates]]：SuperClaude 更像一套完整框架；templates 更像组件市场和安装器
- 和 [[Compound Engineering Plugin]]：SuperClaude 覆盖面更广；Compound 更强调 planning / review / compound learning 闭环

## 风险与边界

- 覆盖面大，容易“一装很多但没想清楚”
- 应先在个人 sandbox repo 试用，再决定是否放进团队项目
