# wshobson-agents

## 它解决什么问题

`wshobson/agents` 提供大量 Claude Code plugins、specialized agents、skills、workflow orchestrators 和 commands，面向多 agent 自动化与软件开发任务分工。

## 为什么现在值得关注

它是社区里“插件化 subagent/skill 生态”的代表样本，适合学习如何把开发角色拆成可安装、可组合、可渐进加载的能力包。

## 它在 stack 的哪一层

- 属于 `subagent / plugin marketplace`
- 位于 Claude Code 的专家分工层和方法层

## 核心组件与架构

- focused plugins
- specialized agents
- agent skills
- workflow orchestrators
- utility commands

## 最适合它的场景

- 按语言、框架、基础设施、测试、安全等安装专门 agent
- 学习插件粒度如何控制 token 和上下文负担
- 设计团队角色型 subagents

## 和相邻项目怎么区分

- 和 [[awesome-claude-code-subagents]]：二者都偏 subagent；wshobson 更强调插件化 marketplace 与工作流编排
- 和 [[SuperClaude Framework]]：wshobson 更偏可组合能力包；SuperClaude 更偏统一框架

## 风险与边界

- 能力很多，不适合全量安装
- 应按项目只安装必要插件，并检查 agents 是否会触发不希望的工具使用
