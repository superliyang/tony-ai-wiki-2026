# Agent ROI and Value Capture

## 这个主题是什么

`Agent ROI and Value Capture` 关注 agent 落地后，组织到底获得了什么价值，以及这些价值如何被衡量、证明和放大。

## 为什么重要

- 很多 agent 项目不是技术上失败，而是业务上无法证明值不值得继续投
- 如果没有 ROI 视角，团队很容易停留在“看起来很酷、偶尔有用”的阶段
- 真正能持续扩张的 agent 系统，往往都能讲清楚：节省了什么、提升了什么、减少了什么风险

## 价值通常来自哪里

- 更少的人类重复劳动
- 更快的任务完成时间
- 更高的信息处理吞吐
- 更低的漏单、漏处理和响应延迟
- 更稳定的流程执行与知识复用

## 不要只看省了多少人力

更完整的价值维度通常包括：

- productivity gain：单位时间内完成更多工作
- quality gain：结果更稳定、更一致、更可审计
- cycle-time reduction：流程周转更快
- coverage expansion：以前做不到的跟踪、复查、整理现在能做了
- risk reduction：由于更早发现问题、更规范执行而减少风险

## 常见误区

- 把一次 demo 的效率提升当成长期 ROI
- 只看模型成本，不看整体流程成本
- 只看节省时间，不看新增的人类审查负担
- 把“能做”误当成“值得做”

## 推荐衡量方式

- 先为场景定义明确的基准线
- 同时跟踪：完成率、耗时、人工接管率、错误率、单位成本
- 优先选择高频、可验证、可量化的任务做 ROI 证明
- 用阶段性目标来判断 agent 是否进入下一层自动化

## 适合先跑 ROI 的场景

- 研究摘要与信息整理
- 日报、周报、异常摘要
- 代码检索、测试建议、review 辅助
- 企业内部知识检索与流程导航
