# 最近半年最值得重投入学习的 AI 主线（截至 2026-04-07）

> 如果只允许我从最近半年出现的 AI 知识里，挑出少数几条真正值得花大精力、大时间去学深、学透、长期掌握的主线，我不会平均分配时间。我会非常偏心，只押那些已经明确改变模型路线、系统路线、产品路线和组织工作方式的东西。

## 先说结论

最近半年，最值得重投入的不是“所有新模型”，而是下面 6 条主线：

1. `Harness / Agent Workbench Engineering`
2. `Reasoning + Coding + Computer Use 的统一工作模型`
3. `Eval Harness、Regression、Release Gate 与可靠性治理`
4. `Voice / Realtime Agents`
5. `OCR / Document AI 作为企业工作流入口`
6. `Open Models + Local-First + Sovereign / Edge Deployment`

如果一定还要再压缩，我会把时间优先级排成：

1. `Harness / Agent Workbench Engineering`
2. `Reasoning + Coding + Computer Use`
3. `Eval / Release Gate / Reliability`
4. `Voice / Realtime`
5. `OCR / Document AI`
6. `Open Models / Local-First`

## 为什么我会这样排

因为最近半年真正发生的事情，不是“模型更强一点”，而是：

- frontier 公司都在把模型装进更完整的工作台
- reasoning 不再是一个 benchmark 点，而是和 coding、tool use、computer use 融在一起
- 企业真正愿意买单的入口，越来越清楚地落在 voice、document、agent workflow、release safety 这些地方
- open models 没有退场，反而在 local-first、edge、sovereign 和开发者生态上重新变得更关键

所以如果你想学的是“长期有复利的 AI 能力”，而不是“看完一波发布会就过期的热点”，就必须抓这些主线。

## 一个要长期记住的方法原则

知识图谱里的边边角角不能放过，因为很多真正的判断，最后都来自那些看似次要的上下文、失败案例、系统边界和历史分歧。

但资源分配一定要有优先级。

更准确地说，应该这样投入：

- `主干主线` 要重投入，花大块时间学深、学透、反复回顾
- `边角知识` 要持续扫过，保持感知，不让图谱失真
- 一旦某个边角开始影响模型路线、系统路线、产品路线或组织工作方式，就把它升级进主干

所以真正的原则不是“只学重点”，也不是“平均地学所有东西”，而是：

`全景不能漏，投入必须分层，优先级必须明确。`

## 1. Harness / Agent Workbench Engineering

### 为什么它最值得重投入

这是我会排第一的主线。

因为最近半年最深的变化，不是 prompt，而是工作台：

- OpenAI 在讲 `Codex harness`、`App Server`、`Harness engineering`
- Anthropic 在讲 `Claude Agent SDK`、`Claude Code`、`Xcode`
- Google 在讲 `Gemini CLI`、`ADK`、`Antigravity`
- xAI 在讲 `Agent Tools API`

这说明决定上限的，越来越不是“模型本身会不会说”，而是：

- agent 在什么环境里工作
- 它能碰哪些动作面
- 它怎么接工具、审批、回放、自动化和治理

### 你真正要学的不是词，而是这 6 层

1. `task environment`
2. `action surfaces`
3. `extension surfaces`
4. `control plane`
5. `automation plane`
6. `feedback / governance`

## 2. Reasoning + Coding + Computer Use 的统一工作模型

### 为什么它值得重投入

最近半年，最重要的模型变化不是“推理能力更强”这句废话，而是：

- reasoning 被做成 specialized mode
- coding 被做成独立模型 lane
- computer use 被做成通用工作能力

也就是说，现在真正该学的是：

- 什么样的模型适合长程工作
- 什么样的模型适合 coding workbench
- 什么样的模型适合 browser / desktop / tool-dense execution

## 3. Eval Harness、Regression、Release Gate 与可靠性治理

### 为什么它值得重投入

这是很多人最容易低估、但我会明确排到前三的主线。

因为最近半年模型和 agent 工作台都在高速迭代，如果你不懂：

- eval harness
- regression suite
- online evals
- release gate
- rollback

那你就只能看发布会、看排行榜、看 demo，没法形成自己的判断系统。

### 这条线真正决定什么

- 你能不能判断系统是不是变好了
- 你能不能发现“看起来更聪明，其实更脆弱”
- 你能不能把 agent 从 demo 推到 production

## 4. Voice / Realtime Agents

### 为什么它值得重投入

这条线我会排第四，不是因为它不重要，而是因为它现在很真，但还不是所有团队都必须立刻投入。

它值得学深，是因为它已经从 feature 变成 lane：

- OpenAI 把 realtime voice 做成 GA API
- Google 把 live audio 做成单独模型路线
- 语音系统开始明确接到 tool use、MCP、SIP、session management 和 safety

### 真正该学的不是语音本身，而是

- 低延迟系统约束
- turn-taking / interruption
- live session orchestration
- voice agent 与工具、审批、合规的关系

## 5. OCR / Document AI 作为企业工作流入口

### 为什么它值得重投入

如果你关心“AI 如何真的进企业”，这条线比很多人想的更重要。

因为真实组织的大量工作不是聊天，而是：

- PDF
- 合同
- 表格
- 发票
- policy / archive
- knowledge ingestion

Document AI 是最近半年里最像“价值入口”的路线之一。

### 真正该学的不是 OCR，而是

- 结构保真
- document-to-markdown/json
- extraction -> workflow trigger
- document AI 如何成为 agent 的前置层

## 6. Open Models + Local-First + Sovereign / Edge Deployment

### 为什么它值得重投入

这条线我会排第六，但它不是“可选项”，而是非常明确的战略分支。

如果你的问题是：

- 我想可控部署
- 我想本地跑
- 我想 device / edge
- 我想 sovereign AI
- 我想不完全依赖闭源 API

那这条线就必须学。

最近半年最该盯的不是泛泛的 open-source 口号，而是：

- [[../03-Models/Gemma 4|Gemma 4]]
- [[../03-Models/DeepSeek-R1|DeepSeek-R1]]
- [[../03-Models/Qwen 系列|Qwen 系列]]
- [[../06-Topics/Open-Weight Models|Open-Weight Models]]

### 真正该学的是这些判断

- open model 在哪些工作负载上会越来越有优势
- local-first 为什么和 coding assistant、document workflows、device AI 绑定
- sovereign / enterprise deployment 为什么会继续强化这条线

## 哪些东西我反而不会花太多时间

如果时间有限，我不会重投入在这些东西上：

- 零散的 prompt 技巧合集
- 每周都变的榜单和 arena 截图
- 所有次级模型 tier 的逐条追更
- 只看 demo、不看 eval 和 operating boundary 的“产品惊艳感”
- 泛泛的“AI 会改变世界”式趋势口号

这些东西可以看，但不值得你把最好的注意力放进去。
