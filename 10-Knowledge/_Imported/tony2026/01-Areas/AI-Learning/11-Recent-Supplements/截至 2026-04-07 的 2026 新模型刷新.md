# 截至 2026-04-07 的 2026 新模型刷新

> 这页解决的是一个很实际的问题：如果你在 `2026-04-07` 回头看 AI 模型格局，哪些新模型已经足够重要，值得直接进这套知识库的主线。

## 先说边界

- 这页不是 `2026` 全年总结，而是截至 `2026-04-07` 的阶段性刷新。
- 这页不追所有 tier 和每一次小版本迭代，只抓真正改变模型路线、系统路线或产品路线的节点。
- 这轮最值得记住的，不只是“模型又变强了”，而是各家公司把 `reasoning / coding / computer use / live audio / open models` 这些能力越来越清楚地拆成了独立路线。

## 这轮刷新后，最该建立的 4 个判断

### 1. frontier 旗舰不再只是“更大的通用模型”

- [[../03-Models/GPT-5-4|GPT-5.4]]
- [[../03-Models/Claude Opus 4-6|Claude Opus 4.6]]
- [[../03-Models/Gemini 3 Deep Think|Gemini 3 Deep Think]]
- [[../03-Models/Gemini 3-1 Pro|Gemini 3.1 Pro]]

这些模型共同说明：旗舰模型正在从“回答更聪明”走向“更适合长程、工具密集、需要规划和验证的真实工作”。

### 2. coding 已经从应用场景，变成模型分层本身

- [[../03-Models/GPT-5-3-Codex|GPT-5.3-Codex]]
- `GPT-5.3-Codex-Spark`
- [[../03-Models/Claude Sonnet 4-6|Claude Sonnet 4.6]]

也就是说，coding 不再只是“通用模型顺便很会写代码”，而是形成了：

- 长任务 coding flagship
- ultra-low-latency coding tier
- 高性价比 coding workhorse

## 3. 实时语音已经变成单独的模型路线

- `Gemini 3.1 Flash Live`
- [[../06-Topics/Voice、Realtime 与语音工作流|Voice、Realtime 与语音工作流]]
- [[../09-Systems/OpenAI Realtime API|OpenAI Realtime API]]

这说明 `voice / realtime` 不再只是 multimodal 的一个 feature，而是一个有自己延迟约束、任务结构和产品面要求的 lane。

## 4. open model 没有退场，反而在 2026 重新变得更战略

- [[../03-Models/Gemma 4|Gemma 4]]
- [[../06-Topics/Open-Weight Models|Open-Weight Models]]

这条线最重要的不是“追上最强闭源”本身，而是：

- agentic workflow
- local-first deployment
- device / edge
- sovereign AI
- open ecosystem compounding
