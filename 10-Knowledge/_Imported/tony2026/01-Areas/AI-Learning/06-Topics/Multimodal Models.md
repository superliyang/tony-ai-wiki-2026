# Multimodal Models

## 这个主题是什么

`Multimodal Models` 关注模型如何同时处理文本、图像、音频、视频等多种模态。

## 为什么重要

- 现实世界的信息本来就是多模态的，不只存在于文本里
- 多模态能力让模型从“文字系统”走向“通用交互系统”
- 最近两年，多模态不再只是“能看图”，而是更明显地走向 `voice / realtime`、`computer use`、`document AI`、`OCR` 和工作流自动化

## 最近两年这条线的变化

在最近两年里，多模态路线有四个很明显的收敛：

1. `从图文能力走向 voice / realtime multimodality`
- [[GPT-4o System Card]]
- [[Moshi：a speech-text foundation model for real-time dialogue]]
- [[Qwen2-5-Omni Technical Report|Qwen2.5-Omni Technical Report]]
- [[OpenAI Realtime API]]

2. `从图文能力走向 agentic multimodality`
- [[Google DeepMind]] 的 `Gemini 3` 把 multimodal、agentic coding 和 CLI / Antigravity 放到同一叙事里
- [[Anthropic]] 的 computer use 路线把视觉理解推进到操作界面与执行任务

3. `从多模态能力走向 OCR / document AI`
- [[Mistral AI]] 的 `Mistral OCR 3`
- [[Zhipu AI]] 的 `GLM-4.6V`
- [[Baidu]] 的原生全模态统一建模

4. `从模型能力走向企业工作流入口`
- 文档、合同、票据、表格和知识库 ingestion 成为企业落地的重要入口

## 你先要抓住什么

- 多模态不只是“能看图”，而是能把不同模态放进一个共享表示空间中理解和生成
- 文本、图像、音频、视频可以被分别编码，再在统一模型中对齐或融合
- 多模态能力既是模型能力问题，也是产品形态问题
- 最近两年最值得注意的是：多模态越来越像 agent 的一个动作与感知表面，而不只是展示能力

## 关键问题

- 不同模态如何表示和对齐
- 多模态模型是一个统一模型，还是多个模块拼接
- 多模态能力为什么会改变 agent、assistant 和搜索产品
- 为什么 OCR 与 document AI 正重新变成前沿主线的一部分

## 典型能力

- 图文理解
- 图像生成与编辑
- 语音输入输出
- 实时语音对话
- 视频理解
- computer use
- document understanding
- OCR / layout understanding
