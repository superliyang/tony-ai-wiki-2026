# Agent与工具调用题库

## Q1. 如何保证 Function Calling 的可靠性？
- 考察点：是否具备生产级可靠性设计。
- 回答骨架：Schema 校验、策略门禁、幂等键、超时重试、熔断、审计日志、人工兜底。
- 追问：如何避免重复扣费/重复写入？

## Q2. Tool Calling 的权限边界怎么设计？
- 考察点：安全与治理。
- 回答骨架：最小权限临时 token、工具白名单、参数级策略检查、危险动作审批。
- 追问：如何处理 Prompt 注入导致的越权调用？

## Q3. 多 Agent 协作中如何避免失控与循环调用？
- 考察点：编排与状态管理能力。
- 回答骨架：任务 DAG、步骤预算、深度限制、状态总线、终止条件、故障回退。
- 追问：怎么做冲突裁决？

## Q4. 主 Agent 如何动态派生 subAgent？
- 考察点：动态调度能力。
- 回答骨架：Planner 拆解任务，Router 按能力/SLA/成本选子代理，Coordinator 合并结果。
- 追问：什么时候不该拆分？

## Q5. Agent 为什么“偶尔很强、偶尔很差”？
- 考察点：稳定性分析能力。
- 回答骨架：上下文漂移、工具波动、非确定性采样、检索噪声、缺少回归集。
- 追问：如何把“偶然成功”变“稳定成功”？

## Q6. 你如何设计 Agent 的观测指标？
- 考察点：AgentOps 能力。
- 回答骨架：任务完成率、步骤数、工具成功率、重试率、人工接管率、失败类型分布。
- 追问：你如何定义“成功任务”？

## Q7. Agent 工具结果不可信怎么办？
- 考察点：结果验证机制。
- 回答骨架：二次读取、校验和、跨源比对、关键操作必须可验证证据。
- 追问：证据冲突时怎么裁决？

## Q8. 如何从 Prompt Agent 升级到可维护 Agent 平台？
- 考察点：平台化能力。
- 回答骨架：抽象工具层、统一策略网关、可回放日志、Eval Harness、发布门禁。
- 追问：你会先做哪一层？为什么？
