# Claude 系列

## 简介

`Claude 系列` 是 Anthropic 的核心模型家族，也是安全导向模型产品化的一条代表路线。

## 为什么重要

- 是闭源高能力模型中的关键选手
- 在 coding、long-context 和 enterprise 场景中影响很大
- 经常被拿来与 OpenAI、Google DeepMind 进行横向比较

## 这条路线代表什么

- 它代表了“高能力模型 + 安全叙事 + 企业工作流”的一条产品化路径
- Claude 的讨论重点，往往不只是模型强不强，而是它在复杂知识工作中的稳定性和可控性

## 学习时重点看什么

- Claude 的能力和 safety 叙事如何结合
- 它为什么在 coding 与知识工作场景里常被重点讨论
- 它与 [[Claude Code]] 的关系

## 你学习 Claude 时最该建立的判断

- 为什么 Anthropic 强调安全、长期主义和企业使用场景
- 为什么 Claude 经常和 `long context`、`coding`、`agentic workflow` 绑定出现
- 它与 GPT 的差异到底来自模型本身，还是来自产品定位

## 截至 2026-04-07 最值得记住的 2026 节点

- [[Claude Opus 4-6|Claude Opus 4.6]]：最强旗舰，继续强化 `1M context + hybrid reasoning + frontier coding`
- [[Claude Sonnet 4-6|Claude Sonnet 4.6]]：更高性价比的主力层，把高能力工作负载进一步下沉

到 `2026`，`Claude 系列` 更像一条非常清楚的双层路线：

- `Opus`：高强度、长任务、低容错旗舰
- `Sonnet`：大规模生产部署中的高能力 workhorse

## 2026-04-07 之后的新节点

- [[Claude Opus 4-7|Claude Opus 4.7]]：继续强化 Claude 在 long-running coding、复杂指令遵循、vision 和安全边界上的 flagship work model 位置

这意味着 Claude 路线越来越适合用 `model capability -> Claude Code -> enterprise workflow -> safety / security boundary` 来理解。
