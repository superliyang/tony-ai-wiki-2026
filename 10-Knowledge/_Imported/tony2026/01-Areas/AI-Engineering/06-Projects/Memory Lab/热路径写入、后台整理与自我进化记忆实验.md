# 热路径写入、后台整理与自我进化记忆实验

## 实验目标

比较两种 memory system：

- `hot-path first`
- `background consolidation first`

并且模拟一条“准自进化”路径：

- trace -> memory candidate -> review / eval -> stable memory / rule update

## 你应该记录什么

- 延迟差异
- memory 污染率
- stale / invalid memory 的出现方式
- recall 噪声
- 是否真的提升后续任务 success rate

## 最关键的判断

如果一条 memory 会影响：

- tool use
- routing
- safety boundary
- project defaults

那么它通常不适合直接热路径无审查写入。
