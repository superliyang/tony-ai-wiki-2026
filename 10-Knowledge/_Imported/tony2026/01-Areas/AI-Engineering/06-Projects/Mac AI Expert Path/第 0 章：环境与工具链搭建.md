# 第 0 章：环境与工具链搭建

## 本章目标

在这一章结束时，你要把 `MacBook Pro Max M4` 变成一个可靠的本地 AI 实验室，而不是一个偶尔能跑命令的试玩环境。

## 最小工具链

- Xcode command line tools
- Python 环境管理（推荐 `venv` 或 `uv`）
- `PyTorch`
- `Ollama`
- `llama.cpp`
- `MLX` / `MLX-LM`
- Jupyter 或 notebook 环境
- Git 与一个你能长期维护的项目目录

## 建议安装顺序

1. 先确认基础环境：`xcode-select --install`
2. 建一个干净 Python 环境
3. 安装 `PyTorch` 并验证 `mps`
4. 装 `Ollama` 跑通第一个模型
5. 装 `llama.cpp` 跑通本地推理
6. 安装 `mlx` / `mlx-lm`
7. 建一个 notebook 或脚本目录，作为你的长期实验室

## 你至少要完成的验证

### 验证 1：PyTorch MPS 可用

```python
import torch
print(torch.backends.mps.is_available())
print(torch.device('mps'))
```

### 验证 2：本地模型可运行

- 任选 `Ollama` 或 `llama.cpp`
- 跑通一个能正常回答问题的模型

### 验证 3：MLX-LM 最小示例可运行

- 让 `MLX-LM` 成功加载一个模型并完成一次生成

## 目录建议

你可以在本地维持这样一个学习目录：

- `notebooks/`：概念验证和小实验
- `scripts/`：可重复运行的实验脚本
- `datasets/`：样本数据
- `artifacts/`：模型输出、评测结果、图表
- `apps/`：RAG / agent / UI 原型

## 本章输出物

完成本章后，至少保留这些东西：

- 一份环境安装记录
- 一个可复用的 Python 环境
- 一个验证 `mps` 的最小脚本
- 一个本地模型运行脚本
- 一个 `MLX-LM` 最小示例

## 常见误区

- 误区 1：一开始就追求最复杂框架
- 误区 2：环境混乱，没有可重复性
- 误区 3：只会点工具按钮，不知道验证标准
- 误区 4：把“跑通一次”误以为“工具链已经稳定”
