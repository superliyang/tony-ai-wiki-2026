# Game Agent、AI NPC 与创作助手

## 先把概念拆开

这个领域现在最容易混淆的，是把所有“游戏里的 AI”都叫成一件事。

其实至少有 3 条线：

### 1. 创作助手

例子：

- `Unity AI`
- `Roblox Studio Assistant`
- `Google AI Studio`

它们主要帮助：

- 写代码
- 做资产
- 出文案
- 跑 workflow

### 2. 游戏内角色 Agent

例子：

- `NVIDIA ACE`
- `Inworld`
- `Convai`

它们主要帮助：

- NPC 对话
- scene-aware 角色行为
- companion / host / guide

### 3. 运行时协议和工作台

例子：

- `MCP`
- editor MCP server
- agent runtime / graph / eval loop

它们解决的是：

- 怎么接工具
- 怎么给上下文
- 怎么让 agent 在工作流里真的可控

## 对行业的前沿判断

现在的前沿不只是“NPC 会说话”。

真正更前沿的是：

- editor 里的 agent
- 运行时里的 agent
- 跨工具的协议层
- 生成媒体和角色 runtime 的联动

## 对你们当前项目的建议

### 值得立刻用的

- 创作助手
- 文案/视觉/宣传生成
- TTS / host voice

### 适合观察但别先重押的

- AI NPC
- companion systems
- 更复杂的 agentic runtime

因为你们当前的成功关键仍然是：

- 可玩 loop
- 公平性
- 结果解释性
- 上手门槛
