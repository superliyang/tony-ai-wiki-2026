# 适合 Skills Gaming 的玩法原型

## 先看头部公司在做什么

从 `Papaya`、`AviaGames`、`WorldWinner` 的 portfolio 看，反复出现的并不是超复杂玩法，而是这些方向：

- solitaire / card
- bingo
- bubble shooter
- pool / precision sports
- 2048 / number puzzle
- match / flip / tile puzzle
- casual arcade with clear scoring

这说明一个很稳定的产品判断：

- 最适合这个品类的，不是“机制特别新”，而是“玩家一看就懂，但 skill expression 足够清楚”的玩法

## 好的原型通常满足 6 个条件

1. 玩法一眼能懂
2. 有明确 skill surface
3. 分数可以稳定定义
4. session 足够短
5. luck 不会压过 skill
6. 美术和工程能在短时间内做出像样版本

## 从 hackathon 视角，最有希望的 4 类原型

### 1. 熟悉规则 + 高 skill ceiling

比如：

- solitaire-like
- precision puzzle
- lane-clear / pathing puzzle

### 2. 明确速度与精度竞争

比如：

- word sprint
- number merge sprint
- aim / trickshot

### 3. 低资产负担 + 高 replayability

比如：

- UI-heavy card/puzzle games
- board / tile variants

### 4. 可活动化扩展的 core loop

比如：

- 同一玩法可以换 event rules
- 同一 scoring 模型可以换 daily goals

## 不太适合 hackathon 第一版的方向

- 重剧情和大量 content production
- 强 3D 物理和高复杂控制
- 强随机驱动、争议很多的玩法
- 太依赖多人实时对抗的复杂网络同步

## 最重要的一句

对你们这次场景，最好的创新通常不是“最陌生的玩法”，而是：

- 一个熟悉玩法
- 一个新颖但讲得清的 scoring / competition 包装
- 一套更稳的 fairness 和 demo 体验
