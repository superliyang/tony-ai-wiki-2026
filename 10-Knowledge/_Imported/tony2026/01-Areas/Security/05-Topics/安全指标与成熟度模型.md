# 安全指标与成熟度模型

## 定位

安全指标用来回答：

- 风险是否下降
- 控制是否覆盖
- 事件响应是否变快
- 漏洞是否更快修复
- 合规证据是否持续生成
- 安全投入是否有效

## 指标分层

### 1. 覆盖类指标

- MFA 覆盖率
- EDR 覆盖率
- 日志接入覆盖率
- 关键资产入库率
- 加密覆盖率
- 安全扫描覆盖率

### 2. 风险类指标

- critical vulnerability count
- exposed asset count
- high-risk permission count
- stale account count
- third-party high risk count

### 3. 响应类指标

- MTTD
- MTTR
- containment time
- incident recurrence rate
- tabletop exercise completion

### 4. 工程类指标

- security gate pass rate
- SAST / SCA critical finding aging
- secret leakage count
- patch SLA breach rate
- exception aging

### 5. 合规类指标

- control evidence freshness
- audit finding count
- policy exception count
- privacy request SLA
- incident notification readiness

## 成熟度模型

### Level 1：反应式

- 事故后才处理
- 资产不清
- 控制靠人记

### Level 2：基础控制

- 关键系统有 MFA、日志、备份、扫描
- 有基本响应流程

### Level 3：流程化

- 安全进入 SDLC
- 漏洞和风险有 owner、SLA、例外机制
- SOC 或 MDR 可运行

### Level 4：平台化

- 安全控制产品化到平台和 CI/CD
- 证据自动生成
- 检测工程持续迭代

### Level 5：风险驱动

- 指标与业务风险、合规和投资决策相连
- 安全从 gatekeeper 变成业务可信增长能力
