# 安全能力模型：知道什么、会做什么

> 这页是安全学习的“能力清单”。不是按名词堆知识，而是把每个方向拆成：应该知道什么、能够做什么、能产出什么。

## 总原则

安全能力不是“会攻击”或“会背标准”，而是能稳定完成五类工作：

1. 识别风险：知道资产、攻击面、威胁、影响。
2. 设计控制：能设计预防、检测、响应、恢复、证据。
3. 嵌入工程：能让安全进入研发、发布、运维和数据流程。
4. 运营响应：能发现攻击、分诊告警、组织事件响应。
5. 治理证明：能让风险、例外、控制和审计证据可管理。

## 能力矩阵

| 能力域 | 应该知道什么 | 能够做什么 | 典型产出 |
|---|---|---|---|
| 安全基础思维 | 资产、攻击面、威胁、风险、控制、残余风险 | 把模糊问题翻译成安全问题 | 风险说明、优先级、控制建议 |
| 威胁建模 | STRIDE、attack tree、kill chain、信任边界 | 为系统找攻击路径和控制点 | 威胁模型、数据流图、控制清单 |
| 身份与访问 | authn/authz、MFA、SSO、RBAC/ABAC、PAM、secret | 设计权限模型和高权限治理 | IAM 模型、权限矩阵、密钥轮换方案 |
| 网络安全 | 分段、边界、VPN/ZTNA、WAF、DDoS、DNS、IDS/IPS | 判断暴露面和流量控制 | 网络分段方案、边界控制、访问策略 |
| 应用/API 安全 | OWASP、鉴权、授权、输入输出、业务逻辑、API abuse | 做上线安全评审和 API 风险判断 | 安全评审报告、release gate、修复建议 |
| 客户端/终端 | EDR、MDM、钓鱼、恶意软件、App 加固、设备态势 | 判断终端和客户端入口风险 | 终端基线、App 加固策略、钓鱼防护方案 |
| 数据安全与隐私 | 数据分类、加密、脱敏、DLP、保留删除、跨境 | 做数据处理活动安全和隐私评审 | 数据流图、DPIA、访问审计、删除策略 |
| 云与基础设施 | 云 IAM、CSPM、CNAPP、容器、K8s、IaC、runtime | 发现云配置和运行时风险 | 云安全基线、K8s 基线、IaC 检查 |
| 供应链安全 | SCA、SBOM、签名、provenance、CI/CD、第三方 | 判断依赖、构建、产物和发布可信度 | SBOM、构建加固方案、供应商评估 |
| 漏洞与攻击面 | CVSS、EPSS、暴露面、可利用性、补偿控制 | 给漏洞排序、修复、接受或隔离 | 漏洞优先级、修复 SLA、例外记录 |
| 安全运营 | 日志源、SIEM、SOAR、EDR、检测规则、威胁情报 | 分诊告警、设计检测、组织响应 | 检测规则、事件时间线、复盘报告 |
| 事件响应 | 分级、止血、取证、根除、恢复、通报、复盘 | 处理账号接管、密钥泄露、数据泄露等事件 | IR runbook、证据包、整改计划 |
| GRC | 风险登记、控制库、审计证据、例外、第三方风险 | 把安全变成组织机制 | risk register、control mapping、审计包 |
| 地区合规 | GDPR、CCPA/CPRA、PIPL、PCI DSS、SOC 2、ISO 27001 等坐标 | 把地区/行业要求转成技术控制和证据 | 合规矩阵、数据处理记录、通报流程 |
| 安全架构沟通 | 架构权衡、业务约束、成本、风险接受、管理层汇报 | 让安全方案被业务和工程团队采纳 | 架构决策记录、路线图、指标看板 |

## 能力分级

### L1：概念识别

能做到：

- 解释常见安全术语。
- 看懂安全架构图。
- 按清单发现显而易见的问题。

还不能稳定做到：

- 处理真实 incident。
- 设计跨团队控制。
- 做业务风险取舍。

### L2：单点执行

能做到：

- 对一个系统做基础安全评审。
- 对一个漏洞做优先级判断。
- 按 playbook 处理常见告警。
- 写出基本修复建议。

关键产出：

- 安全评审记录。
- 漏洞分级。
- 简单威胁模型。
- 修复 checklist。

### L3：流程贯通

能做到：

- 把安全控制嵌入 SDLC / CI/CD。
- 建立身份、数据、云、供应链等专项治理流程。
- 组织事件响应和复盘。
- 把控制转成审计证据。

关键产出：

- release gate。
- 安全基线。
- incident response runbook。
- risk register。

### L4：架构设计

能做到：

- 设计企业安全目标架构。
- 统一身份、网络、数据、应用、云、供应链和运营能力。
- 选择自建、采购、托管、平台化之间的路线。
- 用指标证明安全能力提升。

关键产出：

- 企业安全架构图。
- 安全能力路线图。
- 控制库和指标体系。
- 管理层安全汇报。

### L5：组织能力塑造

能做到：

- 在业务增长、工程效率、合规要求和安全风险之间做取舍。
- 把安全从安全团队责任变成组织默认能力。
- 培养工程团队安全判断力。
- 通过演练、事故、审计、红队和指标持续进化。

关键产出：

- 安全 operating model。
- 平台化安全能力。
- 长期治理机制。
- 安全文化和复盘系统。

## 三类核心问题

### 1. 我应该会什么

最低组合：

- 安全基础思维
- 身份与访问
- 应用/API 安全
- 数据安全与隐私
- 云与基础设施
- 漏洞与攻击面
- 安全运营与事件响应
- GRC 基础

进阶组合：

- 威胁建模
- DevSecOps
- 供应链安全
- 零信任
- 检测工程
- 地区合规
- 企业安全架构

### 2. 我应该能做什么

至少要能完成：

- 快速诊断一个安全问题。
- 评审一个系统或 API。
- 判断一个漏洞的优先级。
- 设计一组控制，而不是只推荐一个工具。
- 处理一次常见安全事件。
- 解释一个数据处理活动的隐私风险。
- 把风险、控制和证据写清楚。

### 3. 我应该怎么证明自己掌握了

用产物证明，而不是用“看过很多文章”证明：

- 一张系统威胁模型。
- 一份 API 安全评审。
- 一份数据隐私评审。
- 一份事件响应复盘。
- 一份企业安全能力路线图。
- 一份风险登记册。
- 一张指标看板。
