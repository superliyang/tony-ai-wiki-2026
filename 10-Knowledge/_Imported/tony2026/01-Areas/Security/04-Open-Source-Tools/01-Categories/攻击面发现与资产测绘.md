# 攻击面发现与资产测绘

## 解决什么问题

- 不知道公网有哪些资产、域名、子域名、端口、Web 入口和服务指纹。
- 漏洞管理和红蓝紫演练缺少资产基线。
- 安全团队需要把 EASM / ASM 的思路用开源工具先跑出可见性。

## 代表项目

- [OWASP Amass](https://github.com/owasp-amass/amass)：攻击面映射与资产发现。
- [subfinder](https://github.com/projectdiscovery/subfinder)：被动子域名枚举。
- [httpx](https://github.com/projectdiscovery/httpx)：HTTP 探测与指纹。
- [naabu](https://github.com/projectdiscovery/naabu)：快速端口扫描。
- [katana](https://github.com/projectdiscovery/katana)：爬虫与链接发现。
- [Nmap](https://github.com/nmap/nmap)：网络扫描经典工具。
- [masscan](https://github.com/robertdavidgraham/masscan)：高速端口扫描。
- [RustScan](https://github.com/bee-san/RustScan)：现代端口扫描器。
- [dnstwist](https://github.com/elceef/dnstwist)：钓鱼域名和 typo-squatting 发现。
- [SpiderFoot](https://github.com/smicallef/spiderfoot)：OSINT 与攻击面自动化。

## 典型组合

`subfinder / Amass -> dnsx -> httpx -> naabu / Nmap -> nuclei -> 工单 / 资产库`

## 学习价值

- 理解 EASM/ASM 的底层不是神秘平台，而是资产枚举、探测、指纹、验证、去重和 owner 归属。
- 训练“先看资产，再谈漏洞”的安全思维。

## 风险与边界

- 对外部目标扫描必须有授权。
- 高速扫描会造成噪声、误伤和封禁风险。
- 资产发现结果必须进入资产库和 owner 机制，否则只是一次性报告。
