# 安全能力自评表

> 用途：把“我感觉懂了”变成可检查的掌握证据。

## 评分方式

- `0`：没概念。
- `1`：能解释术语。
- `2`：能按 checklist 执行。
- `3`：能独立评审和处理问题。
- `4`：能设计流程、控制和指标。
- `5`：能在业务约束下做架构取舍，并教别人。

## 核心能力自评

| 能力 | 0-5 | 证据产物 | 下一步 |
|---|---:|---|---|
| 安全基础思维 |  | 风险诊断、资产/攻击面说明 |  |
| 威胁建模 |  | DFD、attack tree、控制点 |  |
| 身份与访问 |  | 权限矩阵、PAM/SSO/MFA 方案 |  |
| 网络安全 |  | 分段方案、边界策略、流量路径 |  |
| 应用/API 安全 |  | API 安全评审、release gate |  |
| 客户端/终端安全 |  | 终端基线、App 加固策略 |  |
| 数据安全与隐私 |  | 数据流图、DPIA、删除/保留策略 |  |
| 云原生安全 |  | 云基线、K8s 基线、IaC 策略 |  |
| 供应链安全 |  | SBOM、SCA、签名、CI/CD 加固 |  |
| 漏洞与攻击面 |  | 暴露面清单、漏洞优先级、SLA |  |
| SOC 检测工程 |  | 检测规则、日志覆盖、分诊流程 |  |
| 事件响应 |  | incident timeline、复盘、整改计划 |  |
| GRC |  | risk register、control mapping、审计证据 |  |
| 地区合规 |  | 合规矩阵、通报流程、数据处理记录 |  |
| 安全架构沟通 |  | 路线图、ADR、管理层汇报 |  |

## 复盘问题

- 哪些能力只是“知道”，还不能产出？
- 哪些能力能产出，但还不能解释取舍？
- 哪些能力只在单系统有效，还不能组织级落地？
- 哪些能力缺真实案例验证？
- 下一轮最应该补哪个专项？
