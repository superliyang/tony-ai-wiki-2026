## 本领域解决什么问题

- 你要保护什么资产：账号、终端、代码、服务、数据、模型、供应链、业务流程
- 你面对什么威胁：攻击者、恶意软件、漏洞利用、账号接管、内部威胁、供应链投毒、合规风险
- 攻击面在哪里：网络、客户端、应用、API、云、身份、数据、第三方、人员与流程
- 用什么控制：预防、防护、检测、响应、恢复、治理、审计
- 怎么形成能力：不是买工具，而是建立 `风险识别 -> 控制设计 -> 检测响应 -> 复盘改进 -> 合规证明` 的闭环

## 领域边界

### 包含

- 网络安全、终端/客户端安全、应用与 API 安全
- 身份与访问控制、数据安全、隐私保护、地区合规
- 云原生、基础设施、供应链、DevSecOps
- 安全运营、检测响应、威胁情报、事件复盘
- GRC、审计、风险管理、政策制度与安全组织机制

### 暂不深入

- 纯军事安全、物理安防、传统企业风控
- 非计算机语境下的公共安全
- 只和 AI 相关的安全细节会保留在 [[../AI-Engineering/专题总览|AI-Engineering]]，但这里会建立连接

## 这套安全图谱的核心判断

安全不是一个工具清单，而是一个分层系统：

`资产 -> 攻击面 -> 威胁 -> 控制 -> 检测 -> 响应 -> 恢复 -> 治理 -> 合规证明`

如果只学漏洞和攻击技巧，很容易变成碎片。

如果只学合规条款，又会离真实攻击太远。

真正稳定的安全理解，要把 `攻防技术`、`工程控制`、`运营流程`、`组织治理` 和 `地区监管` 放到一张图里。

工具平台层的正确位置是：把这些控制、流程和证据落到真实系统里。对应入口：[[./02-Tools/工具平台分类索引|工具平台分类索引]]、[[./02-Tools/GitHub 热门开源安全工具雷达|GitHub 热门开源安全工具雷达]]、[[./04-Open-Source-Tools/专题总览|Security Open Source Tools 专题总览]]、[[./06-Maps/安全工具平台能力地图|安全工具平台能力地图]]。

行业控制层的正确位置是：把通用安全主干迁移到具体业务资产、威胁、控制、指标和证据里。对应入口：[[./03-Industry-Controls/行业控制清单索引|行业控制清单索引]]、[[./06-Maps/行业安全控制迁移地图|行业安全控制迁移地图]]。

## 什么叫真正掌握安全

这套知识库的掌握目标不是背术语，而是形成点线面能力：

- 点：知道身份、网络、应用、数据、云、供应链、终端、密码、漏洞、日志、GRC 等基础控制。
- 线：能把点串成攻击链、安全开发线、事件响应线、数据生命周期、身份生命周期、供应链线和审计线。
- 面：能设计企业安全架构、安全运营体系、安全工程平台、风险治理体系和数据隐私治理体系。
- 输出：能诊断、能设计、能评审、能响应、能证明、能讲清楚。

对应入口：[[./06-Maps/安全点线面能力地图|安全点线面能力地图]]、[[./05-Topics/安全能力模型：知道什么、会做什么|安全能力模型：知道什么、会做什么]]
