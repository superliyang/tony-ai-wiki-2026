# 零信任落地 Playbook

> 用途：把“零信任”从口号变成可实施路线，覆盖身份、设备、应用、数据、网络、服务和持续监控。

## 一句话

零信任不是“替换 VPN”，而是把访问决策从“你在内网所以可信”改成：

`身份可信 + 设备可信 + 上下文合理 + 权限最小 + 持续监控 + 可审计`

## 适用场景

- 员工远程办公、外包、供应商访问内部系统。
- SaaS、云平台、管理后台、生产环境需要统一访问控制。
- VPN 权限过大，进来后能访问太多内网资源。
- 高权限操作缺少设备态势、二次验证、审批和审计。
- 想从网络边界安全迁移到身份和策略驱动安全。

## 落地输入

至少准备：

- 用户、员工、外包、供应商身份清单。
- 应用和系统清单。
- 高权限角色和生产权限清单。
- 设备类型、MDM/EDR 覆盖情况。
- 当前 VPN、堡垒机、PAM、SSO、MFA 状态。
- 关键数据和高风险操作清单。
- 日志、审计、异常检测能力。

## 七层能力模型

### 1. 身份层

目标：所有访问都能绑定到明确主体。

最低控制：

- 统一 IdP / SSO。
- MFA 覆盖员工、高权限和外部访问。
- 员工、外包、供应商账号生命周期可管理。
- service account 有 owner、用途、权限范围、过期和轮换。

失败信号：

- 共享账号。
- 离职账号未及时回收。
- 高权限账号没有 MFA。
- API key 和长期 token 没有 owner。

### 2. 设备层

目标：访问决策不能只看人，还要看设备是否可信。

最低控制：

- 设备注册和资产归属。
- MDM / EDR 基础覆盖。
- 设备加密、补丁、屏锁、恶意软件状态检查。
- 不可信设备限制访问敏感系统。

失败信号：

- 个人设备可直接访问生产后台。
- 设备感染、越狱、root 后仍可访问。
- 设备离线或失管后权限不变。

### 3. 应用访问层

目标：从“连入网络”变成“访问具体应用”。

最低控制：

- 应用级访问策略。
- 按角色、设备、地点、风险评分授权。
- 管理后台、生产工具、数据导出工具单独分级。
- 高风险操作二次验证和审计。

失败信号：

- VPN 进入后可扫内网。
- 所有内部系统共享一套粗粒度权限。
- 管理后台只靠网络白名单。

### 4. 数据层

目标：访问数据要按敏感等级和业务目的控制。

最低控制：

- 数据分类分级。
- 高敏数据访问审批和审计。
- 数据导出限流、水印、审批、追踪。
- 按字段、表、租户、区域做访问边界。

失败信号：

- 有系统权限就能批量导出全部数据。
- 日志、BI、客服系统复制敏感数据但无审计。

### 5. 网络与服务层

目标：假设内网已经被突破，限制横向移动。

最低控制：

- 网络分段。
- service-to-service policy。
- 管理面和数据面隔离。
- 生产环境、办公网、测试环境隔离。
- 关键服务默认拒绝，按需开放。

失败信号：

- flat network。
- 测试环境能访问生产数据库。
- 服务之间没有身份和策略。

### 6. 策略与自动化层

目标：策略可解释、可测试、可回滚。

最低控制：

- 策略即代码或集中策略管理。
- 例外有 owner、原因、期限、补偿控制。
- 高风险策略变更有审批和回滚。
- 权限定期复核。

失败信号：

- 策略散落在各系统。
- 例外永久存在。
- 权限变更无法追溯。

### 7. 监控与响应层

目标：持续验证，不是一次登录后永久信任。

最低控制：

- 登录、权限变更、设备状态、数据访问、管理操作日志。
- 异常地点、异常设备、异常时间、异常访问量检测。
- 高风险访问自动升级验证或阻断。
- 事件响应和取证链路。

失败信号：

- 只有登录日志，没有资源访问日志。
- 告警无人处理。
- 出事后无法还原访问路径。

## 落地路线

### Phase 1：先补身份

- 统一 SSO。
- MFA 覆盖高风险入口。
- 清理离职、外包、共享账号。
- 梳理高权限账号和 service account。

### Phase 2：应用级访问

- 盘点内部应用。
- 替换粗粒度 VPN 访问。
- 对管理后台、生产工具、数据导出做分级访问。
- 建立访问审批和审计。

### Phase 3：设备态势

- 建立设备资产清单。
- 接入 MDM / EDR。
- 把设备状态纳入访问策略。
- 对非托管设备做只读、隔离或禁止策略。

### Phase 4：最小权限和 PAM

- 高权限角色最小化。
- 生产权限临时化。
- 高风险操作审批、录屏、审计。
- 权限定期复核。

### Phase 5：服务和数据细粒度控制

- 服务间身份。
- 网络微分段。
- 数据字段、租户、区域级访问控制。
- 数据访问异常检测。

### Phase 6：持续验证

- 风险评分。
- 行为分析。
- 自动升级验证。
- 例外治理和指标看板。

## 关键产物

- 零信任目标架构图。
- 应用访问分级清单。
- 高权限账号和 service account 清单。
- 设备态势接入清单。
- 权限矩阵和复核机制。
- 例外登记表。
- 零信任指标看板。

## 指标

- MFA 覆盖率。
- SSO 覆盖率。
- 高权限账号数量和老化情况。
- service account owner 覆盖率。
- 非托管设备访问敏感系统次数。
- VPN 到应用级访问迁移比例。
- 权限复核完成率。
- 例外过期率。

## 常见误区

- 把零信任等同于 ZTNA 产品。
- 只做员工访问，不做服务账号和服务间访问。
- 只做身份，不做设备、数据和日志。
- 策略过细但不可维护。
- 没有例外治理，最后所有人都走例外。
