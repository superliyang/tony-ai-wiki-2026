# 安全快速诊断 Playbook

> 用途：当你听到一个安全问题、告警、漏洞、审计发现或上线担忧时，用这套流程快速判断“问题是什么、严重到哪里、下一步做什么”。

## 一句话

安全诊断的核心不是立刻找工具，而是把模糊问题翻译成：

`资产 -> 攻击面 -> 威胁 -> 弱点 -> 影响 -> 控制 -> 检测 -> 响应 -> 证据`

## 30 分钟诊断流程

### 1. 定义资产

必须先问：

- 受影响的系统、账号、数据、业务流程是什么？
- asset owner 是谁？
- 是否涉及高权限身份、生产环境、支付、个人信息、密钥或客户数据？
- 这个资产是否在关键路径上，例如登录、支付、发布、客服、运维、数据导出？

输出：资产清单和 owner。

### 2. 找攻击面

继续问：

- 攻击者从哪里能接触到它：公网、内网、VPN、第三方、移动端、浏览器、CI/CD、云控制面？
- 是否有匿名访问入口？
- 是否依赖用户输入、文件上传、回调、Webhook、第三方 SDK？
- 是否存在服务账号、API key、token、证书或长期密钥？

输出：入口路径和信任边界。

### 3. 判断威胁和可利用性

核心问题：

- 谁会利用：外部攻击者、内部人、供应商、恶意用户、自动化脚本？
- 是否已有 PoC、在野利用、漏洞公告或攻击迹象？
- 利用是否需要登录、高权限、特定网络位置或用户交互？
- 攻击是否可自动化、批量化、低成本复现？

输出：威胁模型和可利用性判断。

### 4. 判断影响

影响不能只说“有风险”，要具体：

- 数据影响：读取、修改、删除、导出、重识别？
- 权限影响：提权、横向移动、接管账号、拿到云权限？
- 业务影响：资金损失、欺诈、服务中断、库存/价格/订单被操纵？
- 合规影响：是否涉及个人信息、跨境、行业监管、通报时限？

输出：影响范围和 blast radius。

### 5. 给出优先级

一个实用的优先级判断：

- `P0`：正在被利用，或已经发生数据泄露/业务中断/高权限接管。
- `P1`：公网可利用，影响敏感数据、高权限或关键业务。
- `P2`：需要特定条件，但影响明确，必须排期修复。
- `P3`：低可利用性或低影响，可纳入治理 backlog。

输出：优先级、owner、截止时间。

## 风险判断矩阵

| 维度 | 高风险信号 | 低风险信号 |
|---|---|---|
| 暴露面 | 公网、匿名、第三方可达 | 仅本地或受强身份控制 |
| 可利用性 | 有 PoC、在野利用、可自动化 | 理论可行、条件苛刻 |
| 权限 | 高权限、生产、云控制面 | 低权限、隔离环境 |
| 数据 | 个人信息、支付、密钥、商业秘密 | 无敏感数据 |
| 检测 | 没有日志、没有告警 | 有审计、可回溯 |
| 恢复 | 无备份、无法回滚 | 可隔离、可回滚 |

## 常见误区

- 只看 CVSS，不看真实暴露面。
- 只修代码，不查日志和攻击迹象。
- 只问漏洞，不问数据和权限。
- 只做临时修复，不补 release gate。
- 只满足审计，不验证控制是否真的有效。
