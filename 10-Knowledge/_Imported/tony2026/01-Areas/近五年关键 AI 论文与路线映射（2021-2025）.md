# 近五年关键 AI 论文与路线映射（2021-2025）

> 这页不追求“把近五年所有重要论文收全”，而是挑出真正改路线的锚点论文，然后把它们接回模型路线、系统路线和产品路线。

如果你想继续补最近两年的 `long context / multimodal / computer use` 新路线，下一页去看 [[AI-Learning/11-Recent-Supplements/2024-2026 AI 新路线补线：Long Context、Multimodal 与 Computer Use|2024-2026 AI 新路线补线：Long Context、Multimodal 与 Computer Use]]。

## 先记住这页怎么用

- 不要把它当 `paper list`，要把它当 `paper -> model -> system -> product` 的回顾页。
- 一篇论文是否值得反复看，不只看 benchmark，而看它有没有改后续的工程习惯、产品形态和行业叙事。
- 最近五年的关键变化，主要集中在 `后训练`、`reasoning`、`tool use`、`open-weight`、`multimodal frontier` 和 `reasoning by RL`。

## 路线一：后训练把模型变成助手

这条线最该连着读的是：

1. [[AI-Learning/04-Papers/Training language models to follow instructions with human feedback|Training language models to follow instructions with human feedback]]
2. [[AI-Learning/04-Papers/Constitutional AI|Constitutional AI]]
3. [[AI-Learning/04-Papers/Direct Preference Optimization：Your Language Model is Secretly a Reward Model|Direct Preference Optimization]]
4. [[AI-Learning/04-Papers/GPT-4 Technical Report|GPT-4 Technical Report]]

这条线真正解释的是：

- 为什么 `next-token predictor` 最后会变成 [[AI-Learning/09-Systems/ChatGPT|ChatGPT]] 这种日常助手
- 为什么 `alignment / preference optimization` 会直接影响产品体验、语气、稳定性和企业 adoptability
- 为什么今天理解模型，不够只看 pretraining，还得看 post-training

## 路线二：compute-optimal 训练重新定义 frontier model economics

这条线最该连着读的是：

1. [[AI-Learning/04-Papers/An Empirical Analysis of Compute-Optimal Large Language Model Training|An Empirical Analysis of Compute-Optimal Large Language Model Training]]
2. [[AI-Learning/04-Papers/GPT-4 Technical Report|GPT-4 Technical Report]]

这条线真正解释的是：

## 路线三：reasoning、tool use 与 agent loop

这条线最该连着读的是：

1. [[AI-Learning/04-Papers/Chain-of-Thought Prompting Elicits Reasoning in Large Language Models|Chain-of-Thought Prompting Elicits Reasoning in Large Language Models]]
2. [[AI-Learning/04-Papers/ReAct：Synergizing Reasoning and Acting in Language Models|ReAct: Synergizing Reasoning and Acting in Language Models]]
3. [[AI-Learning/04-Papers/Toolformer：Language Models Can Teach Themselves to Use Tools|Toolformer: Language Models Can Teach Themselves to Use Tools]]
4. [[AI-Learning/04-Papers/DeepSeek-R1 Technical Report|DeepSeek-R1 Technical Report]]

这条线真正解释的是：

- 为什么今天的 `agent` 不是突然冒出来的，而是从 `reasoning trace -> acting loop -> tool use -> policy/runtime` 一步步长出来的
- 为什么 [[AI-Learning/09-Systems/LangGraph|LangGraph]]、[[AI-Learning/09-Systems/ChatGPT Agent|ChatGPT Agent]]、[[AI-Learning/06-Topics/Browser Agents 与 Computer Use|Browser Agents 与 Computer Use]] 这类系统，本质上都继承了这条论文线
- 为什么 `reasoning model` 会同时是模型问题、runtime 问题和产品问题

## 路线四：open-weight 与低成本定制

这条线最该连着读的是：

1. [[AI-Learning/04-Papers/Llama 2：Open Foundation and Fine-Tuned Chat Models|Llama 2: Open Foundation and Fine-Tuned Chat Models]]
2. [[AI-Learning/04-Papers/QLoRA：Efficient Finetuning of Quantized LLMs|QLoRA: Efficient Finetuning of Quantized LLMs]]
3. [[AI-Learning/04-Papers/Llama 3 Herd of Models|Llama 3 Herd of Models]]
4. [[AI-Learning/04-Papers/DeepSeek-R1 Technical Report|DeepSeek-R1 Technical Report]]

这条线真正解释的是：

- 为什么 open-weight 不只是“开源更自由”，而是会重写微调、部署、蒸馏和行业定制的经济结构
- 为什么 [[AI-Learning/09-Systems/Ollama|Ollama]]、[[AI-Learning/09-Systems/llama-cpp|llama.cpp]]、[[AI-Learning/09-Systems/MLX|MLX]] 这些系统能快速成为本地 AI 和私有 AI 的入口
- 为什么过去几年产品路线不再只有 `frontier API`，还多了一条 `私有部署 / 边缘部署 / 行业定制`
