# 技术管理全景图

这张图回答一个问题：技术管理者如何把业务目标转成组织技术产能。

```mermaid
flowchart TB
  Biz["业务目标"] --> Strategy["技术战略"]
  Strategy --> Org["组织设计"]
  Org --> People["人才梯队"]
  Org --> Delivery["交付治理"]
  Strategy --> Tech["架构治理"]
  Delivery --> Quality["质量与稳定性"]
  Tech --> Platform["平台化能力"]
  People --> Culture["技术文化"]
  Quality --> Result["业务结果"]
  Platform --> Result
  Culture --> Result
  Result -.复盘.-> Biz

  subgraph Cross["横向能力"]
    Up["向上管理"]
    Coop["跨部门协同"]
    Cost["成本与预算"]
    Risk["安全与风险"]
    AI["AI 赋能研发"]
  end

  Up -.-> Strategy
  Coop -.-> Delivery
  Cost -.-> Strategy
  Risk -.-> Quality
  AI -.-> Platform

  classDef business fill:#fff3d6,stroke:#d99a1e,color:#3a2600;
  classDef core fill:#e9f8ee,stroke:#4a9f65,color:#10351d;
  classDef ops fill:#f1e9ff,stroke:#8b61d1,color:#261344;
  classDef risk fill:#ffe9e9,stroke:#d45b5b,color:#4a1010;
  class Biz,Result business;
  class Strategy,Org,People,Tech core;
  class Delivery,Quality,Platform,Culture,Up,Coop,Cost,AI ops;
  class Risk risk;
```

## 图的读法

- 主线是 `业务目标 -> 技术战略 -> 组织与交付 -> 业务结果`
- 中间层是技术管理者真正要设计的系统：组织、人才、交付、架构、平台、质量、文化
- 横向能力是管理者越往上越绕不开的接口：向上管理、跨部门协同、成本、安全、AI 赋能
