# AI 时代技术管理者能力迁移

## 核心判断

AI 时代，技术管理者不是简单“让团队用 AI 工具”，而是要重新设计研发、知识、质量、评测、安全和组织学习方式。

真正的变化是：

`个人效率工具 -> 团队工作流 -> 工程治理 -> 组织能力 -> 业务创新`

## 管理者要看的五层

### 1. 个人生产力

- AI coding assistant
- 文档总结
- 调研
- 测试生成
- 代码解释

风险：每个人自己玩，团队没有沉淀。

### 2. 团队工作流

- 需求澄清
- 技术方案草稿
- 代码 review
- 测试用例生成
- 知识库问答
- 事故复盘

风险：没有统一规范，输出质量不稳定。

### 3. 工程治理

- prompt / workflow versioning
- eval set
- bad-case library
- 权限和审计
- release gate
- 数据安全

风险：把 AI 当工具，不当系统。

### 4. 组织学习

- 团队最佳实践库
- 项目复盘自动整理
- 新人 onboarding
- 技术债和事故模式归档
- 管理者决策辅助

风险：知识只在聊天记录里，不能成为组织资产。

### 5. 业务创新

- AI 原生功能
- 智能客服
- 游戏运营助手
- 内容生产辅助
- 反作弊和安全分析
- 数据分析和策划辅助

风险：只追 demo，不做 ROI、质量和风险治理。

## 技术管理者的新能力

| 能力 | 过去 | AI 时代 |
|---|---|---|
| 研发效能 | 流程、工具、平台 | AI-assisted SDLC、agent workflow、自动化评测 |
| 知识管理 | Wiki、文档、培训 | RAG、知识助手、复盘沉淀 |
| 质量治理 | 测试、review、CI | eval、bad cases、AI 生成代码治理 |
| 安全治理 | 权限、漏洞、合规 | prompt injection、数据泄露、tool safety |
| 组织学习 | 经验传承 | 记忆、案例库、AI-assisted onboarding |

## 推荐实践

1. 先在个人和小组层面试点 AI coding / research / document workflow
2. 为 2-3 个高频流程建立 prompt、模板和评测样本
3. 建立 bad-case library，不让失败样例消失
4. 把 AI 使用规范写入工程流程，而不是靠口头约定
5. 再进入团队级 agent、知识库和研发平台改造
