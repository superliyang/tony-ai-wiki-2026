# Confluent

## 一句话定位

Confluent 是围绕 Apache Kafka 的 data streaming / event platform 公司，重点是把 Kafka、schema、connectors、stream governance 和托管运维做成企业级平台。

## 放在全景架构哪一层

- Streaming / event platform
- Kafka managed service
- Connectors
- Schema registry
- Stream governance
- Real-time data integration

## 为什么值得研究

- 它代表“事件流成为企业数据中枢”的路线
- 它适合理解 Kafka 从开源组件变成企业级 streaming platform 的过程
- 它是 batch-centric data platform 之外的重要实时数据平台样本

## 读它时重点看

- Kafka 的 event log 思想
- schema registry 和 data contract
- connectors 生态
- stream governance
- Flink / stream processing 结合
- 和 lakehouse / warehouse 的上下游关系

## 容易误解

- 不要把 Confluent 只当托管 Kafka
- 企业级 streaming platform 还包括 schema、connector、governance、monitoring、security 和 cloud operations
- 它不替代 warehouse / lakehouse，而是为它们提供实时事件轨道
