# Snowflake

## 一句话定位

Snowflake 是典型的 cloud data platform / data cloud 公司，强项是托管式数据仓库体验、SQL 分析、数据共享、治理和企业数据协作。

## 放在全景架构哪一层

- Storage / warehouse
- Compute / SQL engine
- Governance / access
- Data sharing
- AI / app serving 的上层能力

## 为什么值得研究

- 它代表“托管式 cloud warehouse / data cloud”路线
- 它把存储、计算、治理、共享和应用生态打包成企业级数据平台体验
- 它适合观察企业数据平台如何从 BI 扩展到 data apps 和 AI data workload

## 读它时重点看

- SQL 与 warehouse 体验
- compute / storage 分离后的成本模型
- data sharing / marketplace
- governance、policy、access control
- 和 Iceberg / open table / external data 的关系
- AI workload 如何接入既有企业数据

## 容易误解

- 不要把 Snowflake 只当“更快的数据库”
- 它的关键是企业数据平台体验，而不只是查询引擎
- 也不要默认它能替代所有 lakehouse / streaming / orchestration 层
