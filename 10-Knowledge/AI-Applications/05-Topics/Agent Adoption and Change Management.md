# Agent Adoption and Change Management

## 这个主题是什么

`Agent Adoption and Change Management` 关注的不是 agent 能不能做事，而是组织和个人会不会真的把它用起来，并把它纳入日常工作流。

## 为什么重要

- 很多 agent 产品最大的问题不是能力不够，而是没有进入用户习惯
- 只要工作方式发生变化，就会出现学习成本、信任问题、角色重划和流程摩擦
- adoption 往往决定 agent 是停留在试点，还是进入真实生产

## 常见 adoption 阻力

- 用户不知道什么任务适合交给 agent
- 用户不信任结果，不知道什么时候该接管
- agent 改变了原有的审批、协作和责任边界
- 输出太长、太慢、太不稳定，用户很难形成习惯

## 真正难的地方

- adoption 不是培训一次就结束
- 用户愿不愿意长期使用，取决于系统是否低摩擦、低惊吓、可预期
- 组织 adoption 还要考虑角色分工、审计、合规和绩效边界

## 推荐推进方式

- 先定义“哪些任务交给 agent 最划算”
- 先从低风险高频任务建立正反馈
- 让 agent 输出进入已有流程，而不是要求用户完全改工作方式
- 给用户清晰的成功范式：什么时候用、怎么接管、怎么纠错

## 关键指标

- weekly active usage
- repeated usage per workflow
- human override rate
- task completion with agent assistance
- time-to-trust / time-to-habit
