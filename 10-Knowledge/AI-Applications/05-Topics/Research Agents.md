# Research Agents

## 这个主题是什么

`Research Agents` 关注 agent 如何帮助人类完成信息搜索、资料筛选、来源比对、脉络整理和阶段性结论输出。

## 为什么这是高价值场景

- 研究任务天然是多步任务
- 常常需要搜索、浏览、提炼、回看和重新追问
- 结果虽然不一定完全自动化，但非常适合“agent 先做一轮，再由人判断”

## 最适合的任务类型

- 领域入门与地图搭建
- 产品/竞品调研
- 公司、论文、技术趋势跟踪
- 多来源信息整理与比较
- 周期性更新报告

## 真正难的地方

- 来源质量参差不齐
- agent 容易把“检索到的信息”误当成“已经验证的事实”
- 研究任务常常边界变化大，计划很容易漂移
- 人类很容易被冗长但低价值的输出淹没

## 推荐工作方式

- 先定义研究问题，再让 agent 扩展资料
- 让 agent 输出结构化摘要，而不是长篇流水账
- 关键结论要带来源、时间和不确定性说明
- 高价值研究任务适合“agent 首轮 + 人类二次判断”
