# Healthcare Agents

## 这个行业方向是什么

`Healthcare Agents` 指的是把 agent 用在临床文档、索赔处理、患者沟通、 care coordination、证据检索、 administrative workflows 和 clinical support 上的应用方向。

## 为什么这个方向很重要

- 医疗行业长期受行政负担过重、信息碎片化和专业人员时间不足影响
- 很多任务天然是 agent 适合的：总结、检索、结构化、流程协调、患者沟通辅助
- 一旦真正落地，价值不只是效率提升，还包括 burnout reduction 和 care quality improvement

## 为什么它比普通企业场景更难

- 隐私、合规、PHI/HIPAA、可追溯性要求非常高
- 输出质量不仅要看“是否有帮助”，还要看是否安全、是否符合组织政策、是否能支撑临床工作
- 医疗 agent 不能只靠通用 demo，上线必须依赖治理、评测、证据引用和明确的人类责任边界

## 适合先落地的任务

- 临床文档草拟与结构化
- 医疗记录摘要与检索
- 索赔与保险运营辅助
- 患者教育材料生成与改写
- care coordination 与 discharge workflow 辅助

## 关键价值指标

- documentation time reduction：文档耗时下降
- ticket / escalation resolution time：复杂工单与索赔问题处理时间下降
- clinician adoption：临床与运营团队采纳度
- quality and safety review outcomes：安全与质量评审结果
- burnout reduction：人员负担是否下降

## 行业深水区判断

- 医疗 agent 的长期胜负手不是“会不会答”，而是“是否能进入真实工作流且不破坏安全边界”
- evidence retrieval、organization policy alignment、access control、auditability 在这里都属于核心产品能力
- 很多最有价值的医疗 agent 其实是“受控工作流助手”，不是完全自动诊疗系统
