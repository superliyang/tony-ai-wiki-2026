# Cursor 产品卡

## 简介

`Cursor` 的独特性在于它不是单纯的 IDE autocomplete，也不是纯聊天 agent，而是把 IDE 内协作和异步 background agents 结合起来。

## 核心功能

- IDE-native coding assistant
- background agents 在远程环境里异步编辑和运行代码
- GitHub 连接、分支 handoff、环境配置

## 价值点

- 同一个产品里同时支持前台协作和后台异步任务
- 非常适合“你先开始，agent 接力继续跑”的开发节奏
- 对团队来说，background agents 让 coding workflow 更接近任务委派系统

## 使用场景

- 并行推进多个工程任务
- 让 agent 在远程环境里跑测试和修改代码
- 从 IDE 内发起、跟踪并接管 agent 任务

## 边界与注意点

- background agents 会自动执行命令，安全边界和 prompt injection 风险更高
- 需要更清晰的 repo、branch、环境和审批策略
- 对开发团队流程成熟度有要求

## 竞争与差异

- 和 `Claude Code` 比，它更强在 IDE-native + background agent 组合
- 和 `Devin` 比，它更像围绕开发者工作台的 agent，而不是独立“软件工程师”形态
