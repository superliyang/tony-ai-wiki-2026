# GitHub Copilot at Accenture

## 背景

很多团队都觉得 coding assistant “好像有用”，但真正难的是在企业环境里证明它是否真的值得规模化部署。

## 做了什么

- GitHub 与 Accenture 做了随机对照试验和后续 adoption 分析
- 既看开发者主观体验，也看安装、使用频率、建议接受等客观指标
- 不是只做 demo，而是把工程组织级 adoption 当成核心问题

## 结果

- 官方披露：90% 的开发者觉得工作更有满足感
- 官方披露：95% 的开发者更享受编码
- 官方披露：67% 的参与者每周至少使用 5 天
- 官方披露：81.4% 在拿到 license 当天就安装了扩展

## 为什么值得学

- 这个案例的价值不只在“提速”，而在于它给了我们一个企业级 agent ROI 的衡量方法
- 在工程场景里，satisfaction、adoption、suggestion acceptance 和 cycle-time 往往比单次 benchmark 更有意义

## 迁移启发

- 不要只看“代码写得快不快”，要看是否进入日常工作流
- 评估 coding agent 时，要同时看满意度、使用深度、质量与回归风险
- 如果没有组织级遥测与实验设计，就很难形成真正可信的 ROI 结论
