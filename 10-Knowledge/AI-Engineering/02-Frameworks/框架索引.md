## 2026-06 判断框架

现在选框架不能只问“哪个更强”，要按 4 个工程面切：

| 维度 | 该问的问题 | 相关入口 |
|---|---|---|
| Control Flow | 是固定 workflow、状态机，还是动态 agent planning | [[10-Knowledge/AI-Engineering/07-Topics/Agent、Workflow、Runtime 与 Harness：边界与组合关系]] |
| Runtime | 是否支持持久状态、恢复、并发、重试、回放 | [[10-Knowledge/AI-Engineering/07-Topics/Agent Runtime Architecture]] |
| Tool Boundary | 工具是 SDK、CLI、MCP server、browser 还是 desktop | [[10-Knowledge/AI-Engineering/07-Topics/动作面设计：Tools、CLI、MCP、Browser、Desktop 与 Channel]] |
| Safety Gate | 是否有 approval、sandbox、trace、eval、rollback | [[10-Knowledge/AI-Engineering/07-Topics/Agent Security、Sandbox 与 Approval Architecture]] |

## Protocol and Framework Split

| 类别 | 放什么 | 不放什么 |
|---|---|---|
| Framework | LangGraph、AutoGen、ADK、OpenHands 这类运行时或开发框架 | 单个公司产品宣传 |
| Protocol | MCP、A2A、tool calling schema、rich agent protocol | 未形成复用边界的内部脚本 |
| Harness | Codex、Claude Code、browser-use、computer use 等任务执行环境 | 单次 prompt 技巧 |
| Security Control | MCP server、skills、coding agent 的信任边界 | 泛泛“注意安全” |
