## 2026-06 Refresh Lens

本轮更新后，部署层不再只看“模型怎么 serve”，而是拆成 5 条线：

1. **Serving data plane**：关注 vLLM / TensorRT-LLM / Dynamo / Fireworks / Groq 这类推理数据面，核心问题是吞吐、延迟、KV cache、prefill/decode 分离和多模型路由。
2. **Agent runtime plane**：关注 background agents、long-running agents、tool execution、session replay、权限和任务恢复。
3. **Eval and release plane**：上线前必须有 task success、regression suite、security eval、human approval 和 rollback 条件。
4. **Tool security plane**：MCP server、CLI、browser、desktop、filesystem、secret access 都要进入工具权限边界。
5. **Incident learning plane**：每次 agent 失败不只写 postmortem，还要回灌到 eval dataset、release gate 和 playbook。
