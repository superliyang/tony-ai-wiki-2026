# 第 1 周启动任务：模型与本地推理闭环

这一周的目标不是学完大模型，而是建立一个可持续的学习闭环：

```text
问题 -> 实验 -> 记录 -> bad cases -> 判断 -> 下一步
```

## 本周核心问题

1. 模型输入输出到底经过了哪些步骤？
2. 本地 runtime 为什么会影响速度、质量、内存？
3. 我现在的 Mac 环境是否足够稳定，能不能支撑后续训练和微调？

## 必做任务

### 任务 1：确认环境

完成 [[第 0 章实操：把 Mac 变成 AI 实验室]]。

最低验收：

- `python` / `uv` / `git` 可用。
- `torch.backends.mps.is_available()` 返回可解释结果。
- `ollama list` 可用。
- `mlx_lm.generate --help` 可用。

### 任务 2：跑一个本地模型

用 `Ollama` 或 `MLX-LM` 跑一个小模型，不追求大。

记录：

- 模型名称。
- 模型大小。
- 运行方式。
- 速度体感。
- 内存占用。
- 3 个回答好的例子。
- 3 个回答坏的例子。

### 任务 3：解释一次生成链路

用自己的话写出这条链路：

```text
文本 -> tokenizer -> token ids -> embeddings -> transformer blocks -> logits -> sampling -> 文本
```

不要求数学完全严谨，但要能解释：

- tokenizer 做了什么；
- logits 是什么；
- temperature / top-p 改变了什么；
- context window 为什么有限；
- KV cache 为什么能加速长对话。

### 任务 4：建立第一份实验记录

复制 [[Mac AI 实验记录模板]]，新建一份本周实验记录。

建议标题：

```text
2026-06-21 Mac AI 实验 001：本地推理与生成链路
```

## 本周不要做什么

- 不要一上来微调。
- 不要同时装 10 个框架。
- 不要追求最大模型。
- 不要把 API demo 当成学习成果。

## 本周输出物

- 一份环境记录。
- 一份本地推理实验记录。
- 一张生成链路图或文字说明。
- 一个下周问题：训练循环到底如何改变模型参数？
