# 推理成本、服务层级与容量规划

## 目标

学会把一个 AI 产品 workload 映射成：

- serverless
- dedicated / on-demand
- reserved capacity
- batch / flex / async tier

之间的组合选择。

## 你应该回答的问题

1. 这是实时对话、后台任务，还是 nightly batch
2. 峰值和均值流量差多少
3. 是否有强 SLA
4. 上下文是否长、cache 是否可复用
5. 安全复核是否会引入额外延迟与成本

## 最小实验

选择 3 种 workload，分别做服务层级判断：

- 高频低风险 chat
- 中风险 coding agent
- 低频高负载 batch summarization

## 输出模板

- workload 描述
- latency 目标
- failure cost
- 预算边界
- 推荐 tier
- fallback 方案
- 是否需要 reserved capacity
