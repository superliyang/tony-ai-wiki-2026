## What Is Stronger Now

- AI Coding is no longer only a Claude Code product branch; it now has an explicit team rollout path and security decision map.
- AI Coding is now executable as a lab: choose a low-risk repo task, record the run, score it, and convert repeatable behavior into skill / subagent / hook / plugin assets.
- The engineering route now separates three jobs: teaching personal capability, rolling out team workflow, and governing permission/security boundaries.
- The Claude Code branch now has an engineering lifecycle, not just a product note: intake, repo map, plan, edit, verify, review, PR, retrospective, and capability promotion.
- The harness branch now has a direct bridge from product ecosystem to team practice: `skills / plugins / subagents / hooks / MCP / permissions / release gates`.
- The Mac AI path now has a clear restart surface: open the workbench, choose an `E0-E12` experiment, copy the experiment template, run one bounded test, and write down the judgment.
- The path is now more useful for systematic play: it explicitly separates fast local inference, runtime comparison, quantization, PyTorch MPS, MLX, LoRA, eval, RAG, agent, and cloud mapping.
- `AI-Engineering` is now much more balanced across training, inference, security, harness, and runtime.
- The runtime branch now covers not just architecture, but also failure patterns, rollout controls, and recovery thinking.
- Memory is no longer buried as one subsection; it now has its own engineering spine around state, durable memory, artifacts, consolidation, evaluation, forgetting, correction, and promotion-to-skill flows.
- `Agent Runtime Lab` now supports both structural study and postmortem / release-gate exercises, and `Memory Lab` adds the hands-on memory design side.
- `Memory Lab` is now not just about memory theory; it also contains a real local plugin line that can replay incidents and test promotion gates.
- The evaluation layer is now less fragmented: there is a clearer path from `Agent Evaluation and Reliability` to `Eval Harness` to `online feedback` to actual open-source tooling choices.
- It is also more operational now: there are actual templates, checklists, and starter packs instead of only conceptual topic pages.
- There are now also three more realistic starter case packs, so the branch is much closer to something a team could adapt directly.
