# 领域专家感学习法

这个 playbook 用来训练两种能力：

1. **专家问题感**：问出让内行觉得你懂系统结构、变量、权衡和边界的问题。
2. **领域结构感**：用一张图快速看懂一个领域的目标、角色、流程、指标、问题、解法和失败模式。

## Core Rule

```text
先建立领域结构，再生成专家问题。
先问权衡和边界，再问概念和工具。
```

## Expert Question Formula

一个专业问题通常包含 5 个要素：

| Element | Question |
|---|---|
| 对象 | 我问的是哪个系统、场景、角色或流程？ |
| 变量 | 影响结果的关键指标和约束是什么？ |
| 权衡 | 哪些目标会互相冲突？ |
| 边界 | 这个判断在什么条件下成立，什么时候失效？ |
| 演化 | 规模、阶段、市场、技术变化后，答案会不会变？ |

模板：

```text
在【具体场景】下，如果目标是【核心结果】，你们如何在【指标 A】、【指标 B】、【风险 C】之间取舍？
这个策略在【阶段/规模/市场】变化后，最先失效的假设是什么？
```

## Domain Master Map

每个领域都应该能压缩成一张图：

```text
领域目标
  -> 关键角色
  -> 核心对象
  -> 关键流程
  -> 核心指标
  -> 关键问题
  -> 解决方案族
  -> 失败模式
  -> 专家问题
  -> 学习路径
```

## 10 Expert Question Types

| Type | Question Pattern |
|---|---|
| 目标问题 | 这个领域最终优化什么结果？ |
| 角色问题 | 谁参与，谁付钱，谁承担风险，谁做决策？ |
| 流程问题 | 价值、信息、资金、风险如何流动？ |
| 指标问题 | 哪 3 个指标最能代表系统健康？ |
| 权衡问题 | 哪些指标不能同时最大化？ |
| 分层问题 | 新手看到的是表面问题，高手看到的底层问题是什么？ |
| 失败问题 | 最常见、最昂贵、最隐蔽的失败模式是什么？ |
| 策略问题 | 主流解法有哪些，各自适合什么约束？ |
| 组织问题 | 哪个团队拥有这个问题，跨团队摩擦在哪里？ |
| 演化问题 | 当规模、市场、监管、技术变化时，什么假设会过期？ |

## Output Package

学习一个领域时，最终沉淀 5 个产物：

1. `领域全貌图`
2. `专家问题清单`
3. `新手误区`
4. `高手权衡`
5. `一页复习卡`
