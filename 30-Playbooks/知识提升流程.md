# 知识提升流程

## Rule

```text
AI can create candidates.
Tony judges direction.
Codex crystallizes accepted material.
Canonical knowledge lives in reviewed Markdown.
```

## Flow

1. Capture raw signal into `00-Inbox-AI/signals/`.
2. Summarize and score it into `00-Inbox-AI/candidates/`.
3. Put judgment items into `00-Inbox-AI/review-queue/pending/`.
4. Tony chooses `study`, `watch`, `discard`, `promote`, or `build`.
5. Codex promotes accepted material into `10-Knowledge/`, `20-Maps/`, `30-Playbooks/`, or `90-Agent-System/`.
6. Update shared memory when the decision changes future behavior.
7. Preserve meaningful changes with Git.

## Quality Gate

Before promotion, ask:

- Will this be reused?
- Does it belong to a clear domain or workflow?
- Is it stable enough to preserve?
- Does it contain sensitive information?
- Does it improve Tony's future thinking or AI capability?
