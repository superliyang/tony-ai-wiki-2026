## Directory Meaning

| Path | Meaning | Write Rule |
|---|---|---|
| `00-Home/` | 人类每日入口和当前主线 | 改结构或重点时同步更新 |
| `00-Inbox-AI/` | 历史 AI staging、候选、报告、共享记忆 | 2026-06-16 后冻结；新内容去 working vault |
| `10-Knowledge/` | 正式知识资产 | 需要 review 或明确请求 |
| `10-Knowledge/*` formal domains | extracted legacy and current knowledge assets | use as normal reading surface |
| old import folder | removed after extraction validation | do not recreate |
| `20-Maps/` | 跨领域地图和导航 | 结构变化时更新 |
| `30-Playbooks/` | 人类和 AI 都能复用的方法 | 稳定流程才进入 |
| `40-Projects/` | 项目、迁移和构建工作 | 项目内自带状态 |
| `60-Agents/` | 工具角色和边界 | 工具职责变化时更新 |
| `90-Agent-System/` | Agent 能力资产、协议、规则、脚本说明 | AI 协作机制变化时更新 |
| `output-feishu/` | 清洗后的飞书可发布中间层 | 只放可发布 projection，不做主库 |
| `output-public/` | 泛化公开/半公开输出说明 | 飞书链路优先用 `output-feishu/` |

## Before Editing

1. Identify the layer.
2. Check whether the content belongs in the main vault or a future split vault via [[90-Agent-System/decisions/2026-06-16-vault-boundary-split]].
3. Read the nearest README or map.
4. Preserve source links when promoting imported material.
5. Update maps when navigation changes.
6. Keep runtime files, secrets, and app state out of the vault.
