# 命名规范

## Language

- 面向 Tony 阅读的说明优先中文。
- 技术术语保留英文，例如 AgentOps、LLMOps、RAG、workflow、review queue。
- 旧库导入内容保留原名，不为了统一而批量改名。

## Folder Names

- Top-level folders use numbered English names: `10-Knowledge/`, `20-Maps/`.
- Domain folders use stable English identifiers: `AI-Engineering/`, `International-Payments/`.
- Human-facing notes can use Chinese names: `专题总览.md`, `学习路径.md`, `恢复笔记.md`.

## Note Types

| Type | Preferred Name |
|---|---|
| Domain overview | `专题总览.md` |
| Learning path | `学习路径.md` |
| Topic index | `主题索引.md` |
| Map | `地图.md` or a specific map name |
| Progress memory | `学习进度.md` |
| Resume memory | `恢复笔记.md` |
| Project entry | `README.md` |
| Workflow | concise action name under `90-Agent-System/workflows/` |

## Links

- Use Obsidian wikilinks for internal notes.
- Link to imported source notes when promoting old material.
- Prefer stable overview pages in public navigation instead of deep imported files.

## Status Values

Use simple status values in frontmatter:

- `seed`: created but not yet mature.
- `active`: currently useful.
- `imported`: preserved from old source.
- `draft`: incomplete and not canonical.
- `archived`: kept for history, not current.
