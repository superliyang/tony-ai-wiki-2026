# Applications Spine

Use this spine when expanding AI agent applications.

## Sequence

1. application topic
2. industry card
3. workflow or product card
4. case study
5. ROI assessment template or filled assessment
6. failure-review note when the deployment risks are meaningful

## Questions to answer

- what task is being improved
- why agent is a better fit than plain chat or plain automation
- what tools or systems are connected
- where approvals happen
- what metrics prove value
- what errors are acceptable versus unacceptable
- what organizational change is required

## Common application buckets

- customer support
- enterprise operations
- software engineering
- sales and revenue
- research and analysis
- personal assistant workflows

## Common failure buckets

- task boundary too wide
- no single source of truth
- missing approval gates
- weak evaluation
- no ROI proof
- adoption collapse because outputs are untrusted
