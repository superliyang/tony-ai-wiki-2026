# Project Intake Checklist

For each project, answer:

- What problem does it solve?
- Why does it matter now?
- Which stack layer does it belong to?
- Is it a `底座`, `壳层`, `平台`, or `子系统`?
- How does it work internally?
- What does its core architecture or operating model look like?
- What are the main components, objects, or abstractions?
- What is the main request path or data path?
- What architecture diagram or request/dataflow diagram should be drawn?
- What adjacent projects should it be compared with?
- Is it suitable for local experiments, production study, or both?
- What hidden costs, operational constraints, or risks deserve explicit mention?
- What is the next experiment or reading move?
