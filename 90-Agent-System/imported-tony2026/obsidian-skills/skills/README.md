## 新的默认工作方式

现在这几套核心 skill 默认都支持 `autonomous batch delivery`：

- 当用户说“继续做”“中间不用打断”“你自己判断下一步”时，skill 会默认进入连续推进模式
- 它会先选一个边界清晰、杠杆高的 slice
- 做完整个 slice 后再反思下一步，而不是每补一页就停
- 只有遇到高风险决策、拓扑大改或自然的版本边界时，才会停下来汇报

## 推荐搭配方式

### 研究新领域

1. 先用 `obsidian-domain-research-graph`
2. 再配合 `obsidian-learning-methodology`
3. 最后用 `knowledge-vault-research-sync` 做持续维护和 Git 同步

### 维护已有知识库

1. 先扩写内容和索引
2. 更新 `学习进度.md` 与 `恢复笔记.md`
3. 再用 `knowledge-vault-research-sync` 做校验与 checkpoint

### 持续推进而不中断

1. 用 `obsidian-learning-methodology` 决定 batch 的边界
2. 用具体领域 skill 执行内容扩展
3. 用 `knowledge-vault-research-sync` 在一个自然版本边界做校验与提交

## 这套方法最适合什么场景

- 深入研究一个新领域
- 建立实体 / 主题 / 系统 / 地图分层
- 用 Obsidian 持续沉淀知识图谱
- 支持中断后快速恢复
- 需要连续做几轮自我探索、反思和收口
- 需要定期更新并同步到 GitHub
