# International Payments Learning Skill

Use this skill when the user wants to build, expand, or maintain an Obsidian-based learning system for international payments.

## Core Goal

Turn payments knowledge into a durable study system that supports:

- business-first learning
- Chinese-first notes with key English terms retained
- expert capability maps
- practical cases
- dashboard and reporting views
- progress tracking and interruption recovery
- business and technical expert tracks
- team onboarding and role-based learning paths
- market-case learning for region-specific judgment
- country deep dives for priority markets
- scenario playbooks for concrete business models
- combo cases for country x scenario decisions
- operating runbooks for real incident and management response
- incident cases for country x scenario x failure-pattern learning
- advanced owner-level layers: methods, compliance ops, treasury, data governance, management governance, commercial procurement, and card-payment deep dives

## Workflow

1. Start from the business problems, not infrastructure history.
2. Build or update the domain `专题总览.md`, `主题索引.md`, `学习进度.md`, and `恢复笔记.md`.
3. Add role-based onboarding pages when the domain is mature enough.
4. Add market cases and country deep dives.
5. Add scenario playbooks and combo cases.
6. Add runbooks and incident cases when the user needs to operate and respond.
7. Add advanced governance and commercial layers when the user wants to move from understanding to owner-level management.
8. Pair learning pages with templates, dashboards, thresholds, and action libraries whenever useful.
9. Before ending a study session, update progress and resume notes.
