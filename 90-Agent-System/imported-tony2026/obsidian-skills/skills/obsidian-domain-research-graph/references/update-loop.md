# Update Loop

## Stable vs Volatile Content

Separate notes into two broad classes:

- stable: foundational concepts, enduring distinctions, mature maps
- volatile: company landscapes, product comparisons, news-driven areas

Stable pages should change slowly.
Volatile pages should have clear update points.

## Recommended Refresh Triggers

Refresh a domain graph when:

- a new major entity enters the field
- the topology becomes confusing
- a comparison page no longer reflects the current landscape
- the recommended learning path changes
- progress and resume notes point to pages that no longer matter

## Periodic Maintenance Pass

A good maintenance pass usually does this:

1. scan the domain indexes
2. identify missing or stale entity pages
3. refresh maps that no longer match reality
4. update progress and resume notes if the path changed
5. checkpoint to Git if changes are meaningful
