# Layered Vault Pattern

## Default Pattern

- Foundations
- Entities
- Topics
- Systems
- Maps
- Templates
- Progress
- Resume

## Use This Pattern When

- the domain has both concepts and concrete products
- the user wants to pause and resume often
- the field changes over time and needs maintenance

## Common Mistakes

- mixing products into topic folders
- putting all navigation into one giant README
- skipping progress and resume notes
- scaling note count before deciding relationships
