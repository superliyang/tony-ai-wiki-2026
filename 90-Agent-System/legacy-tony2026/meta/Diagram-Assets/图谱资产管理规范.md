# 图谱资产管理规范

> 这份规范用于管理 ChatGPT、Mermaid、Excalidraw、Obsidian Canvas、SVG、PNG 等图谱资产。目标不是收藏图片，而是把图变成可维护、可复用、可检索、可演进的知识资产。

## 核心原则

1. **Markdown 说明页是入口**：任何正式图谱都必须有一页 `.md` 解释它回答什么问题、怎么读、和哪些知识页关联。
2. **源文件优先于图片**：Mermaid / Excalidraw / Canvas / SVG / Prompt 是可维护资产，PNG/JPG 只是展示资产。
3. **一图一问题**：每张图只回答一个核心问题；太复杂就拆成全景图 + Drill-down 图。
4. **版本化而不是覆盖**：重要图谱按日期或版本保存，旧图标记 `deprecated`，不要无记录覆盖。
5. **索引化**：每张正式图都要进入对应领域的 `地图索引.md`，必要时也进入 `专题总览.md`。
6. **资产与知识分离**：图片、源文件、提示词放在 `_assets/`，知识解释放在地图说明页。

## 推荐目录结构

每个领域使用自己的地图目录，内部统一放一个 `_assets`：

```text
01-Areas/<Domain>/06-Maps/
  地图索引.md
  某某全景架构图.md
  某某链路图.md
  _assets/
    images/
      某某全景架构图-20260515-v1.png
      某某全景架构图-20260515-v2.png
    sources/
      某某全景架构图-20260515-v1.mmd
      某某全景架构图-20260515-v1.excalidraw
      某某全景架构图-20260515-v1.canvas
      某某全景架构图-20260515-v1.svg
    prompts/
      某某全景架构图-20260515-v1.prompt.md
```

> 少数领域不是 `06-Maps`，例如 `AI-Learning/07-Maps`、`AI-Engineering/08-Maps`、`Cloud-Native/04-Maps`。这些领域沿用现有目录，只在对应 Maps 目录下加 `_assets`。

## 文件命名规则

### Markdown 说明页

```text
<图谱主题>.md
```

示例：

- `安全上帝视角全景架构图.md`
- `手机游戏端到端安全架构图.md`
- `AI 全景架构图.md`

### 图片和源文件

```text
<图谱主题>-YYYYMMDD-vN.<ext>
```

示例：

- `安全上帝视角全景架构图-20260515-v1.png`
- `安全上帝视角全景架构图-20260515-v1.mmd`
- `安全上帝视角全景架构图-20260515-v1.prompt.md`

### 变体命名

如果同一主题有不同用途：

```text
<图谱主题>-<用途>-YYYYMMDD-vN.<ext>
```

示例：

- `AI全景架构图-演示版-20260515-v1.png`
- `AI全景架构图-学习版-20260515-v1.mmd`
- `安全全景图-领导汇报版-20260515-v1.png`

## 每张图的生命周期

### 1. Capture：收进来

从 ChatGPT 或其他工具生成图后，至少保存：

- 图片：放到 `_assets/images/`
- 提示词：放到 `_assets/prompts/`
- 如果有可编辑源文件：放到 `_assets/sources/`

### 2. Explain：写说明页

每张正式图必须有 Markdown 说明页，至少包含：

- 这张图回答什么问题
- 适合谁看
- 怎么读
- 关键分层/链路
- 图片或源文件链接
- 关联知识页
- 下一步 Drill-down

### 3. Index：进索引

更新最近的：

- `地图索引.md`
- 领域 `专题总览.md` 的快速入口或图谱导航
- 如果图改变学习路径，也更新 `学习进度.md` / `恢复笔记.md`

### 4. Refine：沉淀成可维护版本

如果 ChatGPT 生成的是图片，建议再转成一种可维护版本：

- Mermaid：适合流程、架构、能力图，版本控制友好。
- Excalidraw：适合手绘风、汇报、演示。
- Canvas：适合 Obsidian 交互式知识导航。
- SVG：适合精修视觉资产。

### 5. Deprecate：旧图退役

旧图不要删除，除非确定无价值。做法：

- 文件名保留版本号。
- Markdown 说明页里标记 `status: deprecated` 或在版本记录里写明。
- 地图索引中移动到“历史版本 / 旧图”。

## 图谱类型

| 类型 | 用途 | 推荐格式 |
|---|---|---|
| 全景图 | 上帝视角，建立领域坐标 | Mermaid / PNG / Excalidraw |
| 架构图 | 系统分层、组件关系、控制点 | Mermaid / Excalidraw / SVG |
| 链路图 | 数据流、攻击链、业务流程 | Mermaid |
| 能力地图 | 团队、平台、能力边界 | Mermaid / Canvas |
| 决策图 | 选型、治理、路线判断 | Mermaid |
| 运营闭环图 | SecOps、DataOps、MLOps、风控响应 | Mermaid |
| 路线图 | 学习路线、建设路线、迁移路线 | Mermaid / Canvas |
| 汇报图 | 给领导或跨团队沟通 | PNG / Excalidraw / SVG |

## 状态标记

Markdown 说明页 frontmatter 推荐：

```yaml
---
title:
type: diagram
diagram_type: panorama-map
status: active
domain:
version: v1
created:
updated:
source_image:
source_file:
prompt_file:
---
```

`status` 建议：

- `draft`：草稿，还没接入索引。
- `active`：当前推荐版本。
- `needs-review`：可用但需要复核。
- `deprecated`：旧版本，仅保留历史。

## 什么时候需要更新图

需要更新图谱的触发条件：

- 新增重要主题、系统、平台或案例。
- 领域结构发生变化。
- 用户反馈“看不懂 / 太丑 / 太密 / 关系不清”。
- 旧图与当前知识库内容不一致。
- 从图片版沉淀出了 Mermaid / Excalidraw / Canvas 可维护版。

不需要更新图谱的情况：

- 只是新增一个很小的 note。
- 只是补充正文细节，不改变结构。
- 没有改变读者进入领域的路径。

## ChatGPT 生成图的接收流程

当你在 ChatGPT 里生成了一张图：

1. 先从 [[图谱提示词库：全景图、明细图与资产归档]] 复制合适提示词。
2. 导出图片，优先 PNG 或 SVG。
3. 复制生成提示词，保存为 `.prompt.md`。
4. 如果 ChatGPT 给了 Mermaid / JSON / Excalidraw 源码，同时保存。
5. 放入对应领域的 `_assets` 子目录。
6. 新建或更新图谱说明页。
7. 更新地图索引。
8. 如果图用于全局导航，再更新 `专题总览.md`。

## 全领域地图索引要求

每个正式领域的 `Maps/地图索引.md` 都应该包含：

- 指向本规范的链接。
- 指向 [[图谱资产清单]] 的链接。
- 指向 [[图谱提示词库：全景图、明细图与资产归档]] 的链接。
- 本领域 `_assets/` 目录位置。
- 当前地图、候选地图和地图读法。

如果某个领域没有 `地图索引.md`，新建地图或接收图片资产时应先补上。

## 推荐协作方式

你可以把图片拖进项目，或告诉我图片路径。我会按这个流程处理：

1. 判断图属于哪个领域。
2. 给它命名和归档。
3. 建 Markdown 说明页。
4. 把图接入 `地图索引.md` 和 `专题总览.md`。
5. 如果需要，我再把图片内容重构成 Mermaid / Excalidraw / Canvas。
6. 校验 wikilinks。
7. 提交 Git。

## 常见错误

- 只存 PNG，不存提示词和源文件。
- 一张图塞太多内容，无法维护。
- 图放进文件夹但没有说明页和索引入口。
- 每次重生成都覆盖旧图，导致无法追溯。
- 不区分“学习版”“汇报版”“细节版”。
- 图里用了大量小字，移动端和 Obsidian 里看不清。
