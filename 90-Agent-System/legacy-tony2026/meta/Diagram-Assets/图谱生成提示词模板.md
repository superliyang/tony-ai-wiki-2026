# <图谱标题> Prompt

## 目标

生成一张 <全景图/架构图/链路图/能力地图/决策图/运营闭环图/路线图/汇报图>，回答：

> <这张图要回答的问题>

## 受众

- <领导 / 架构师 / 工程师 / 安全团队 / 学习者>

## 输出格式

- 目标格式：Mermaid / Excalidraw / Obsidian Canvas / PNG / SVG
- 画布比例：16:9 / A4 / 长图
- 风格：专业、简洁、低密度、可汇报

## 内容约束

- 第一层模块不超过 8 个。
- 每个节点 4-10 个中文字符或 2-4 个英文词。
- 不要箭头毛线团。
- 复杂内容拆成 Drill-down。
- 颜色要有语义：基础、核心、运行、风险、治理、业务。

## 生成提示词

```text
<在 ChatGPT 或图片工具里使用的完整提示词>
```

## 负面约束

```text
不要使用过多小字。
不要把所有细节塞进一张图。
不要生成不可读的密集网络。
不要使用花哨但无语义的颜色。
```

## 生成质量检查

- 这张图是否只回答一个核心问题？
- 第一层模块是否控制在 4-8 个？
- 读者是否能在 5 秒内找到阅读路径？
- 是否有语义颜色，而不是装饰颜色？
- 是否有推荐 Drill-down，而不是把所有细节塞进主图？
- 是否能保存图片、源文件和 prompt 三类资产？
