# 怎么使用图谱资产管理规范

> 这页是给日常操作用的：你在 ChatGPT 里生成了一张图，应该怎么放、怎么命名、怎么让我维护。

## 最简单用法

如果你不想自己整理，直接把图片给我，并告诉我：

```text
这张图属于哪个领域：AI / Security / Big-Data / Cloud-Native / Macro-Insight / Skills-Gaming / 其他
这张图想解决什么问题：
这是全景图、架构图、链路图、能力图、决策图、路线图还是汇报图：
是否有生成提示词或 Mermaid / Excalidraw 源文件：
```

我会负责：

1. 放到对应领域的 `_assets/images/`
2. 保存 prompt 到 `_assets/prompts/`
3. 保存源文件到 `_assets/sources/`
4. 建或更新图谱说明页
5. 更新 `地图索引.md`
6. 必要时更新 `专题总览.md`
7. 校验 wikilinks
8. Git 提交

如果你还没生成图，先打开 [[图谱提示词库：全景图、明细图与资产归档]]，复制对应类型的提示词：

- 全景图：用 `Mermaid 全景图提示词`
- 细节图：用 `Mermaid 明细架构图提示词`
- 汇报图：用 `Excalidraw 汇报图提示词`
- 可点击导航：用 `Obsidian Canvas 导航图提示词`
- 图片海报：用 `图片模型全景图提示词`

## 你自己放图时怎么放

### Step 1：判断领域

| 图类型 | 推荐目录 |
|---|---|
| AI 全景 / Agent / 模型 / AI 产品 | `01-Areas/AI-Learning/07-Maps/_assets/` |
| AI 工程 / LLMOps / AgentOps | `01-Areas/AI-Engineering/08-Maps/_assets/` |
| 安全 / 撞库 / AppSec / SecOps | `01-Areas/Security/06-Maps/_assets/` |
| 大数据 / Lakehouse / 实时链路 | `01-Areas/Big-Data/06-Maps/_assets/` |
| 云原生 / K8s / 平台工程 | `01-Areas/Cloud-Native/04-Maps/_assets/` |
| 宏观认知 / 世界模型 / 趋势 | `01-Areas/Macro-Insight/06-Maps/_assets/` |
| 游戏设计 / 游戏开发 / 游戏技术 | `01-Areas/Skills-Gaming/06-Maps/_assets/` |

### Step 2：按资产类型放

```text
_assets/images/   放 PNG、JPG、SVG 导出图片
_assets/sources/  放 Mermaid、Excalidraw、Canvas、SVG 源文件
_assets/prompts/  放 ChatGPT / image model 生成提示词
```

### Step 3：按规范命名

```text
<图谱主题>-YYYYMMDD-v1.png
<图谱主题>-YYYYMMDD-v1.mmd
<图谱主题>-YYYYMMDD-v1.prompt.md
```

示例：

```text
AI全景架构图-20260515-v1.png
AI全景架构图-20260515-v1.prompt.md
AI全景架构图-20260515-v1.mmd
```

### Step 4：建说明页

复制模板：

- [[图谱说明页模板]]

说明页放在对应 Maps 目录下，例如：

```text
01-Areas/Security/06-Maps/安全全景架构图.md
```

说明页要回答：

- 这张图解决什么问题？
- 谁该看？
- 怎么读？
- 有哪些核心模块？
- 关联哪些知识页？
- 下一步 drill-down 看什么？

### Step 5：更新索引

至少更新：

```text
<Domain>/06-Maps/地图索引.md
```

如果这张图是主入口，还要更新：

```text
<Domain>/专题总览.md
```

如果它改变学习路径，再更新：

```text
<Domain>/学习进度.md
<Domain>/恢复笔记.md
```

## 图片、Mermaid、Excalidraw、Canvas 怎么选

| 目标 | 推荐 |
|---|---|
| 版本控制和长期维护 | Mermaid |
| 演示和好看 | Excalidraw / SVG / PNG |
| Obsidian 交互式导航 | Canvas |
| 领导汇报封面或概念海报 | PNG / SVG |
| 复杂领域学习路径 | Markdown 说明页 + Mermaid + Canvas |

## 我建议的默认流程

日常最省心的方式：

1. ChatGPT 先生成图片。
2. 把图片和提示词给我。
3. 我先归档 PNG + prompt。
4. 如果图值得长期维护，我再转成 Mermaid 或 Excalidraw。
5. 进入地图索引和专题总览。

也就是说：

```text
先收进来 -> 再解释 -> 再索引 -> 再沉淀源文件
```

## 什么图值得收进 vault

值得收：

- AI / 安全 / 大数据 / 游戏等领域全景图。
- 能帮你给领导解释方案的架构图。
- 能帮团队分工的能力地图。
- 能指导落地的链路图、流程图、运营闭环图。
- 能长期复习的学习路线图。

不太值得收：

- 只是好看的海报，没有知识结构。
- 一次性随手草图。
- 内容和已有图重复，但没有更清楚。
- 文字密到看不清，也没有源文件。

## 给我的最佳输入格式

你可以这样说：

```text
我把几张图放到 /path/to/images 了：
1. ai-panorama.png：AI 全景图，适合放 AI-Learning
2. security-architecture.png：安全架构图，适合放 Security
3. bigdata-lakehouse.png：大数据架构图，适合放 Big-Data

请按图谱资产规范归档，建立说明页，更新地图索引。
```

如果还有提示词：

```text
每张图旁边有同名 .prompt.md，帮我一起保存。
```

如果想重画：

```text
这张图太丑，帮我转成更清晰的 Mermaid / Excalidraw。
```
