# 文档成熟度补强标准

## 目标

这份标准用于判断 vault 里的文档是否“足够可学习、可恢复、可决策、可落地”。

它不是要求每篇都写很长，而是避免关键文档只有标题和几个空段落，导致后续无法复用。

## 成熟度分级

| 等级 | 名称 | 判断 |
|---|---|---|
| L0 | 占位 | 只有标题或目录，无法学习 |
| L1 | 简介 | 说明了是什么，但缺少结构 |
| L2 | 可学习 | 有背景、核心概念、边界和相关链接 |
| L3 | 可决策 | 有取舍、适用场景、风险、判断标准 |
| L4 | 可落地 | 有步骤、模板、验收、复盘、案例 |
| L5 | 可运营 | 有指标、周期、责任、更新机制和长期维护方式 |

## 不同文档的最低要求

### 主题页

最低应达到 `L2`，核心结构：

- 这个主题是什么
- 为什么重要
- 核心概念
- 易混边界
- 典型场景
- 相关系统 / 项目 / 图谱
- 下一步学习路径

### 系统 / 产品页

最低应达到 `L2-L3`，核心结构：

- 它解决什么问题
- 为什么现在值得关注
- 核心能力
- 架构 / 工作原理
- 适用场景
- 和相邻系统的差异
- 风险与边界
- 官方入口

### 开源项目页

最低应达到 `L3`，核心结构：

- 它解决什么问题
- stack 位置
- 核心组件与对象模型
- 主流程
- 工程质量观察
- 和相邻项目差异
- 本地 / 生产适配度
- 下一步实验

### 地图页

最低应达到 `L2`，核心结构：

- 图谱意图
- 图谱本体
- 怎么读
- 关键洞察
- 推荐钻取
- 相关页面

### 模板页

最低应达到 `L4`，核心结构：

- 适用场景
- 使用前提
- 可复制结构
- 必填字段说明
- 验收标准
- 常见误用
- 相关模板

### 项目 / Lab 页

最低应达到 `L4`，核心结构：

- 项目定位
- 目标与非目标
- 文档清单
- 实验批次
- 记录模板
- 验收标准
- 风险与退出条件

### 索引页

最低应达到 `L2`，核心结构：

- 当前分组
- 推荐阅读顺序
- 每组入口说明
- 如何使用这一层
- 关联上级 / 下级索引

## 补强优先级

优先补：

1. `专题总览.md`
2. `学习进度.md`
3. `恢复笔记.md`
4. 地图索引 / 主题索引 / 项目索引
5. 高复用模板
6. 当前主线的主题页、项目页、系统页
7. 长尾资料页

## 批量审查方法

### 先找薄文档

使用行数、字符数、标题密度找候选：

```bash
python3 - <<'PY'
from pathlib import Path
for p in Path("01-Areas").rglob("*.md"):
    txt = p.read_text(errors="ignore")
    lines = txt.count("\n") + 1
    if lines < 80:
        print(lines, p)
PY
```

### 再判断是否值得补

不是所有短文档都必须补。优先问：

- 是否处在当前主线
- 是否是入口页
- 是否是模板 / 索引 / 地图
- 是否影响多个后续文档质量
- 是否能提升恢复和决策效率

## 补强后的验收

- 能不能让一个月后的自己快速恢复
- 能不能支持一个具体决策
- 能不能指导一个具体动作
- 有没有连接到最近的索引 / 地图
- 有没有必要的风险和边界
- wikilink 是否有效
