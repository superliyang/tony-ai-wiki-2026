# 广告投放 · 一页复习卡

---

## 一句话定义

**从程序化广告和移动 UA 视角看，广告投放是在极短时间内完成的多方匹配和 ROI 决策系统。** 广告主想便宜买高质量用户，媒体想高价卖每次曝光，中间平台做拍卖设计。核心能力是**在信号持续损失的环境下做可测量的 ROI 决策。**

---

## 生态系统（5 个角色，1 条链路）

```
广告主 → DSP → ADX ← SSP ← 媒体
              ↕
            DMP/MMP/验证
```

| 角色 | 要什么 | 看什么 |
|---|---|---|
| 广告主 | 最低成本获取高价值用户 | ROAS, CPA, LTV |
| DSP | 高效花掉广告主预算 | 胜率, eCPM, 响应延迟 |
| ADX | 高效撮合 | 撮合率, 吞吐量, 延迟 |
| SSP | 最大化每次曝光收益 | 填充率, eCPM, 底价策略 |
| 媒体 | 变现流量 | RPM, 用户体验 |

**通信协议：OpenRTB | 竞拍耗时：<100ms | DSP 决策：<30ms | 规模：百亿级请求/天**

---

## 核心指标漏斗

```
曝光 Impressions
  │ CPM = Cost / Imp × 1000
  ▼
点击 Clicks
  │ CTR = Clicks / Imp  │  CPC = Cost / Clicks
  ▼
转化 Conversions (安装/购买/注册)
  │ CVR = Conv / Clicks  │  CPA = Cost / Conv
  ▼
收入 Revenue
  │ ROAS = Revenue / Cost  │  LTV = 用户终生价值
  ▼
回收 Payback Period = LTV > CPA 所需天数 ← 真正的底线指标
```

**移动游戏关键数字要按品类、渠道、地区和现金约束校准：D30 ROAS、LTV/CPI 和 Payback Period 应一起看，不能只看 CPI 或短窗口 ROAS。**

## 三种交易模式

| | Open Exchange | PMP | Programmatic Guaranteed |
|---|---|---|---|
| CPM | $1–4 | $5–15 | $10–30 |
| 欺诈率 | 8–15% | 2–5% | <1% |
| 可见率 | 50–60% | 70–85% | 80–92% |
| 适用 | 量最大化 | 质+量平衡 | 品牌安全优先 |

**质量调整后的真实成本：PMP 往往比 Open Exchange 更优。**

---

## 三板斧（应对信号损失）

```
1. 服务端归因
   Meta CAPI → 恢复 20–35% 信号
   Google EC → 恢复 5–15% 信号
   目标：EMQ >8 (Meta), Match Rate High (Google)

2. 增量测试
   Geo Holdout → 季度一次 → 测 Top 3 渠道
   平台 ROAS × 0.5–0.7 ≈ 真实 ROAS

3. SKAN 信号工程
   6-bit Fine Value → 用早期行为（非收入）预测 LTV
   Coarse 必须与 Fine 对齐
```

## 五个致命坑

| # | 坑 | 后果 |
|---|---|---|
| 1 | 只看 CPI 不看 LTV | 便宜安装=垃圾用户，越放越亏 |
| 2 | 信平台 ROAS 不校准 | 高估 30–100%，预算全押在假象上 |
| 3 | 短窗口（D7）达标就放量 | D90 可能亏损，Payback 无限拉长 |
| 4 | Cookie 时代惯性不改 | 拿着不对的数字做对的决策 |
| 5 | UA/产品/财务 KPI 不统一 | 三方全"达标"但公司在亏钱 |

---

## 一个专家级决策框架

```
放量决策 = 渠道 D30 ROAS × 现金约束 × 增量校准

当 D30 ROAS < 40%：
  ① 拆分渠道看增量 ROAS（不是平台 ROAS）
  ② 优先短 Payback 渠道（现金约束）
  ③ 高 CPI 渠道要对应更高的 LTV/CPI 门槛

当平台 ROAS 3.5x 但增量 1.8x：
  内部目标 = 平台目标 × 0.5 校准系数
  日常出价用平台信号，预算决策用增量信号
```

| 概念 | 锚点 |
|---|---|

| Meta CAPI 信号恢复 | **20–35%** — 单次最高 ROI 的技术投入 |
| D30 ROAS 及格线 | **40%** — 中核/休闲游戏的健康底线 |
| Payback 放量线 | **<30 天** — 现金引擎安全运转的硬约束 |
